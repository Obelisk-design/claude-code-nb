## 1. 概述

### QueryEngine 的设计目标

`QueryEngine.ts` 中的 `QueryEngine` 类是 Claude Code CLI 的**对话生命周期管理核心**。它从原本的 `ask()` 函数中提取核心逻辑，封装成一个可复用的独立类，用于：

- **SDK/Headless 模式**：非交互式调用场景（如自动化脚本、API 调用）
- ** REPL 交互模式**（未来扩展）：交互式终端对话

核心设计理念：

```
一个 QueryEngine 实例 = 一场完整的对话会话
每个 submitMessage() 调用 = 会话中的一个对话轮次（turn）
状态（消息、文件缓存、使用量）在轮次间持久保持
```

### 为什么需要对话引擎抽象层

原始 `ask()` 函数是一个**一次性调用**的设计，每次调用都会创建新的会话状态。这对于以下场景存在限制：

1. **多轮对话需要状态保持**：SDK 调用者希望在多次 `submitMessage` 之间保持对话上下文
2. **内存管理**：长时间运行的会话需要控制消息累积，避免内存溢出
3. **模块解耦**：将对话逻辑从 REPL/SDK 入口点分离，便于测试和维护

QueryEngine 通过**状态封装**解决了这些问题：

```typescript
// QueryEngine.ts:184-199

  private discoveredSkillNames: Set<string> // 技能发现追踪
  private loadedNestedMemoryPaths: Set<string> // 嵌套内存路径
}
```

```typescript
// QueryEngine.ts:209-271
async *submitMessage(
  prompt: string | ContentBlockParam[],
  options?: { uuid?: string; isMeta?: boolean },
): AsyncGenerator<SDKMessage, void, unknown> {
  const { cwd, commands, tools, mcpClients, ... } = this.config

  this.discoveredSkillNames.clear()  // 每轮清空技能发现
  setCwd(cwd)                        // 设置全局工作目录
  const persistSession = !isSessionPersistenceDisabled()
  const startTime = Date.now()

  // 权限检查包装器：追踪拒绝记录用于 SDK 报告
  const wrappedCanUseTool: CanUseToolFn = async (
    tool, input, toolUseContext, assistantMessage, toolUseID, forceDecision
  ) => {
    const result = await canUseTool(tool, input, toolUseContext, ...)

    // 追踪拒绝记录
    if (result.behavior !== 'allow') {
      this.permissionDenials.push({
        tool_name: sdkCompatToolName(tool.name),
        tool_use_id: toolUseID,
        tool_input: input,
      })
    }

    return result
  }
```

**关键设计点：**

1. **AsyncGenerator 返回类型**：允许流式输出消息，SDK 调用者可以实时处理
2. **wrappedCanUseTool**：包装原始权限检查函数，拦截拒绝记录用于结果报告
3. **discoveredSkillNames.clear()**：每轮清空，避免跨轮次累积导致内存增长

```typescript
// QueryEngine.ts:757-968
switch (message.type) {
  case 'tombstone':
    // 墓碑消息：控制信号，用于移除消息，跳过
    break

  case 'assistant':
    // 助手响应
    if (message.message.stop_reason != null) {
      lastStopReason = message.message.stop_reason
    }
    this.mutableMessages.push(message)
    yield* normalizeMessage(message)
    break

  case 'progress':
    // 进度消息
    this.mutableMessages.push(message)
    if (persistSession) {
      messages.push(message)
      void recordTranscript(messages)
    }
    yield* normalizeMessage(message)
    break

  case 'user':
    // 用户消息（工具结果）
    this.mutableMessages.push(message)
    yield* normalizeMessage(message)
    break

  case 'stream_event':
    // 流式事件：处理使用量统计
    if (message.event.type === 'message_start') {
      currentMessageUsage = EMPTY_USAGE
      currentMessageUsage = updateUsage(currentMessageUsage, message.event.message.usage)
    }
    if (message.event.type === 'message_delta') {
      currentMessageUsage = updateUsage(currentMessageUsage, message.event.usage)
      if (message.event.delta.stop_reason != null) {
        lastStopReason = message.event.delta.stop_reason
      }
    }
    if (message.event.type === 'message_stop') {
      // 累计当前消息使用量
      this.totalUsage = accumulateUsage(this.totalUsage, currentMessageUsage)
    }
    break

  case 'attachment':
    // 附件消息
    this.mutableMessages.push(message)
    if (persistSession) {
      messages.push(message)
      void recordTranscript(messages)
    }
    // 处理特殊附件类型
    if (message.attachment.type === 'structured_output') {
      structuredOutputFromTool = message.attachment.data
    }
    else if (message.attachment.type === 'max_turns_reached') {
      // 达到最大轮次，返回错误结果
      yield { type: 'result', subtype: 'error_max_turns', ... }
      return
    }
    break

  case 'system':
    // 系统消息：处理 snip 边界
    const snipResult = this.config.snipReplay?.(message, this.mutableMessages)
    if (snipResult !== undefined && snipResult.executed) {
      this.mutableMessages.length = 0
      this.mutableMessages.push(...snipResult.messages)
    }
    // 处理 compact_boundary：释放预压缩消息
    if (message.subtype === 'compact_boundary') {
      const mutableBoundaryIdx = this.mutableMessages.length - 1
      if (mutableBoundaryIdx > 0) {
        this.mutableMessages.splice(0, mutableBoundaryIdx)
      }
    }
    break
}
```

### queryLoop() - 核心对话循环

```typescript
// query.ts:241-280
async function* queryLoop(
  params: QueryParams,
  consumedCommandUuids: string[],
```

### 4.1 工具执行编排机制

QueryEngine 通过 `runTools()` 函数编排工具执行，这是对话循环与工具系统的桥梁：

```
模型响应(tool_use) → runTools() → Tool.execute() → 工具结果 → 继续对话
```

**核心流程：**

1. **工具发现** - 根据模型响应中的 `tool_use.name` 查找注册的工具
2. **并发执行** - 多个 `tool_use` 可并行执行（非依赖关系时）
3. **结果聚合** - 收集所有工具结果，构建 `tool_result` 消息

**简化的执行逻辑：**

```typescript
async function runTools(toolUses: ToolUse[], context: ToolUseContext) {
  const results = await Promise.all(
    toolUses.map(async (toolUse) => {
      const tool = getTool(toolUse.name)
      if (!tool) throw new Error(`Unknown tool: ${toolUse.name}`)
      
      // 工具执行（包含权限检查）
      const result = await tool.execute(toolUse.input, context)
      return { toolUseId: toolUse.id, result }
    })
  )
  return results
}
```

## 5. 从零实现指南

### 5.1 如何设计对话引擎

**核心设计原则：**

1. **状态封装**：将跨轮次状态封装在类中，而非散落在函数参数
2. **流式输出**：使用 AsyncGenerator 支持实时消息输出
3. **中断支持**：通过 AbortController 支持用户中断
4. **压缩策略**：多层次压缩防止内存溢出

**最小实现框架：**

```typescript
class SimpleQueryEngine {
  private messages: Message[] = []
  private abortController = new AbortController()

  async *submitMessage(prompt: string): AsyncGenerator<Message> {
    // 1. 添加用户消息
    const userMessage = createUserMessage({ content: prompt })
    this.messages.push(userMessage)
    yield userMessage

    // 2. 调用 API
    for await (const response of this.callAPI(this.messages)) {
      if (response.type === 'assistant') {
        this.messages.push(response)
        yield response

        // 3. 检查是否需要工具执行
        const toolUseBlocks = response.message.content
          .filter(c => c.type === 'tool_use')

        if (toolUseBlocks.length > 0) {
          // 4. 执行工具
          for (const block of toolUseBlocks) {
            const result = await this.executeTool(block)
            const toolResult = createToolResult(block.id, result)
            this.messages.push(toolResult)
            yield toolResult
          }

          // 5. 继续循环（递归调用 API）
          yield* this.submitMessage('')  // 空提示，让模型继续
        }
      }
    }
  }

  interrupt() {
    this.abortController.abort()
  }
}
```

### 5.2 消息累积策略

**Claude Code 的策略：**

```typescript
// 三层消息存储
this.mutableMessages  // 完整历史（跨轮次持久）
messages               // 本轮快照（用于 transcript）
messagesForQuery       // 压缩后消息（发送给 API）

// 累积时机
this.mutableMessages.push(message)  // 每条消息到达时立即累积

// 压缩时机
messagesForQuery = getMessagesAfterCompactBoundary(messages)  // 每轮开始时
if (compactionResult) {
  this.mutableMessages.splice(0, boundaryIdx)  // 压缩边界到达时释放
}
```

**实现建议：**

1. **分离存储层**：不要将所有消息放入一个数组
2. **延迟压缩**：在 API 调用前进行压缩，而非消息到达时
3. **边界标记**：使用 `compact_boundary` 消息标记压缩点
4. **内存释放**：压缩后主动 splice 释放旧消息

### 5.3 流式响应处理

**核心模式：**

```typescript
// 使用量统计状态机
let currentMessageUsage = EMPTY_USAGE

for await (const message of callModel(...)) {
  if (message.type === 'stream_event') {
    switch (message.event.type) {
      case 'message_start':
        currentMessageUsage = EMPTY_USAGE
        currentMessageUsage = updateUsage(currentMessageUsage, message.event.message.usage)
        break
      case 'message_delta':
        currentMessageUsage = updateUsage(currentMessageUsage, message.event.usage)
        // 注意：stop_reason 在 message_delta 中到达
        if (message.event.delta.stop_reason != null) {
          lastStopReason = message.event.delta.stop_reason
        }
        break
      case 'message_stop':
        totalUsage = accumulateUsage(totalUsage, currentMessageUsage)
        break
    }
  }
}
```

**关键点：**

- `message_start` 初始化当前消息使用量
- `message_delta` 累加增量使用量 +捕获 stop_reason
- `message_stop` 将当前消息使用量累计到总量

### 5.4 工具执行编排

**流式工具执行（StreamingToolExecutor）：**

```typescript
// 流式模式：工具在 API 响应过程中并发执行
const streamingToolExecutor = new StreamingToolExecutor(tools, canUseTool, context)

for await (const message of callModel(...)) {
  if (message.type === 'assistant') {
    // 每收到一个 tool_use 块，立即加入执行队列
    for (const block of message.message.content.filter(c => c.type === 'tool_use')) {
      streamingToolExecutor.addTool(block, message)
    }
  }
  // 同时获取已完成的结果
  for (const result of streamingToolExecutor.getCompletedResults()) {
    yield result.message
  }
}

// API 响应结束后，获取剩余结果
for await (const result of streamingToolExecutor.getRemainingResults()) {
  yield result.message
}
```

**传统模式（runTools）：**

```typescript
// API 响应完全结束后，批量执行所有工具
const toolUpdates = runTools(toolUseBlocks, assistantMessages, canUseTool, context)
for await (const update of toolUpdates) {
  yield update.message
}
```

**选择建议：**

- 流式模式：减少等待时间，适合长时间工具调用
- 传统模式：简单可控，适合工具调用较快的场景
