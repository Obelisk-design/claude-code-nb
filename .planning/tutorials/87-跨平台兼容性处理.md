# 87. 跨平台兼容性处理

**分析日期:** 2026-04-02

Claude Code CLI 是一个跨平台终端应用，支持 macOS、Linux、WSL(Windows Subsystem for Linux)和 Windows。本文档深入分析其跨平台兼容性处理架构，逐行解析关键代码。

---

## 一、跨平台概述

### 1.1 支持的平台类型

Claude Code 定义了完整的平台类型系统：

```typescript
// utils/platform.ts:7-9
export type Platform = 'macos' | 'windows' | 'wsl' | 'linux' | 'unknown'

export const SUPPORTED_PLATFORMS: Platform[] = ['macos', 'wsl']
```

**平台分类说明：**
- `macos`: macOS (darwin)
- `windows`: 原生 Windows
- `wsl`: Windows Subsystem for Linux (WSL1/WSL2)
- `linux`: 原生 Linux
- `unknown`: 未知平台

**官方支持平台**：目前官方完全支持 `macos` 和 `wsl`，Linux 和 Windows 为部分支持。

### 1.2 平台检测核心逻辑

```typescript
// utils/platform.ts:11-49
export const getPlatform = memoize((): Platform => {
  try {
    // 第一层：检测 macOS
    if (process.platform === 'darwin') {
      return 'macos'
    }

    // 第二层：检测 Windows
    if (process.platform === 'win32') {
      return 'windows'
    }

    // 第三层：检测 Linux/WSL
    if (process.platform === 'linux') {
      // 关键：通过 /proc/version 区分 WSL 和原生 Linux
      try {
        const procVersion = getFsImplementation().readFileSync(
          '/proc/version',
          { encoding: 'utf8' },
        )
        // WSL 标识检测
        if (
          procVersion.toLowerCase().includes('microsoft') ||
          procVersion.toLowerCase().includes('wsl')
        ) {
          return 'wsl'
        }
      } catch (error) {
        // 读取失败则假定是原生 Linux
        logError(error)
      }

      // 原生 Linux
      return 'linux'
    }

    // 未知平台
    return 'unknown'
  } catch (error) {
    logError(error)
    return 'unknown'
  }
})
```

**检测策略分析：**

1. **优先级检测**：按 macOS → Windows → Linux/WSL 顺序检测
2. **WSL 特殊处理**：通过读取 `/proc/version` 文件内容，查找 `microsoft` 或 `wsl` 关键字
3. **错误容错**：所有检测步骤都包裹在 try-catch 中，失败时返回 `unknown`
4. **性能优化**：使用 `memoize` 缓存结果，避免重复检测

### 1.3 WSL 版本检测

```typescript
// utils/platform.ts:51-79
export const getWslVersion = memoize((): string | undefined => {
  // 仅在 Linux 系统上检测
  if (process.platform !== 'linux') {
    return undefined
  }
  try {
    const procVersion = getFsImplementation().readFileSync('/proc/version', {
      encoding: 'utf8',
    })

    // 第一优先级：显式 WSL 版本标记 (WSL2, WSL3 等)
    const wslVersionMatch = procVersion.match(/WSL(\d+)/i)
    if (wslVersionMatch && wslVersionMatch[1]) {
      return wslVersionMatch[1]
    }

    // 第二优先级：隐式 Microsoft 标记 → WSL1
    // 原始 WSL1 格式: "4.4.0-19041-Microsoft"
    if (procVersion.toLowerCase().includes('microsoft')) {
      return '1'
    }

    return undefined
  } catch (error) {
    logError(error)
    return undefined
  }
})
```

**WSL 版本区分逻辑：**
- 显式标记 `WSL2`/`WSL3` → 返回对应版本号
- 仅含 `Microsoft` → 默认 WSL1
- 无法判断 → 返回 `undefined`

### 1.4 Linux 发行版检测

```typescript
// utils/platform.ts:81-116
export type LinuxDistroInfo = {
  linuxDistroId?: string
  linuxDistroVersion?: string
  linuxKernel?: string
}

export const getLinuxDistroInfo = memoize(
  async (): Promise<LinuxDistroInfo | undefined> => {
    if (process.platform !== 'linux') {
      return undefined
    }

    const result: LinuxDistroInfo = {
      linuxKernel: osRelease(),  // 内核版本
    }

    try {
      // 解析 /etc/os-release 文件
      const content = await readFile('/etc/os-release', 'utf8')
      for (const line of content.split('\n')) {
        const match = line.match(/^(ID|VERSION_ID)=(.*)$/)
        if (match && match[1] && match[2]) {
          const value = match[2].replace(/^"|"$/g, '')  // 去除引号
          if (match[1] === 'ID') {
            result.linuxDistroId = value  // 如: ubuntu, fedora, arch
          } else {
            result.linuxDistroVersion = value  // 如: 22.04, 39
          }
        }
      }
    } catch {
      // /etc/os-release 可能不存在于某些 Linux 系统
    }

    return result
  },
)
```

### 2.1 Keychain 安全存储

macOS 使用系统级 Keychain 服务存储敏感凭证，是最安全的平台。

```typescript

```

**Keychain 操作详解：**

```typescript
// utils/secureStorage/macOsKeychainStorage.ts:26-176
export const macOsKeychainStorage = {
  name: 'keychain',
  
  // 读取凭证
  read(): SecureStorageData | null {
    const prev = keychainCacheState.cache
    // TTL 缓存检查 (5秒)
    if (Date.now() - prev.cachedAt < KEYCHAIN_CACHE_TTL_MS) {
      return prev.data
    }

    try {
      const storageServiceName = getMacOsKeychainStorageServiceName(
        CREDENTIALS_SERVICE_SUFFIX,
      )
      const username = getUsername()
      // 使用 security 命令读取
      const result = execSyncWithDefaults_DEPRECATED(
        `security find-generic-password -a "${username}" -w -s "${storageServiceName}"`,
      )
      if (result) {
        const data = jsonParse(result)
        keychainCacheState.cache = { data, cachedAt: Date.now() }
        return data
      }
    } catch (_e) {
      // fall through
    }
    
    // 关键：Stale-while-error 策略
    // 如果之前有值且刷新失败，继续返回旧值而非 null
    // 防止 transient failure 导致 "Not logged in" 错误
    if (prev.data !== null) {
      logForDebugging('[keychain] read failed; serving stale cache', {
        level: 'warn',
      })
      keychainCacheState.cache = { data: prev.data, cachedAt: Date.now() }
      return prev.data
    }
    
    keychainCacheState.cache = { data: null, cachedAt: Date.now() }
    return null
  },
  
  // 更新凭证
  update(data: SecureStorageData): { success: boolean; warning?: string } {
    clearKeychainCache()  // 先清除缓存

    try {
      const storageServiceName = getMacOsKeychainStorageServiceName(
        CREDENTIALS_SERVICE_SUFFIX,
      )
      const username = getUsername()
      const jsonString = jsonStringify(data)

      // 安全编码：转为十六进制避免 shell 转义问题
      const hexValue = Buffer.from(jsonString, 'utf-8').toString('hex')

      // stdin 行长度限制处理 (4096 字节 BUFSIZ)
      const command = `add-generic-password -U -a "${username}" -s "${storageServiceName}" -X "${hexValue}"\n`

      let result
      if (command.length <= SECURITY_STDIN_LINE_LIMIT) {
        // 正常路径：通过 stdin 传递 (避免进程监控器看到 payload)
        result = execaSync('security', ['-i'], {
          input: command,
          stdio: ['pipe', 'pipe', 'pipe'],
          reject: false,
        })
      } else {
        // 大 payload 路径：直接使用 argv (超过 stdin 限制)
        logForDebugging(
          `Keychain payload (${jsonString.length}B JSON) exceeds security -i stdin limit; using argv`,
          { level: 'warn' },
        )
        result = execaSync(
          'security',
          [
            'add-generic-password',
            '-U',
            '-a',
            username,
            '-s',
            storageServiceName,
            '-X',
            hexValue,
          ],
          { stdio: ['ignore', 'pipe', 'pipe'], reject: false },
        )
      }

      if (result.exitCode !== 0) {
        return { success: false }
      }

      keychainCacheState.cache = { data, cachedAt: Date.now() }
      return { success: true }
    } catch (_e) {
      return { success: false }
    }
  },
  
  // 删除凭证
  delete(): boolean {
    clearKeychainCache()
    try {
      const storageServiceName = getMacOsKeychainStorageServiceName(
        CREDENTIALS_SERVICE_SUFFIX,
      )
      const username = getUsername()
      execSyncWithDefaults_DEPRECATED(
        `security delete-generic-password -a "${username}" -s "${storageServiceName}"`,
      )
      return true
    } catch (_e) {
      return false
    }
  },
} satisfies SecureStorage
```

**Keychain 安全设计要点：**
1. **stdin 优先**：payload 通过 stdin 传递，避免进程监控器(CrowdStrike等)看到明文
2. **hex 编码**：转换为十六进制，防止 shell 转义攻击
3. **行长度限制**：security -i 有 4096 字节的 fgets 缓冲区限制，超过时自动切换到 argv
4. **Stale-while-error**：刷新失败时返回旧值，防止瞬态故障导致全局登录状态丢失

### 2.2 Keychain 锁定检测

```typescript
// utils/secureStorage/macOsKeychainStorage.ts:198-231
let keychainLockedCache: boolean | undefined

/**
 * 检测 macOS Keychain 是否锁定
 * 返回 true 表示锁定状态（常见于 SSH 会话）
 * 
 * 缓存策略：进程生命周期内缓存 — security 命令约 27ms
 * 虚拟滚动重渲染时避免每条消息都 spawn 子进程
 */
export function isMacOsKeychainLocked(): boolean {
  if (keychainLockedCache !== undefined) return keychainLockedCache
  
  if (process.platform !== 'darwin') {
    keychainLockedCache = false
    return false
  }

  try {
    const result = execaSync('security', ['show-keychain-info'], {
      reject: false,
      stdio: ['ignore', 'pipe', 'pipe'],
    })
    // 退出码 36 表示 Keychain 锁定
    keychainLockedCache = result.exitCode === 36
  } catch {
    keychainLockedCache = false
  }
  return keychainLockedCache
}
```

### 2.3 防止睡眠 (caffeinate)

macOS 使用 `caffeinate` 命令防止系统在长时间操作时进入睡眠：

```typescript
// services/preventSleep.ts:1-166
const CAFFEINATE_TIMEOUT_SECONDS = 300  // 5分钟超时
const RESTART_INTERVAL_MS = 4 * 60 * 1000  // 4分钟重启间隔

function spawnCaffeinate(): void {
  if (process.platform !== 'darwin') {
    return  // 仅 macOS
  }

  try {
    // -i: 防止空闲睡眠（显示可关闭）
    // -t: 超时秒数（自愈机制，SIGKILL 后自动退出）
    caffeinateProcess = spawn(
      'caffeinate',
      ['-i', '-t', String(CAFFEINATE_TIMEOUT_SECONDS)],
      { stdio: 'ignore' },
    )

    caffeinateProcess.unref()  // 不阻止 Node 进程退出
    logForDebugging('Started caffeinate to prevent sleep')
  } catch {
    caffeinateProcess = null
  }
}

function killCaffeinate(): void {
  if (caffeinateProcess !== null) {
    const proc = caffeinateProcess
    caffeinateProcess = null
    try {
      proc.kill('SIGKILL')  // 立即终止
      logForDebugging('Stopped caffeinate, allowing sleep')
    } catch {
      // 进程可能已退出
    }
  }
}
```

**设计亮点：**
- **超时自愈**：5分钟超时，即使 Node 被 SIGKILL，caffeinate 也会自动退出
- **定期重启**：4分钟重启一次，确保持续生效
- **引用计数**：支持多个操作同时需要防睡眠

## 三、Linux 特有功能

### 3.1 Snap PowerShell 检测与绕过

Linux 上 PowerShell 可能通过 snap 安装，snap launcher 在子进程中会卡住：

```typescript
// utils/shell/powershellDetection.ts:24-57
export async function findPowerShell(): Promise<string | null> {
  const pwshPath = await which('pwsh')
  if (pwshPath) {
    // Snap launcher 问题处理
    if (getPlatform() === 'linux') {
      const resolved = await realpath(pwshPath).catch(() => pwshPath)
      // 检测 snap 路径
      if (pwshPath.startsWith('/snap/') || resolved.startsWith('/snap/')) {
        // 尝试直接二进制路径
        const direct =
          (await probePath('/opt/microsoft/powershell/7/pwsh')) ??
          (await probePath('/usr/bin/pwsh'))
        if (direct) {
          const directResolved = await realpath(direct).catch(() => direct)
          if (
            !direct.startsWith('/snap/') &&
            !directResolved.startsWith('/snap/')
          ) {
            return direct  // 使用非 snap 路径
          }
        }
      }
    }
    return pwshPath
  }

  const powershellPath = await which('powershell')
  if (powershellPath) {
    return powershellPath
  }

  return null
}
```

**Snap 问题说明：**
- snap launcher 在子进程启动时需要 snapd 初始化 confinement
- 这可能导致进程卡住数秒
- 直接使用 `/opt/microsoft/powershell/7/pwsh` 更可靠

### 3.2 Linux Glob 模式限制

Linux 的 bubblewrap 沙箱不完全支持 glob 模式：

```typescript
// utils/sandbox/sandbox-adapter.ts:597-642
function getLinuxGlobPatternWarnings(): string[] {
  const platform = getPlatform()
  if (platform !== 'linux' && platform !== 'wsl') {
    return []  // 仅 Linux/WSL
  }

  try {
    const settings = getSettings_DEPRECATED()
    if (!settings?.sandbox?.enabled) {
      return []
    }

    const permissions = settings?.permissions || {}
    const warnings: string[] = []

    // 检查路径是否包含 glob 字符（排除 trailing /**）
    const hasGlobs = (path: string): boolean => {
      const stripped = path.replace(/\/\*\*$/, '')
      return /[*?[\]]/.test(stripped)
    }

    for (const ruleString of [
      ...(permissions.allow || []),
      ...(permissions.deny || []),
    ]) {
      const rule = permissionRuleValueFromString(ruleString)
      if (
        (rule.toolName === FILE_EDIT_TOOL_NAME ||
          rule.toolName === FILE_READ_TOOL_NAME) &&
        rule.ruleContent &&
        hasGlobs(rule.ruleContent)
      ) {
        warnings.push(ruleString)  // 记录有问题的规则
      }
    }

    return warnings
  } catch (error) {
    logForDebugging(`Failed to get Linux glob pattern warnings: ${error}`)
    return []
  }
}
```

### 4.2 Windows/POSIX 路径转换

Git Bash 使用 POSIX 路径，Node.js 需要 Windows 路径，双向转换：

```typescript
// utils/windowsPaths.ts:127-173
/** Windows 路径 → POSIX 路径 */
export const windowsPathToPosixPath = memoizeWithLRU(
  (windowsPath: string): string => {
    // UNC 路径: \\server\share → //server/share
    if (windowsPath.startsWith('\\\\')) {
      return windowsPath.replace(/\\/g, '/')
    }
    
    // 驱动器路径: C:\Users\foo → /c/Users/foo
    const match = windowsPath.match(/^([A-Za-z]):[/\\]/)
    if (match) {
      const driveLetter = match[1]!.toLowerCase()
      return '/' + driveLetter + windowsPath.slice(2).replace(/\\/g, '/')
    }
    
    // 已经是 POSIX 或相对路径
    return windowsPath.replace(/\\/g, '/')
  },
)

/** POSIX 路径 → Windows 路径 */
export const posixPathToWindowsPath = memoizeWithLRU(
  (posixPath: string): string => {
    // UNC 路径: //server/share → \\server\share
    if (posixPath.startsWith('//')) {
      return posixPath.replace(/\//g, '\\')
    }
    
    // Cygdrive 格式: /cygdrive/c/... → C:\...
    const cygdriveMatch = posixPath.match(/^\/cygdrive\/([A-Za-z])(\/|$)/)
    if (cygdriveMatch) {
      const driveLetter = cygdriveMatch[1]!.toUpperCase()
      const rest = posixPath.slice(('cygdrive/' + cygdriveMatch[1]).length)
      return driveLetter + ':' + (rest || '\\').replace(/\//g, '\\')
    }
    
    // MSYS2/Git Bash 格式: /c/... → C:\...
    const driveMatch = posixPath.match(/^\/([A-Za-z])(\/|$)/)
    if (driveMatch) {
      const driveLetter = driveMatch[1]!.toUpperCase()
      const rest = posixPath.slice(2)
      return driveLetter + ':' + (rest || '\\').replace(/\//g, '\\')
    }
    
    // 已经是 Windows 或相对路径
    return posixPath.replace(/\//g, '\\')
  },
)
```

### 4.3 Windows 命令查找 (where.exe)

Windows 使用 `where.exe` 查找可执行文件，而非 Unix 的 `which`：

```typescript
// utils/which.ts:4-31
async function whichNodeAsync(command: string): Promise<string | null> {
  if (process.platform === 'win32') {
    // Windows: 使用 where.exe
    const result = await execa(`where.exe ${command}`, {
      shell: true,
      stderr: 'ignore',
      reject: false,
    })
    if (result.exitCode !== 0 || !result.stdout) {
      return null
```

```typescript
export type Platform = 'macos' | 'windows' | 'wsl' | 'linux' | 'unknown'

export const getPlatform = memoize((): Platform => {
  // 第一层: Node.js process.platform
  if (process.platform === 'darwin') return 'macos'
  if (process.platform === 'win32') return 'windows'
  
  // 第二层: Linux/WSL 区分
  if (process.platform === 'linux') {
    try {
      const procVersion = require('fs').readFileSync('/proc/version', 'utf8')
      if (procVersion.toLowerCase().includes('microsoft') ||
          procVersion.toLowerCase().includes('wsl')) {
        return 'wsl'
      }
    } catch {}
    return 'linux'
  }
  
  return 'unknown'
})

// WSL 版本检测
export const getWslVersion = memoize((): string | undefined => {
  if (process.platform !== 'linux') return undefined
  
  try {
    const procVersion = require('fs').readFileSync('/proc/version', 'utf8')
    const match = procVersion.match(/WSL(\d+)/i)
    if (match?.[1]) return match[1]
    if (procVersion.toLowerCase().includes('microsoft')) return '1'
  } catch {}
```
