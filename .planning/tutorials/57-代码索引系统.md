## 1. 系统概述

Claude Code CLI 的代码索引系统由三个核心子系统组成：

| 子系统 | 核心文件 | 功能 |
|--------|----------|------|
| **文件模糊搜索** | `native-ts/file-index/index.ts` | 高性能文件路径模糊匹配 |
| **符号提取与LSP集成** | `tools/LSPTool/*.ts` | 代码符号定义、引用、悬停信息 |
| **文件变更监控** | `services/teamMemorySync/watcher.ts` | 实时监控文件变更并同步 |

### 设计哲学

- **纯TypeScript实现**：避免原生模块依赖，保证跨平台兼容性
- **性能优先**：位图拒绝、Top-K选择、分块异步处理
- **渐进式加载**：支持查询就绪后立即搜索，后台继续索引

### 2.2 评分算法详解

文件采用 **nucleo风格模糊匹配评分**，评分公式：

```
总分数 = 匹配字符数 × 16 + 连续奖励 - 间隙惩罚 + 边界奖励 + 长度调整
```

**评分常量定义：**

```typescript
const SCORE_MATCH = 16           // 每个匹配字符基础分
const BONUS_BOUNDARY = 8         // 边界位置奖励（/、-、_、.后）
const BONUS_CAMEL = 6            // camelCase驼峰奖励
const BONUS_CONSECUTIVE = 4      // 连续匹配奖励
const BONUS_FIRST_CHAR = 8       // 首字符匹配奖励
const PENALTY_GAP_START = 3      // 间隙起始惩罚
const PENALTY_GAP_EXTENSION = 1  // 间隙扩展惩罚（每字符）
```

**边界字符定义：**

```typescript
function isBoundary(code: number): boolean {
  return (
    code === 47 ||    // '/' (Unix路径分隔符)
    code === 92 ||    // '\\' (Windows路径分隔符)
    code === 45 ||    // '-' (连字符)
    code === 95 ||    // '_' (下划线)
    code === 46 ||    // '.' (点号)
    code === 32       // 空格
  )
}
```

### 2.3 性能优化策略

#### 位图快速拒绝（Bitmap Rejection）

```typescript
// 第156-167行：位图预计算
private indexPath(i: number): void {
  const lp = this.paths[i]!.toLowerCase()
  this.lowerPaths[i] = lp
  const len = lp.length
  this.pathLens[i] = len
  let bits = 0
  for (let j = 0; j < len; j++) {
    const c = lp.charCodeAt(j)
    if (c >= 97 && c <= 122) bits |= 1 << (c - 97)  // a-z映射到26位
  }
  this.charBits[i] = bits
}

// 第209-210行：位图拒绝检查
if ((charBits[i]! & needleBitmap) !== needleBitmap) continue
```

**原理：** 每个路径预先计算包含哪些字母（a-z），查询时用位运算判断路径是否包含所有查询字符。**O(1)时间复杂度**拒绝89%不匹配路径。

#### Top-K 选择算法

```typescript
// 第201-270行：Top-K维护
const topK: { path: string; fuzzScore: number }[] = []
let threshold = -Infinity

// ... 对每个路径评分后
if (topK.length < limit) {
  topK.push({ path, fuzzScore: score })
  if (topK.length === limit) {
    topK.sort((a, b) => a.fuzzScore - b.fuzzScore)
    threshold = topK[0]!.fuzzScore  // 最小分数作为阈值
  }
} else if (score > threshold) {
  // 二分插入，移除最小元素
  let lo = 0, hi = topK.length
  while (lo < hi) {
    const mid = (lo + hi) >> 1
    if (topK[mid]!.fuzzScore < score) lo = mid + 1
    else hi = mid
  }
  topK.splice(lo, 0, { path, fuzzScore: score })
  topK.shift()  // 移除最小
  threshold = topK[0]!.fuzzScore
}
```

**优势：** 避免 O(n log n) 全排序，仅维护前 limit 个结果。

#### 分块异步处理

```typescript
// 第95-133行：异步索引构建
private async buildAsync(fileList: string[], markQueryable: () => void): Promise<void> {
  const CHUNK_MS = 4  // 每4ms让出事件循环

  }
}
```

## 3. 符号提取系统

### 3.1 基础符号提取

**文件路径：** `tools/LSPTool/symbolContext.ts`

用于在工具消息渲染时提取文件特定位置的符号名称：

```typescript
export function getSymbolAtPosition(
  filePath: string,
  line: number,      // 0-indexed
  character: number, // 0-indexed
): string | null {
  try {
    const fs = getFsImplementation()
    const absolutePath = expandPath(filePath)

    // 仅读取前64KB（覆盖约1000行代码）
    const MAX_READ_BYTES = 64 * 1024
    const { buffer, bytesRead } = fs.readSync(absolutePath, { length: MAX_READ_BYTES })
    const content = buffer.toString('utf-8', 0, bytesRead)
    const lines = content.split('\n')

    if (line < 0 || line >= lines.length) return null

    const lineContent = lines[line]
    if (!lineContent || character < 0 || character >= lineContent.length) return null

    // 符号匹配正则
    const symbolPattern = /[\w$'!]+|[+\-*/%&|^~<>=]+/g
    let match: RegExpExecArray | null

    while ((match = symbolPattern.exec(lineContent)) !== null) {
      const start = match.index
      const end = start + match[0].length
      if (character >= start && character < end) {
        return truncate(match[0], 30)  // 限制30字符
      }
    }
    return null
  } catch (error) {
    logForDebugging(`Symbol extraction failed: ${error.message}`, { level: 'warn' })
    return null
  }
}
```

**正则表达式解析：**

```typescript

```

| 分支 | 匹配内容 | 示例 |
|------|----------|------|
| `[\w$'!]+` | 标识符、Rust生命周期、宏 | `myFunction`, `'lifetime`, `macro_name!` |
| `[+\-*/%&|^~<>=]+` | 操作符和特殊符号 | `+`, `-=`, `&&`, `=>` |

**核心调用流程：**

```typescript
async call(input: Input, _context) {
  const absolutePath = expandPath(input.filePath)

  // 1. 等待LSP初始化完成
  const status = getInitializationStatus()
  if (status.status === 'pending') {
    await waitForInitialization()
  }

  // 2. 获取LSP服务器管理器
  const manager = getLspServerManager()
  if (!manager) return { data: { result: 'LSP server manager not initialized' } }

  // 3. 构造LSP请求参数
  const { method, params } = getMethodAndParams(input, absolutePath)

  // 4. 打开文件（LSP协议要求）
  if (!manager.isFileOpen(absolutePath)) {
    const fileContent = await readFile(absolutePath)
    await manager.openFile(absolutePath, fileContent)
  }

  // 5. 发送请求
  let result = await manager.sendRequest(absolutePath, method, params)

  // 6. 处理调用层级特殊情况
  if (input.operation === 'incomingCalls' || input.operation === 'outgoingCalls') {
    const callItems = result as CallHierarchyItem[]
    result = await manager.sendRequest(absolutePath, callMethod, { item: callItems[0] })
  }

  // 7. 过滤gitignore文件
  result = await filterGitIgnoredLocations(result, cwd)

  // 8. 格式化输出
  return { data: formatResult(input.operation, result, cwd) }
}
```

**位置转换（用户友好 → LSP协议）：**

```typescript
function getMethodAndParams(input: Input, absolutePath: string) {
  const uri = pathToFileURL(absolutePath).href
  // 1-based（用户输入）转 0-based（LSP协议）
  const position = {
    line: input.line - 1,
    character: input.character - 1,
  }
  return { method: 'textDocument/definition', params: { textDocument: { uri }, position } }
}
```

**初始化状态管理：**

```typescript
type InitializationState = 'not-started' | 'pending' | 'success' | 'failed'

export function initializeLspServerManager(): void {
  // bare模式跳过LSP
  if (isBareMode()) return

  // 已初始化则跳过
  if (lspManagerInstance !== undefined && initializationState !== 'failed') return

  // 创建管理器并标记为pending
  lspManagerInstance = createLSPServerManager()
  initializationState = 'pending'

  // 异步初始化（不阻塞启动）
  const currentGeneration = ++initializationGeneration
  initializationPromise = lspManagerInstance.initialize()
    .then(() => {
      if (currentGeneration === initializationGeneration) {
        initializationState = 'success'
        registerLSPNotificationHandlers(lspManagerInstance)
      }
    })
    .catch((error) => {
      if (currentGeneration === initializationGeneration) {
        initializationState = 'failed'
        initializationError = error
        lspManagerInstance = undefined
      }
    })
}
```

**Generation计数器的作用：** 防止过期的初始化Promise更新状态。当重新初始化时，generation递增，旧的Promise完成时会被忽略。

### 4.3 文件生命周期同步

```typescript
async function openFile(filePath: string, content: string): Promise<void> {
  const server = await ensureServerStarted(filePath)
  if (!server) return

  const fileUri = pathToFileURL(path.resolve(filePath)).href

  // 避免重复打开
  if (openedFiles.get(fileUri) === server.name) return

  // 从扩展名映射获取语言ID
  const ext = path.extname(filePath).toLowerCase()
  const languageId = server.config.extensionToLanguage[ext] || 'plaintext'

  await server.sendNotification('textDocument/didOpen', {
    textDocument: { uri: fileUri, languageId, version: 1, text: content }
  })
  openedFiles.set(fileUri, server.name)
}

async function changeFile(filePath: string, content: string): Promise<void> {
  const server = getServerForFile(filePath)
  if (!server || server.state !== 'running') return openFile(filePath, content)

  const fileUri = pathToFileURL(path.resolve(filePath)).href

  // 未打开则先打开
  if (openedFiles.get(fileUri) !== server.name) return openFile(filePath, content)

  await server.sendNotification('textDocument/didChange', {
    textDocument: { uri: fileUri, version: 1 },
    contentChanges: [{ text: content }]
  })
}

async function closeFile(filePath: string): Promise<void> {
  const server = getServerForFile(filePath)
  if (!server || server.state !== 'running') return

  const fileUri = pathToFileURL(path.resolve(filePath)).href
  await server.sendNotification('textDocument/didClose', { textDocument: { uri: fileUri } })
  openedFiles.delete(fileUri)  // 允许后续重新打开
}
```

## 5. 文件监控系统

**文件路径：** `services/teamMemorySync/watcher.ts`

### 5.1 设计背景

文件监控用于团队记忆同步，实时检测目录变更并触发远程同步。关键挑战：

- **避免fd泄漏**：chokidar 4+ 放弃fsevents，Bun的fs.watch fallback使用kqueue（每个文件一个fd）
- **防抖处理**：避免快速连续写入触发过多同步
- **错误抑制**：无OAuth/无仓库等情况应停止重试

### 5.2 核心实现

```typescript
const DEBOUNCE_MS = 2000  // 2秒防抖

// 监控状态
let watcher: FSWatcher | null = null
let debounceTimer: ReturnType<typeof setTimeout> | null = null
let pushInProgress = false
let hasPendingChanges = false
let currentPushPromise: Promise<void> | null = null
let pushSuppressedReason: string | null = null  // 错误抑制标记

// 防抖推送调度
function schedulePush(): void {
  if (pushSuppressedReason !== null) return  // 抑制状态下跳过
  hasPendingChanges = true
  if (debounceTimer) clearTimeout(debounceTimer)
  debounceTimer = setTimeout(() => {
    if (pushInProgress) {
      schedulePush()  // 推送进行中，重新调度
      return
    }
    currentPushPromise = executePush()
  }, DEBOUNCE_MS)
}
```

### 5.3 fs.watch 使用策略

```typescript
async function startFileWatcher(teamDir: string): Promise<void> {
  watcherStarted = true
  await mkdir(teamDir, { recursive: true })

  // 使用原生fs.watch而非chokidar
  watcher = watch(
    teamDir,
    { persistent: true, recursive: true },
    (_eventType, filename) => {
      if (filename === null) {
        schedulePush()
        return
      }

      // 检查抑制状态并处理unlink
      if (pushSuppressedReason !== null) {
        void stat(join(teamDir, filename)).catch((err) => {
          if (err.code !== 'ENOENT') return
          // 文件删除清除抑制状态（恢复操作）
          pushSuppressedReason = null
          schedulePush()
        })
        return
      }
      schedulePush()
    }
  )
}
```

**为什么使用 fs.watch：**

| 方案 | fd消耗 | 平台支持 |
|------|--------|----------|
| chokidar 4+ (kqueue) | O(files) - 500+文件=500+fd | Unix |
| fs.watch (FSEvents) | O(1) - 仅2fd | macOS |
| fs.watch (inotify) | O(subdirs) - 仅目录数 | Linux |

### 5.4 错误分类与抑制

```typescript
export function isPermanentFailure(r: TeamMemorySyncPushResult): boolean {
  // 无法自愈的错误
  if (r.errorType === 'no_oauth' || r.errorType === 'no_repo') return true

  // 4xx客户端错误（除409冲突、429限流）
  if (r.httpStatus !== undefined &&
      r.httpStatus >= 400 && r.httpStatus < 500 &&
      r.httpStatus !== 409 && r.httpStatus !== 429) {
    return true
  }
  return false
}
```

| 错误类型 | HTTP状态 | 处理策略 |
|----------|----------|----------|
| `no_oauth` | - | 永久抑制，重启会话后恢复 |
| `no_repo` | - | 永久抑制 |
| 404/403/413 | 4xx | 永久抑制 |
| 409 Conflict | 409 | 可恢复，重新pull后push |
| 429 Rate Limit | 429 | 可恢复，等待后重试 |

```typescript
while ((match = symbolPattern.exec(lineContent)) !== null) {
  const start = match.index
  const end = start + match[0].length
  if (character >= start && character < end) {
    return match[0].slice(0, 30)  // 限制长度
  }
}
```

### 7.3 实现文件监控器

**步骤1：设计防抖机制**

```typescript
const DEBOUNCE_MS = 2000
let debounceTimer: ReturnType<typeof setTimeout> | null = null

function schedulePush(): void {
  if (debounceTimer) clearTimeout(debounceTimer)
  debounceTimer = setTimeout(executePush, DEBOUNCE_MS)
}
```

**步骤2：启动fs.watch**

```typescript
import { watch } from 'fs'

function startWatcher(dir: string): void {
  watcher = watch(
    dir,
    { persistent: true, recursive: true },
    (_eventType, filename) => {
      if (filename) schedulePush()
    }
  )
}
```

**步骤3：实现错误抑制**

```typescript
let pushSuppressedReason: string | null = null

function schedulePush(): void {
  if (pushSuppressedReason !== null) return  // 抑制
  // ... 正常调度
}

async function executePush(): Promise<void> {
  try {
    const result = await push()
    if (!result.success && isPermanentFailure(result)) {
      pushSuppressedReason = result.errorType
    }
  } catch (e) {
    // 记录错误
  }
}
```
