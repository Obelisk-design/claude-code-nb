## 1. 工具系统架构概览

### 1.1 工具在对话循环中的位置

```
用户输入 → QueryEngine → API请求 → 模型响应(tool_use)
    ↓
runTools() → Tool.execute() → 工具结果 → QueryEngine
    ↓
继续对话循环（工具结果作为新消息发送给模型）
```

**工具系统是AI能力的执行层：**
- 模型通过 `tool_use` 响应指示要调用的工具
- QueryEngine 的 `runTools()` 函数编排工具执行
- 每个工具独立处理输入验证、权限检查、执行逻辑

### 1.2 工具与命令的关系

| 特性 | 工具 (Tool) | 命令 (Command) |
|:---|:---|:---|
| 触发方式 | AI模型自主决定调用 | 用户通过斜杠(`/`)主动触发 |
| 使用场景 | 根据对话上下文动态调用 | 预设的固定操作模板 |
| 灵活性 | 需要AI理解使用场景 | 参数化模板，固定流程 |
| 安全控制 | 细粒度权限检查 | 命令级别权限 |

## 2. Tool系统的设计理念

### 2.1 能力单元抽象概念

Claude Code CLI 的工具系统采用**能力单元抽象**的设计理念。每个工具（Tool）是一个独立的能力单元，封装了：

- **输入验证**（InputSchema）- 定义合法输入的结构约束
- **执行逻辑**（call方法）- 实现核心功能
- **权限检查**（checkPermissions）- 控制访问安全
- **UI渲染**（renderXxx方法）- 提供用户交互界面

这种设计将 AI 的能力边界明确化、模块化。模型通过工具描述（prompt）了解能力，通过输入 Schema 约束行为，通过权限系统控制风险。

### 1.2 工具作为AI能力单元的概念

工具系统将 AI 的操作能力抽象为三个层次：

1. **声明层**（Schema + Description）- 告知模型"我能做什么"
2. **验证层**（validateInput + checkPermissions）- 约束模型"应该怎么做"
3. **执行层**（call + mapToolResult）- 实际执行并返回结果

每个工具遵循统一的接口规范，但可以自定义实现细节。这种标准化确保了：
- 工具注册的一致性
- 权限系统的统一性
- UI渲染的可复用性

### 2.3 buildTool() 工厂函数逐行解析（第757-792行）

```typescript
const TOOL_DEFAULTS = {
  isEnabled: () => true,
  isConcurrencySafe: (_input?: unknown) => false,
  isReadOnly: (_input?: unknown) => false,
  isDestructive: (_input?: unknown) => false,
  checkPermissions: (
    input: { [key: string]: unknown },
    _ctx?: ToolUseContext,
  ): Promise<PermissionResult> =>
    Promise.resolve({ behavior: 'allow', updatedInput: input }),
  toAutoClassifierInput: (_input?: unknown) => '',
  userFacingName: (_input?: unknown) => '',
}

export function buildTool<D extends AnyToolDef>(def: D): BuiltTool<D> {
  return {
    ...TOOL_DEFAULTS,
    userFacingName: () => def.name,
    ...def,
  } as BuiltTool<D>
}
```

**工作原理：**

1. **默认值层叠机制**
   - 先展开 `TOOL_DEFAULTS` 提供安全默认值
   - 再强制设置 `userFacingName` 返回工具名称
   - 最后展开用户定义 `def`，覆盖默认值

2. **安全默认值策略**
   - `isConcurrencySafe: false` - 假设不安全，需要显式声明安全
   - `isReadOnly: false` - 假设写操作，触发权限检查
   - `checkPermissions: allow` - 传递给通用权限系统

3. **类型安全保证**
   - `ToolDef` 类型允许部分属性缺失
   - `BuiltTool` 类型保证所有属性存在
   - TypeScript 编译时确保完整性

**使用示例：**

```typescript
const MyTool = buildTool({
  name: 'my_tool',
  inputSchema: z.object({ path: z.string() }),
  async call(input, context) { ... },
  // 只需提供核心方法，其他使用默认值
})
```

### 2.4 InputSchema/OutputSchema设计

**lazySchema 工具函数** (`utils/lazySchema.ts`):

```typescript
export function lazySchema<T>(factory: () => T): () => T {
  let cached: T | undefined
  return () => (cached ??= factory())
}
```

**设计意图：**
- 延迟Schema构建到首次访问时
- 避免模块初始化时的Zod开销
- 支持条件性Schema（根据feature flag动态调整）

**典型用法：**

```typescript
const inputSchema = lazySchema(() =>
  z.strictObject({
    file_path: z.string().describe('The absolute path'),
    offset: z.number().optional(),
    limit: z.number().optional(),
  })
)

// 在Tool定义中使用 getter
get inputSchema() {
  return inputSchema()
}
```

**Schema设计原则：**

1. 使用 `z.strictObject()` 防止额外字段
2. 每个字段使用 `.describe()` 提供模型友好的描述
3. 使用 `semanticNumber` / `semanticBoolean` 包装器增强语义
4. 通过 `.omit()` 条件性移除字段（用于feature gating）

## 3. tools.ts 分析

### 3.1 getTools() 函数实现（第271-327行）

```typescript
export const getTools = (permissionContext: ToolPermissionContext): Tools => {
  // 1. Simple mode: 仅返回基础工具
  if (isEnvTruthy(process.env.CLAUDE_CODE_SIMPLE)) {
    if (isReplModeEnabled() && REPLTool) {
      const replSimple: Tool[] = [REPLTool]
      return filterToolsByDenyRules(replSimple, permissionContext)
    }
    const simpleTools: Tool[] = [BashTool, FileReadTool, FileEditTool]
    return filterToolsByDenyRules(simpleTools, permissionContext)
  }

  // 2. 获取所有基础工具
  const tools = getAllBaseTools().filter(tool => !specialTools.has(tool.name))

  // 3. 应用拒绝规则过滤
  let allowedTools = filterToolsByDenyRules(tools, permissionContext)

  // 4. REPL模式特殊处理
  if (isReplModeEnabled()) {
    allowedTools = allowedTools.filter(
      tool => !REPL_ONLY_TOOLS.has(tool.name)
    )
  }

  // 5. 检查isEnabled状态
  return allowedTools.filter((_, i) => isEnabled[i])
}
```

**处理流程：**

1. **模式判断** - Simple模式返回精简工具集
2. **工具获取** - `getAllBaseTools()` 返回完整工具列表
3. **权限过滤** - `filterToolsByDenyRules()` 移除被禁用工具
4. **功能过滤** - 检查每个工具的 `isEnabled()` 状态

#### 执行逻辑（call方法，第387-574行）

```typescript
async call(input: FileEditInput, { readFileState, ... }, ...): Promise<ToolResult<FileEditOutput>> {
  // 1. 扩展路径
  const absoluteFilePath = expandPath(file_path)
  
  // 2. 发现技能目录（异步触发）
  const newSkillDirs = await discoverSkillDirsForPaths([absoluteFilePath], cwd)
  addSkillDirectories(newSkillDirs).catch(() => {})  // fire-and-forget
  
  // 3. 文件历史备份
  if (fileHistoryEnabled()) {
    await fileHistoryTrackEdit(updateFileHistoryState, absoluteFilePath, parentMessage.uuid)
  }
  
  // 4. **原子性检查：确认文件未修改**
  const { content: originalFileContents, fileExists, encoding, lineEndings } = 
    readFileForEdit(absoluteFilePath)
  if (fileExists) {
    const lastWriteTime = getFileModificationTime(absoluteFilePath)
    const lastRead = readFileState.get(absoluteFilePath)
    if (lastWriteTime > lastRead.timestamp) {
      // 内容比对作为fallback（Windows时间戳不稳定）
      if (originalFileContents !== lastRead.content) {
        throw new Error(FILE_UNEXPECTEDLY_MODIFIED_ERROR)
      }
    }
  }
  
  // 5. 处理引号样式转换
  const actualOldString = findActualString(originalFileContents, old_string) || old_string
  const actualNewString = preserveQuoteStyle(old_string, actualOldString, new_string)
  
  // 6. 生成patch并写入
  const { patch, updatedFile } = getPatchForEdit({...})
  writeTextContent(absoluteFilePath, updatedFile, encoding, endings)
  
  // 7. 更新readFileState缓存
  readFileState.set(absoluteFilePath, {
    content: updatedFile,
    timestamp: getFileModificationTime(absoluteFilePath),
    offset: undefined,
    limit: undefined
  })
  
  // 8. LSP通知
  lspManager.changeFile(absoluteFilePath, updatedFile).catch(...)
  lspManager.saveFile(absoluteFilePath).catch(...)
  
  return { data: { filePath, oldString, newString, structuredPatch: patch, ... } }
}
```

**关键安全机制：**

1. **read-before-write强制** - `validateInput` 检查 `readFileState` 存在记录
2. **原子性时间戳检查** - 防止并发编辑冲突
3. **内容比对fallback** - Windows时间戳不稳定时的安全网
4. **文件历史备份** - 支持撤销操作

#### 验证逻辑（validateInput，第137-361行）

```typescript
async validateInput(input: FileEditInput, toolUseContext: ToolUseContext): Promise<ValidationResult> {
  // 1. 检查old_string !== new_string
  if (old_string === new_string) {
    return { result: false, message: 'No changes to make...', errorCode: 1 }
  }
  
  // 2. 检查拒绝规则
  const denyRule = matchingRuleForInput(fullFilePath, appState.toolPermissionContext, 'edit', 'deny')
  if (denyRule !== null) {
    return { result: false, message: 'File is in a denied directory...', errorCode: 2 }
  }
  
  // 3. 检查文件大小限制（防止OOM）
  const { size } = await fs.stat(fullFilePath)
  if (size > MAX_EDIT_FILE_SIZE) {
    return { result: false, message: `File too large (${formatFileSize(size)})`, errorCode: 10 }
  }
  
  // 4. 检查文件存在性
  if (fileContent === null && old_string !== '') {
    // 提供相似文件建议
    const similarFilename = findSimilarFile(fullFilePath)
    return { result: false, message: `File does not exist. Did you mean ${similarFilename}?`, errorCode: 4 }
  }
  
  // 5. 检查readFileState（必须先读取）
  const readTimestamp = toolUseContext.readFileState.get(fullFilePath)
  if (!readTimestamp || readTimestamp.isPartialView) {
    return { result: false, message: 'File has not been read yet...', errorCode: 6 }
  }
  
  // 6. 检查字符串存在性
  const actualOldString = findActualString(file, old_string)
  if (!actualOldString) {
    return { result: false, message: 'String to replace not found...', errorCode: 8 }
  }
  
  // 7. 检查多匹配情况
  const matches = file.split(actualOldString).length - 1
  if (matches > 1 && !replace_all) {
    return { result: false, message: `Found ${matches} matches...`, errorCode: 9 }
  }
  
  return { result: true, meta: { actualOldString } }
}
```

**步骤2：实现Tool定义**

```typescript
// MyTool.ts
import { buildTool, type ToolDef } from '../../Tool.js'
import { inputSchema, outputSchema, type MyToolInput, type MyToolOutput } from './types.js'
import { checkWritePermissionForTool } from '../../utils/permissions/filesystem.js'

export const MyTool = buildTool({
  name: 'my_tool',
  searchHint: 'process targets with custom logic',
  maxResultSizeChars: 50_000,  // 输出截断阈值
  
  // Schema getters
  get inputSchema() {
    return inputSchema()
  },
  get outputSchema() {
    return outputSchema()
  },
  
  // 核心方法
  async call(input: MyToolInput, context, _canUseTool, parentMessage) {
    // 实现执行逻辑
    const result = await processTarget(input.target)
    
    // 更新文件状态（如果涉及文件操作）
    context.readFileState.set(input.target, {
      content: result,
      timestamp: Date.now(),
      offset: undefined,
      limit: undefined,
    })
    
    return {
      data: {
        result: result,
        metadata: { duration: Date.now() - startTime },
      }
    }
  },
  
  async description(input) {
    return `Process target: ${input.target}`
  },
  
  async prompt({ tools, getToolPermissionContext }) {
    return 'A tool for processing targets...'
  },
  
  // 验证方法
  async validateInput(input: MyToolInput, context) {
    if (!input.target) {
      return { result: false, message: 'Target is required', errorCode: 1 }
    }
    return { result: true }
  },
  
  async checkPermissions(input, context) {
    const appState = context.getAppState()
    return checkWritePermissionForTool(MyTool, input, appState.toolPermissionContext)
  },
  
  // 状态判断方法
  isConcurrencySafe(input) {
    return true  // 可以并行执行
  },
  
  isReadOnly(input) {
    return false  // 写操作
  },
  
  toAutoClassifierInput(input) {
    return input.target  // 用于auto模式分类器
  },
  
  // UI方法
  userFacingName(input) {
    return input?.target ? `Process ${input.target}` : 'Process'
  },
  
  getToolUseSummary(input) {
    return input?.target ?? null
  },
  
  // 结果序列化
  mapToolResultToToolResultBlockParam(data: MyToolOutput, toolUseID) {
    return {
      tool_use_id: toolUseID,
      type: 'tool_result',
      content: data.result,
    }
  },
  
  // UI渲染（可选，导入分离组件）
  renderToolUseMessage,
  renderToolResultMessage,
} satisfies ToolDef<ReturnType<typeof inputSchema>, MyToolOutput>)
```

```typescript
  const appState = context.getAppState()
  
  // 检查拒绝规则
  if (isDeniedByRule(input, appState.toolPermissionContext)) {
    return {
      behavior: 'deny',
      message: 'Operation denied by permission rules',
      decisionReason: { type: 'rule', rule: denyRule }
    }

  }
```
