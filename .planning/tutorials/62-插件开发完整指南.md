## 一、插件系统架构

### 1.1 核心设计理念

Claude Code CLI 的插件系统采用 **Marketplace（市场）驱动架构**，具有以下特点：

- **Marketplace 作为插件源**：插件必须通过 Marketplace 发布和安装
- **版本化缓存**：每个插件按版本独立缓存，支持多版本共存
- **组件化设计**：插件可提供 Skills、Commands、Agents、Hooks、MCP Servers、LSP Servers 等多种组件
- **依赖管理**：支持插件间依赖声明，自动解决依赖闭包
- **安全边界**：跨 Marketplace 依赖默认被阻止，防止不受信任的代码注入

### 1.2 插件发现优先级

```
插件发现顺序 (pluginLoader.ts):
1. Marketplace-based plugins (plugin@marketplace format in settings)
2. Session-only plugins (from --plugin-dir CLI flag or SDK plugins option)
3. Built-in plugins (内置插件，用户可启用/禁用)
```

### 1.3 核心模块职责

| 模块 | 文件路径 | 职责 |
|------|----------|------|
| PluginLoader | `utils/plugins/pluginLoader.ts` | 插件发现、加载、验证、缓存管理 |
| MarketplaceManager | `utils/plugins/marketplaceManager.ts` | Marketplace 配置、缓存、插件安装 |
| LoadPluginHooks | `utils/plugins/loadPluginHooks.ts` | Hooks 配置加载和注册 |
| LoadPluginCommands | `utils/plugins/loadPluginCommands.ts` | Commands/Skills 加载和转换 |
| LoadPluginAgents | `utils/plugins/loadPluginAgents.ts` | Agents 加载和定义转换 |
| McpPluginIntegration | `utils/plugins/mcpPluginIntegration.ts` | MCP Server 配置加载 |
| DependencyResolver | `utils/plugins/dependencyResolver.ts` | 依赖解析和闭包计算 |
| ValidatePlugin | `utils/plugins/validatePlugin.ts` | 插件 Marketplace 验证 |
| PluginOptionsStorage | `utils/plugins/pluginOptionsStorage.ts` | 用户配置选项存储 |
| Reconciler | `utils/plugins/reconciler.ts` | Marketplace 意图与状态同步 |

### 1.4 插件生命周期

```
启动阶段:
┌─────────────────────────────────────────────────────────────────┐
│  1. loadAllPluginsCacheOnly()                                    │
│     - 加载已知 Marketplaces (known_marketplaces.json)            │
│     - 解析 settings.json 中的 enabledPlugins                     │
│     - 验证依赖闭包 (verifyAndDemote)                              │
│     - 返回 enabled/disabled/errors                               │
├─────────────────────────────────────────────────────────────────┤
│  2. loadPluginHooks()                                            │
│     - 从 enabled 插件加载 hooks.json                             │
│     - 注册到 STATE.registeredHooks                               │
├─────────────────────────────────────────────────────────────────┤
│  3. loadPluginCommands()                                         │
│     - 从 commands/ 目录加载 .md 文件                             │
│     - 从 skills/ 目录加载 SKILL.md                               │
│     - 转换为 Command 对象                                         │
├─────────────────────────────────────────────────────────────────┤
│  4. loadPluginAgents()                                           │
│     - 从 agents/ 目录加载 .md 文件                               │
│     - 转换为 AgentDefinition 对象                                │
├─────────────────────────────────────────────────────────────────┤
│  5. loadPluginMcpServers()                                       │
│     - 从 manifest.mcpServers 加载配置                            │
│     - 支持 .mcp.json 文件和 MCPB 包                              │
└─────────────────────────────────────────────────────────────────┘

运行阶段:
- Hooks 在生命周期事件触发时执行
- Commands 通过 Skill tool 或斜杠命令调用
- Agents 通过 Agent tool 或斜杠命令调用
- MCP Servers 作为外部工具集成
```

```typescript
export function getPluginsDirectory(): string {
```

### 3.4 命名空间规则

```typescript
// loadPluginCommands.ts - 命名计算

// Skill 文件 (SKILL.md)
// skills/frontend/SKILL.md → pluginName:frontend

// Command 文件 (普通 .md)
// commands/build.md → pluginName:build
// commands/utils/test.md → pluginName:utils:test

function getCommandNameFromFile(filePath, baseDir, pluginName) {
  const isSkill = isSkillFile(filePath)  // /^skill\.md$/i
  
  if (isSkill) {
    // Skill: 使用父目录名
    const skillDirectory = dirname(filePath)
    const commandBaseName = basename(skillDirectory)
    // 命名空间从父目录的父目录开始
    const namespace = relativePath.split('/').join(':')
    return namespace 
      ? `${pluginName}:${namespace}:${commandBaseName}`
      : `${pluginName}:${commandBaseName}`
  } else {
    // Command: 使用文件名
    const commandBaseName = basename(filePath).replace(/\.md$/, '')
    const namespace = relativePath.split('/').join(':')
    return namespace 
      ? `${pluginName}:${namespace}:${commandBaseName}`
      : `${pluginName}:${commandBaseName}`
  }
}
```

### 3.5 变量替换机制

```typescript
// loadPluginCommands.ts - getPromptForCommand

async getPromptForCommand(args, context) {
  let finalContent = content
  
  // 1. 参数替换
  finalContent = substituteArguments(finalContent, args, true, argumentNames)
  
  // 2. 插件路径变量
  finalContent = substitutePluginVariables(finalContent, {
    path: pluginPath,      // ${CLAUDE_PLUGIN_ROOT}
    source: sourceName,    // 插件源标识
  })
  
  // 3. 用户配置替换 (仅非敏感)
  if (pluginManifest.userConfig) {
    finalContent = substituteUserConfigInContent(
      finalContent,
      loadPluginOptions(sourceName),
      pluginManifest.userConfig,
    )
  }
  
  // 4. Skill 目录变量 (仅 Skill)
  if (config.isSkillMode) {
    finalContent = finalContent.replace(
      /\$\{CLAUDE_SKILL_DIR\}/g,
      dirname(file.filePath)
    )
  }
  
  // 5. Session ID
  finalContent = finalContent.replace(
    /\$\{CLAUDE_SESSION_ID\}/g,
    getSessionId()
  )
  
  // 6. Shell 命令执行
  finalContent = await executeShellCommandsInPrompt(finalContent, context)
  
  return [{ type: 'text', text: finalContent }]
}
```

### 4.5 Hook 加载流程

```typescript
// loadPluginHooks.ts - 加载流程

export const loadPluginHooks = memoize(async (): Promise<void> => {
  const { enabled } = await loadAllPluginsCacheOnly()
  const allPluginHooks: Record<HookEvent, PluginHookMatcher[]> = { ... }
  
  for (const plugin of enabled) {
    if (!plugin.hooksConfig) continue
    
    const pluginMatchers = convertPluginHooksToMatchers(plugin)
    
    // 合并到全局 Hooks
    for (const event of Object.keys(pluginMatchers) as HookEvent[]) {
      allPluginHooks[event].push(...pluginMatchers[event])
    }
  }
  
  // 原子替换: 清除旧 Hooks，注册新 Hooks
  clearRegisteredPluginHooks()
  registerHookCallbacks(allPluginHooks)
})

// 转换函数: 添加插件上下文
function convertPluginHooksToMatchers(plugin: LoadedPlugin) {
  for (const matcher of matchers) {
    pluginMatchers[event].push({
      matcher: matcher.matcher,
      hooks: matcher.hooks,
      pluginRoot: plugin.path,      // 插件根目录
      pluginName: plugin.name,      // 插件名
      pluginId: plugin.source,      // 插件 ID
    })
  }
}
```

### 4.6 热重载支持

```typescript
// loadPluginHooks.ts - 设置热重载

export function setupPluginHookHotReload(): void {
  if (hotReloadSubscribed) return
  hotReloadSubscribed = true
  
  // 捕获初始快照
  lastPluginSettingsSnapshot = getPluginAffectingSettingsSnapshot()
  
  settingsChangeDetector.subscribe(source => {
    if (source === 'policySettings') {
      const newSnapshot = getPluginAffectingSettingsSnapshot()
      if (newSnapshot === lastPluginSettingsSnapshot) {
        // 无变更，跳过重载
        return
      }
      
      lastPluginSettingsSnapshot = newSnapshot
      
      // 清除缓存
      clearPluginCache('loadPluginHooks: plugin-affecting settings changed')
      clearPluginHookCache()
      
      // 重载 Hooks
      void loadPluginHooks()
    }
  })
}
```

### 5.5 MCP Server 加载流程

```typescript
// mcpPluginIntegration.ts - 加载流程

export async function loadPluginMcpServers(
  plugin: LoadedPlugin,
  errors: PluginError[] = []
): Promise<Record<string, McpServerConfig> | undefined> {
  let servers: Record<string, McpServerConfig> = {}
  
  // 1. 检查 .mcp.json (最低优先级)
  const defaultMcpServers = await loadMcpServersFromFile(plugin.path, '.mcp.json')
  if (defaultMcpServers) {
    servers = { ...servers, ...defaultMcpServers }
  }
  
  // 2. 处理 manifest.mcpServers (更高优先级)
  if (plugin.manifest.mcpServers) {
    const mcpServersSpec = plugin.manifest.mcpServers
    
    if (typeof mcpServersSpec === 'string') {
      // MCPB 文件或 JSON 文件路径
      if (isMcpbSource(mcpServersSpec)) {
        const mcpbServers = await loadMcpServersFromMcpb(plugin, mcpServersSpec, errors)
        if (mcpbServers) servers = { ...servers, ...mcpbServers }
      } else {
        // JSON 文件路径
        const mcpServers = await loadMcpServersFromFile(plugin.path, mcpServersSpec)
        if (mcpServers) servers = { ...servers, ...mcpServers }
      }
    } else if (Array.isArray(mcpServersSpec)) {
      // 数组: 多个配置源
      const results = await Promise.all(mcpServersSpec.map(spec => ...))
      for (const result of results) {
        if (result) servers = { ...servers, ...result }
      }
    } else {
      // 内联配置对象
      servers = { ...servers, ...mcpServersSpec }
    }
  }
  
  return Object.keys(servers).length > 0 ? servers : undefined
}
```

```typescript
    }

  const mcpConfig = convertDxtToMcpConfig(manifest, extractedPath)
```

### 9.2 依赖解析

```typescript
// dependencyResolver.ts - 依赖闭包计算

export async function resolveDependencyClosure(
  rootId: PluginId,
  lookup: (id: PluginId) => Promise<DependencyLookupResult | null>,
  alreadyEnabled: ReadonlySet<PluginId>,
  allowedCrossMarketplaces: ReadonlySet<string> = new Set()
): Promise<ResolutionResult> {
  
  const closure: PluginId[] = []
  const visited = new Set<PluginId>()
  const stack: PluginId[] = []
  
  async function walk(id: PluginId, requiredBy: PluginId): Promise<ResolutionResult | null> {
    // 跳过已启用的依赖 (避免意外设置写入)
    if (id !== rootId && alreadyEnabled.has(id)) return null
    
    // 跨 Marketplace 安全检查
    const idMarketplace = parsePluginIdentifier(id).marketplace
    if (idMarketplace !== rootMarketplace && 
        !(idMarketplace && allowedCrossMarketplaces.has(idMarketplace))) {
      return {
        ok: false,
        reason: 'cross-marketplace',
        dependency: id,
        requiredBy
      }
    }
    
    // 循环检测
    if (stack.includes(id)) {
      return { ok: false, reason: 'cycle', chain: [...stack, id] }
    }
    
    if (visited.has(id)) return null
    visited.add(id)
    
    // 查找依赖
    const entry = await lookup(id)
    if (!entry) {
      return { ok: false, reason: 'not-found', missing: id, requiredBy }
    }
    
    // 递归解析
    stack.push(id)
    for (const rawDep of entry.dependencies ?? []) {
      const dep = qualifyDependency(rawDep, id)  // 规范化
      const err = await walk(dep, id)
      if (err) return err
    }
    stack.pop()
    
    closure.push(id)
    return null
  }
  
  const err = await walk(rootId, rootId)
  if (err) return err
  
  return { ok: true, closure }
}
```

```typescript
// dependencyResolver.ts - 加载时验证

export function verifyAndDemote(plugins: readonly LoadedPlugin[]): {
  demoted: Set<string>
  errors: PluginError[]
} {
  const enabled = new Set(plugins.filter(p => p.enabled).map(p => p.source))
  const errors: PluginError[] = []
  
  let changed = true
  while (changed) {  // 固定点循环
    changed = false
    for (const p of plugins) {
      if (!p.enabled) continue
      
      for (const dep of p.manifest.dependencies ?? []) {
        const qualified = qualifyDependency(dep, p.source)
        
        if (!enabled.has(qualified)) {
          // 依赖未启用 → 降级插件
          enabled.delete(p.source)
          errors.push({
            type: 'dependency-unsatisfied',
            source: p.source,
            plugin: p.name,
            dependency: dep,
            reason: enabledByName.has(depName) ? 'not-enabled' : 'not-found'
          })
          changed = true
          break
        }
      }
    }
  }
  
  return { demoted: new Set(plugins.filter(p => !p.enabled).map(p => p.source)), errors }
}
```

```typescript
export async function calculatePluginVersion(
  pluginId: string,
  source: PluginSource,
  manifest: PluginManifest,
  pluginPath: string,
  declaredVersion?: string,

  // 6. 未知版本
  return 'unknown'
```

### 12.1 路径安全

```typescript
// pluginInstallationHelpers.ts - 路径验证

export function validatePathWithinBase(basePath: string, relativePath: string): string {
  const resolvedPath = resolve(basePath, relativePath)
  const normalizedBase = resolve(basePath) + sep
  
  // 防止路径遍历攻击
  if (!resolvedPath.startsWith(normalizedBase) && resolvedPath !== resolve(basePath)) {
    throw new Error(`Path traversal detected: "${relativePath}" would escape the base directory`)
  }
  
  return resolvedPath
}
```
