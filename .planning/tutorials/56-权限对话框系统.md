# 权限对话框系统

**分析日期:** 2026-04-02

## 1. 权限对话框概述

权限对话框系统是Claude Code CLI的核心安全机制，负责在AI执行敏感操作前请求用户确认。该系统实现了细粒度的权限控制，支持多种工具类型、动态选项生成和可配置的权限规则。

### 核心架构

```
用户操作 -> Tool执行 -> PermissionSystem.check() -> 权限决策
                                              ↓
                          allow -> 直接执行
                          ask   -> PermissionRequest组件
                          deny  -> 拒绝执行
```

### 关键文件

| 文件路径 | 职责 |
|---------|-----|
| `components/permissions/PermissionRequest.tsx` | 权限请求入口，路由到具体组件 |
| `components/permissions/PermissionDialog.tsx` | 通用对话框容器 |
| `components/permissions/PermissionPrompt.tsx` | 选项渲染和用户交互 |
| `components/permissions/hooks.ts` | 日志记录和unary事件追踪 |

## 4. 审批流程

### 4.1 权限决策类型

```typescript
// 来自: utils/permissions/PermissionResult.js
type PermissionDecision = {
  behavior: 'allow' | 'ask' | 'deny' | 'passthrough';
  message?: string;
  suggestions?: PermissionUpdate[];
  decisionReason?: PermissionDecisionReason;
};
```

### 4.2 决策原因类型

```typescript
type PermissionDecisionReason = 
  | { type: 'rule'; rule: PermissionRule }
  | { type: 'mode'; mode: PermissionMode }
  | { type: 'hook'; hookName: string; reason?: string }
  | { type: 'classifier'; classifier: string; reason: string }
  | { type: 'safetyCheck'; reason: string }
  | { type: 'workingDir'; reason: string }
  | { type: 'other'; reason: string };
```

### 4.3 完整审批流程

```
1. Tool准备执行
   │
2. PermissionSystem.check(tool, input)
   │
   ├─ 检查权限规则
   │  ├─ 项目规则
   │  ├─ 用户规则  
   │  └─ 会话规则
   │
   ├─ 检查权限模式
   │  ├─ auto: 自动批准
   │  ├─ plan: 计划模式
   │  └─ acceptEdits: 接受编辑
   │
   ├─ 检查安全规则
   │  └─ 沙箱、分类器等
   │
   └─ 返回 PermissionDecision
      │
      ├─ allow -> 直接执行
      ├─ deny  -> 拒绝执行
      └─ ask   -> 显示对话框
```

  onAddRules,
  onCancel,
  ruleValues,
  ruleBehavior,
  initialContext,
  setToolPermissionContext

  const onSelect = useCallback((selectedValue) => {
    if (selectedValue === 'cancel') {
      onCancel();
      return;

    const updatedContext = applyPermissionUpdate(initialContext, {
      type: 'addRules',
      rules: ruleValues,
      behavior: ruleBehavior,

      type: 'addRules',

      sandboxAutoAllowEnabled
    });
    
    // 4. 回调通知

### 6.4 文件权限选项生成

**文件:** `components/permissions/FilePermissionDialog/permissionOptions.tsx`

```typescript
export function getFilePermissionOptions({
  filePath,
  toolPermissionContext,
  operationType = 'write',
  onRejectFeedbackChange,
  onAcceptFeedbackChange,
  yesInputMode = false,
  noInputMode = false
}): PermissionOptionWithLabel[] {
  const options: PermissionOptionWithLabel[] = [];
  
  // 1. 基础Yes选项
  if (yesInputMode && onAcceptFeedbackChange) {
    options.push({
      type: 'input',
      label: 'Yes',
      value: 'yes',
      placeholder: 'and tell Claude what to do next',
      onChange: onAcceptFeedbackChange,
      option: { type: 'accept-once' }
    });
  } else {
    options.push({
      label: 'Yes',
      value: 'yes',
      option: { type: 'accept-once' }
    });
  }
  
  // 2. 会话选项
  if (inClaudeFolder || inGlobalClaudeFolder) {
    // .claude文件夹特殊选项
    options.push({
      label: 'Yes, and allow Claude to edit its own settings for this session',
      value: 'yes-claude-folder',
      option: {
        type: 'accept-session',
        scope: inGlobalClaudeFolder ? 'global-claude-folder' : 'claude-folder'
      }
    });
  } else {
    // 普通会话选项
    options.push({
      label: 'Yes, during this session',
      value: 'yes-session',
      option: { type: 'accept-session' }
    });
  }
  
  // 3. No选项
  if (noInputMode && onRejectFeedbackChange) {
    options.push({
      type: 'input',
      label: 'No',
      value: 'no',
      placeholder: 'and tell Claude what to do differently',
      onChange: onRejectFeedbackChange,
      option: { type: 'reject' }
    });
  } else {
    options.push({
      label: 'No',
      value: 'no',
      option: { type: 'reject' }
    });
  }
  
  return options;
}
```

## 7. 从零实现

### 7.1 创建基础权限对话框

```typescript
// 1. 定义Props类型
type MyPermissionDialogProps = {
  toolUseConfirm: ToolUseConfirm;
  onDone: () => void;
  onReject: () => void;
};

// 2. 创建组件
export function MyPermissionDialog({
  toolUseConfirm,
  onDone,
  onReject
}: MyPermissionDialogProps) {
  // 定义选项
  const options: PermissionPromptOption<string>[] = [
    {
      label: 'Allow',
      value: 'allow',
      feedbackConfig: { type: 'accept' }
    },
    {
      label: 'Deny',
      value: 'deny',
      feedbackConfig: { type: 'reject' }
    }
  ];
  
  // 处理选择
  const handleSelect = useCallback((value, feedback) => {
    switch (value) {
      case 'allow':
        toolUseConfirm.onAllow(toolUseConfirm.input, [], feedback);
        onDone();
        break;
      case 'deny':
        toolUseConfirm.onReject(feedback);
        onReject();
        onDone();
        break;
    }
  }, [toolUseConfirm, onDone, onReject]);
  
  return (
    <PermissionDialog title="Permission Request">
      <PermissionPrompt
        options={options}
        onSelect={handleSelect}
        question="Do you want to proceed?"
      />
    </PermissionDialog>
  );
}
```

### 9.4 错误处理

所有权限处理函数应该：

1. 调用 `onDone()` 清理状态
2. 调用 `onReject()` 通知拒绝（如果适用）
3. 调用 `toolUseConfirm.onAllow/onReject` 传递决策
