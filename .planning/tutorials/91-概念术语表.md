## 核心概念

### A

**Agent（代理）**
- Claude 模型的角色定位，作为执行任务的主控实体
- 可通过 AgentTool 创建子代理并行处理任务
- 详见: [29-AgentTool工具详解](./29-AgentTool工具详解.md)

**AgentTool**
- 用于创建子代理的工具，支持任务委托和并行执行
- 实现了 Fork 机制，子代理有独立的上下文
- 详见: [29-AgentTool工具详解](./29-AgentTool工具详解.md)

**API Client**
- 与 Claude API 通信的客户端抽象层
- 支持多后端：Anthropic、Bedrock、Vertex、Foundry
- 详见: [47-多后端API支持](./47-多后端API支持.md)

### B

**BashTool**
- 最核心的工具，执行 Shell 命令
- 集成沙箱隔离和权限审批机制
- 详见: [27-BashTool工具详解](./27-BashTool工具详解.md)

**buildTool**
- 工厂函数，用于创建符合 Tool 接口的对象
- 封装了输入验证、权限检查、执行逻辑的绑定
- 详见: [03-工具系统设计](./03-工具系统设计.md#3-buildtool-工厂函数)

**bypassPermissions**
- 权限模式之一，完全跳过权限检查
- 仅用于紧急/调试场景，生产环境禁用
- 详见: [41-权限决策系统](./41-权限决策系统.md)

### C

**Command（命令）**
- 用户通过 `/command-name` 语法主动触发的操作
- 分为 prompt、local、local-jsx 三种类型
- 详见: [04-命令系统实现](./04-命令系统实现.md)

**Compact**
- 消息压缩机制，通过生成摘要减少 Token 使用
- 支持手动 `/compact` 和自动压缩
- 详见: [23-Token节约优化详解](./23-Token节约优化详解.md)

**Context（上下文）**
- 发送给模型的环境信息，包括文件内容、Git 状态等
- 通过 `/context` 命令可视化当前上下文
- 详见: [15-context上下文详解](./15-context上下文详解.md)

**Coordinator Mode**
- 多代理协调模式，实现任务分发和结果合并
- 用于复杂任务的并行处理
- 详见: [66-协调器模式详解](./66-协调器模式详解.md)

### D

**DANGEROUS_BASH_PATTERNS**
- 危险 Bash 命令模式列表，用于安全检测
- 包括 python、node、bash 等解释器命令
- 详见: [07-权限与安全系统](./07-权限与安全系统.md)

### F

**Feature Flag**
- 特性开关，控制功能启用/禁用
- 分为编译时（Dead Code Elimination）和运行时（GrowthBook）
- 详见: [48-FeatureFlag系统](./48-FeatureFlag系统.md)

**FileEditTool**
- 精确字符串替换编辑工具
- 实现原子性操作防止并发冲突
- 详见: [28-FileEditTool工具详解](./28-FileEditTool工具详解.md)

**FileReadTool**
- 多格式文件读取工具
- 支持文本、图像、PDF、Jupyter Notebook
- 详见: [30-FileReadTool工具详解](./30-FileReadTool工具详解.md)

### G

**GlobTool**
- 快速文件模式匹配工具
- 支持 glob 语法和修改时间排序
- 详见: [33-GlobTool工具详解](./33-GlobTool工具详解.md)

**GrepTool**
- 基于 ripgrep 的高性能内容搜索工具
- 支持正则表达式和多种输出模式
- 详见: [32-GrepTool工具详解](./32-GrepTool工具详解.md)

### H

**Hook（钩子）**
- 生命周期事件注入机制
- 包括 PreToolUse、PostToolUse、SessionStart 等事件
- 详见: [40-Hook钩子系统](./40-Hook钩子系统.md)

### I

**Ink**
- React for CLI 框架，用于终端 UI 渲染
- Claude Code 使用定制 fork 版本
- 详见: [54-Ink终端渲染器](./54-Ink终端渲染器.md)

**InputSchema**
- 工具输入的 Zod Schema 定义
- 约束模型提供的参数格式
- 详见: [03-工具系统设计](./03-工具系统设计.md)

### K

**Keybinding（键绑定）**
- 快捷键配置系统
- 支持多上下文绑定和弦序列
- 详见: [52-键绑定系统](./52-键绑定系统.md)

### L

**LocalCommand**
- 命令类型之一，本地执行返回文本结果
- 不需要 UI 交互，非交互模式也可执行
- 详见: [04-命令系统实现](./04-命令系统实现.md)

**LocalJSXCommand**
- 命令类型之一，渲染 React UI 组件
- 用于需要用户交互的复杂 UI
- 详见: [04-命令系统实现](./04-命令系统实现.md)

**lazySchema**
- 延迟加载的 Schema 定义方式
- 避免模块加载时的循环依赖
- 详见: [03-工具系统设计](./03-工具系统设计.md)

### M

**MCP（Model Context Protocol）**
- 外部工具集成协议
- 支持工具发现、资源访问、OAuth 认证
- 详见: [06-MCP集成机制](./06-MCP集成机制.md)

**Memory（记忆系统）**
- 持久化记忆机制，存储在 CLAUDE.md 文件
- 支持 team 和 personal 双轨架构
- 详见: [12-Memory记忆系统详解](./12-Memory记忆系统详解.md)

### N

**NotebookEditTool**
- Jupyter Notebook 编辑工具
- 支持单元格级别的替换、插入、删除操作
- 详见: [35-NotebookEditTool工具详解](./35-NotebookEditTool工具详解.md)

### O

**OAuth PKCE**
- 安全的 OAuth 授权流程
- 使用 code_verifier 防止授权码拦截
- 详见: [38-OAuth认证系统](./38-OAuth认证系统.md)

### P

**PermissionMode**
- 权限模式，控制工具访问权限
- 包括 default、acceptEdits、plan、auto、bypass 等
- 详见: [41-权限决策系统](./41-权限决策系统.md)

**Plugin（插件）**
- Marketplace 驱动的扩展机制
- 支持版本化缓存和组件化设计
- 详见: [62-插件开发完整指南](./62-插件开发完整指南.md)

**PromptCommand**
- 命令类型之一，返回 prompt 文本发送给模型
- 用于需要模型执行复杂任务的命令
- 详见: [04-命令系统实现](./04-命令系统实现.md)

**Prompt Caching**
- API 级别的缓存机制，减少重复 Token 计费
- 缓存系统提示词和工具定义
- 详见: [23-Token节约优化详解](./23-Token节约优化详解.md)

### Q

**QueryEngine**
- 对话生命周期管理核心类
- 封装消息累积、中断控制、状态持久化
- 详见: [02-对话引擎核心](./02-对话引擎核心.md)

### R

**REPL**
- Read-Eval-Print Loop，交互式对话界面
- 使用 Ink 框架渲染终端 UI
- 详见: [05-终端UI渲染](./05-终端UI渲染.md)

**Resume**
- 会话恢复机制，从 JSONL 转录文件恢复对话
- 支持跨项目恢复
- 详见: [25-resume会话恢复详解](./25-resume会话恢复详解.md)

**Rewind**
- 时间旅行功能，恢复到历史节点
- 支持对话和代码的双层回退
- 详见: [80-Rewind回退机制](./80-Rewind回退机制.md)

### S

**Sandbox（沙箱）**
- OS 级安全隔离机制
- macOS 使用 sandbox-exec，Linux 使用 bubblewrap
- 详见: [39-沙箱隔离系统](./39-沙箱隔离系统.md)

**Skill（技能）**
- 可复用的能力模板
- 分为 Bundled Skills 和用户自定义 Skills
- 详见: [11-内置Skills全集详解](./11-内置Skills全集详解.md)

**Store<T>**
- 轻量级状态管理，发布订阅模式
- 用于 React 组件状态共享
- 详见: [53-React状态管理](./53-React状态管理.md)

**StreamingToolExecutor**
- 流式工具执行器，支持并发执行多个工具
- 处理 SSE 事件流
- 详见: [45-流式响应处理系统](./45-流式响应处理系统.md)

### T

**Tool（工具）**
- AI 模型可调用的能力单元
- 封装输入验证、权限检查、执行逻辑
- 详见: [03-工具系统设计](./03-工具系统设计.md)

**tool_use**
- Claude API 的工具调用响应类型
- 包含工具名称和输入参数
- 详见: [02-对话引擎核心](./02-对话引擎核心.md)

### W

**WebFetchTool**
- 网页内容获取工具
- 支持 HTML→Markdown 转换和 AI 内容处理
- 详见: [31-WebFetchTool工具详解](./31-WebFetchTool工具详解.md)

**Worktree**
- Git Worktree 扩展，创建隔离开发环境
- 支持并行任务处理
- 详见: [65-Worktree工作树管理](./65-Worktree工作树管理.md)、[70-EnterWorktreeTool详解](./70-EnterWorktreeTool详解.md)

**WriteTool**
- 文件原子写入工具
- 支持并发写入冲突检测
- 详见: [34-WriteTool工具详解](./34-WriteTool工具详解.md)
