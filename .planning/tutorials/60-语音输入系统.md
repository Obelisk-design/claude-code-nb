# 语音输入系统详解

**分析日期:** 2026-04-02

## 1. 语音系统概述

### 1.1 功能定位

Claude Code CLI 的语音输入系统是一个完整的"按住说话"（Push-to-Talk）解决方案，允许用户通过语音输入命令和对话内容。系统集成了：

- **音频捕获**: 多平台麦克风录音（原生模块、SoX、arecord）
- **语音识别**: Anthropic voice_stream WebSocket API（Deepgram后端）
- **实时转录**: 流式STT，支持20+语言
- **Focus Mode**: 终端聚焦自动录音，多窗口无缝切换

#### 导入依赖 (1-6行)

**关键点**:

- GrowthBook配置使用**缓存值**（可能过期但避免异步阻塞）

#### isVoiceGrowthBookEnabled (16-23行)

```typescript
export function isVoiceGrowthBookEnabled(): boolean {
  // Positive ternary pattern — see docs/feature-gating.md.
  // Negative pattern (if (!feature(...)) return) does not eliminate
  // inline string literals from external builds.
  return feature('VOICE_MODE')
    ? !getFeatureValue_CACHED_MAY_BE_STALE('tengu_amber_quartz_disabled', false)
    : false
}
```

**设计精妙之处**:

1. **编译时消除**: 非 `ant` 构建时，`feature('VOICE_MODE')` 为 `false`，整个函数返回 `false`，相关代码被消除
2. **Kill-switch**: `tengu_amber_quartz_disabled` GrowthBook flag 可紧急关闭语音功能
3. **Positive ternary**: 正向三元表达式确保字符串字面量也被消除（避免敏感endpoint泄露）

**为什么不用 `if (!feature(...))`?**

```typescript
// ❌ 错误模式
if (!feature('VOICE_MODE')) return false
const endpoint = 'wss://api.anthropic.com/...'  // 字符串会被保留在构建产物中
```

```typescript
// ✅ 正确模式
return feature('VOICE_MODE') ? !getFeatureValue(...) : false
// 当 feature('VOICE_MODE') === false 时，整个分支被消除，字符串也消失
```

#### hasVoiceAuth (32-44行)

```typescript
export function hasVoiceAuth(): boolean {
  // Voice mode requires Anthropic OAuth — it uses the voice_stream
  // endpoint on claude.ai which is not available with API keys,
  // Bedrock, Vertex, or Foundry.
  if (!isAnthropicAuthEnabled()) {
    return false
  }
  // isAnthropicAuthEnabled only checks the auth *provider*, not whether
  // a token exists. Without this check, the voice UI renders but
  // connectVoiceStream fails silently when the user isn't logged in.
  const tokens = getClaudeAIOAuthTokens()
  return Boolean(tokens?.accessToken)
}
```

**关键点**:
- **OAuth专属**: 语音功能**仅支持 Claude.ai OAuth 认证**，不支持 API Key、Bedrock、Vertex
- **双重检查**: 先检查认证类型，再检查token是否实际存在
- **性能优化**: `getClaudeAIOAuthTokens` 已memoize，首次调用触发 macOS `security` 命令（~20-50ms）

## 3. services/voice.ts 详解

### 3.1 功能概述

这是音频捕获的核心服务，实现了**三级降级策略**：

```
┌──────────────┐
│ audio-capture│  ← 首选：原生模块（cpal）
│    -napi     │     - macOS: CoreAudio + AudioUnit
│              │     - Linux: ALSA
│              │     - Windows: WASAPI
└──────────────┘
        ↓ (失败)
┌──────────────┐
│   arecord    │  ← Linux降级：ALSA工具
│  (ALSA utils)│     仅Linux可用
└──────────────┘
        ↓ (失败/非Linux)
┌──────────────┐
│   SoX rec    │  ← 最终降级：跨平台录音工具
│              │     macOS/Linux通用
└──────────────┘
        ↓ (失败)
    ❌ 不可用
```

### 3.2 原生音频模块懒加载 (20-36行)

```typescript
type AudioNapi = typeof import('audio-capture-napi')
let audioNapi: AudioNapi | null = null
let audioNapiPromise: Promise<AudioNapi> | null = null

function loadAudioNapi(): Promise<AudioNapi> {
  audioNapiPromise ??= (async () => {
    const t0 = Date.now()
    const mod = await import('audio-capture-napi')
    // vendor/audio-capture-src/index.ts defers require(...node) until the
    // first function call — trigger it here so timing reflects real cost.
    mod.isNativeAudioAvailable()
    audioNapi = mod
    logForDebugging(`[voice] audio-capture-napi loaded in ${Date.now() - t0}ms`)
    return mod
  })()
  return audioNapiPromise
}
```

**为什么懒加载？**

注释详解（16-19行）：
```typescript
// Lazy-loaded native audio module. audio-capture.node links against
// CoreAudio.framework + AudioUnit.framework; dlopen is synchronous and
// blocks the event loop for ~1s warm, up to ~8s on cold coreaudiod
// (post-wake, post-boot). Load happens on first voice keypress — no
// preload, because there's no way to make dlopen non-blocking and a
// startup freeze is worse than a first-press delay.
```

**性能代价**:
- **暖启动**: ~1s（最近用过coreaudiod）
- **冷启动**: ~8s（重启后、唤醒后首次加载）
- **阻塞**: `dlopen` 是**同步操作**，无法用 `setImmediate` 绕过

**策略**: 首次按键时加载，避免启动卡顿。

### 3.4 arecord探测 (75-118行)

**为什么需要探测？**

注释详解（63-71行）：
```typescript
// Probe whether arecord can actually open a capture device. hasCommand()
// only checks PATH; on WSL1/Win10-WSL2/headless Linux the binary exists
// but fails at open() because there is no ALSA card and no PulseAudio
// server. On WSL2+WSLg (Win11), PulseAudio works via RDP pipes and arecord
// succeeds. We spawn with the same args as startArecordRecording() and race
// a short timer: if the process is still alive after 150ms it opened the
// device; if it exits early the stderr tells us why. Memoized — audio
// device availability does not change mid-session, and this is called on
// every voice keypress via checkRecordingAvailability().
```

**场景分析**:

| 环境 | arecord存在 | 实际可用 | 原因 |
|------|-------------|---------|------|
| WSL2+WSLg (Win11) | ✓ | ✓ | PulseAudio via RDP |
| WSL1/Win10-WSL2 | ✓ | ❌ | 无ALSA设备 |
| Headless Linux | ✓ | ❌ | 无声卡 |

**探测逻辑**:
```typescript
function probeArecord(): Promise<ArecordProbeResult> {
  arecordProbe ??= new Promise(resolve => {
    const child = spawn('arecord', [
      '-f', 'S16_LE',    // 16-bit signed little-endian
      '-r', '16000',     // 16kHz
      '-c', '1',         // mono
      '-t', 'raw',       // raw PCM
      '/dev/null',       // discard output
    ], { stdio: ['ignore', 'ignore', 'pipe'] })
    
    let stderr = ''
    child.stderr?.on('data', (chunk: Buffer) => {
      stderr += chunk.toString()
    })
    
    const timer = setTimeout((c, r) => {
      c.kill('SIGTERM')
      r({ ok: true, stderr: '' })  // 150ms后仍存活 → 设备可用
    }, 150, child, resolve)
    
    child.once('close', code => {
      clearTimeout(timer)
      void resolve({ ok: code === 0, stderr: stderr.trim() })
    })
  })
  return arecordProbe
}
```

**150ms超时**: 
- 成功打开设备 → 进程持续运行 → 超时后返回 `{ok: true}`
- 打开失败 → 进程立即退出 → 返回 `{ok: false, stderr}`

### 3.5 依赖检查 (190-227行)

```typescript
export async function checkVoiceDependencies(): Promise<{
  available: boolean
  missing: string[]
  installCommand: string | null
}> {
  // Native audio module (cpal) handles everything on macOS, Linux, and Windows
  const napi = await loadAudioNapi()
  if (napi.isNativeAudioAvailable()) {
    return { available: true, missing: [], installCommand: null }
  }

  // Windows has no supported fallback — native module is required
  if (process.platform === 'win32') {
    return {
      available: false,
      missing: ['Voice mode requires the native audio module (not loaded)'],
      installCommand: null,
    }
  }

  // On Linux, arecord (ALSA utils) is a valid fallback recording backend
  if (process.platform === 'linux' && hasCommand('arecord')) {
    return { available: true, missing: [], installCommand: null }
  }

  const missing: string[] = []
  if (!hasCommand('rec')) {
    missing.push('sox (rec command)')
  }

  const pm = missing.length > 0 ? detectPackageManager() : null
  return {
    available: missing.length === 0,
    missing,
    installCommand: pm?.displayCommand ?? null,
  }
}
```

**平台差异**:
- **Windows**: 必须 native module，无降级方案
- **Linux**: native → arecord → SoX 三级降级
- **macOS**: native → SoX 降级

```typescript
export async function checkRecordingAvailability(): Promise<RecordingAvailability> {
  // Remote environments have no local microphone
  if (isRunningOnHomespace() || isEnvTruthy(process.env.CLAUDE_CODE_REMOTE)) {
    return {
      available: false,
      reason:
        'Voice mode requires microphone access, but no audio device is available in this environment.\n\nTo use voice mode, run Claude Code locally instead.',
    }
  }

  // Native audio module (cpal) handles everything
  const napi = await loadAudioNapi()
  if (napi.isNativeAudioAvailable()) {
    return { available: true, reason: null }
  }

  // Windows has no supported fallback
  if (process.platform === 'win32') {
    return {
      available: false,
      reason:
        'Voice recording requires the native audio module, which could not be loaded.',
    }
  }

  // On Linux (including WSL), probe arecord
  if (process.platform === 'linux' && hasCommand('arecord')) {
    const probe = await probeArecord()
    if (probe.ok) {
      return { available: true, reason: null }
    }
    if (getPlatform() === 'wsl') {
      return { available: false, reason: wslNoAudioReason }
    }
    // fall through to SoX
  }

  // Fallback: check for SoX
  if (!hasCommand('rec')) {
    if (getPlatform() === 'wsl') {
      return { available: false, reason: wslNoAudioReason }
    }
    const pm = detectPackageManager()
    return {
      available: false,
      reason: pm
        ? `Voice mode requires SoX for audio recording. Install it with: ${pm.displayCommand}`
        : 'Voice mode requires SoX for audio recording. Install SoX manually...',
    }
  }

  return { available: true, reason: null }
}
```

**环境检测顺序**:
1. 远程环境检查（ Homespace / `CLAUDE_CODE_REMOTE` ）
2. Native module可用性
3. Linux + arecord探测
4. WSL特殊处理
5. SoX检查

### 3.7 录音启动 (335-396行)

```typescript
export async function startRecording(
  onData: (chunk: Buffer) => void,
  onEnd: () => void,
  options?: { silenceDetection?: boolean },
): Promise<boolean> {
  logForDebugging(`[voice] startRecording called, platform=${process.platform}`)

  // Try native audio module first
  const napi = await loadAudioNapi()
  const nativeAvailable =
    napi.isNativeAudioAvailable() &&
    (process.platform !== 'linux' || (await linuxHasAlsaCards()))
  const useSilenceDetection = options?.silenceDetection !== false
  
  if (nativeAvailable) {
    // Ensure any previous recording is fully stopped
    if (nativeRecordingActive || napi.isNativeRecordingActive()) {
      napi.stopNativeRecording()
      nativeRecordingActive = false
    }
    const started = napi.startNativeRecording(
      (data: Buffer) => {
        onData(data)
      },
      () => {
        if (useSilenceDetection) {
          nativeRecordingActive = false
          onEnd()
        }
      },
    )
    if (started) {
      nativeRecordingActive = true
      return true
    }
    // Native recording failed — fall through to platform fallbacks
  }

  // Windows has no supported fallback
  if (process.platform === 'win32') {
    return false
  }

  // On Linux, try arecord before SoX
  if (
    process.platform === 'linux' &&
    hasCommand('arecord') &&
    (await probeArecord()).ok
  ) {
    return startArecordRecording(onData, onEnd)
  }

  // Fallback: SoX rec
  return startSoxRecording(onData, onEnd, options)
}
```

**silenceDetection参数**:
- `true` (默认): 静音自动停止（自动录音模式）
- `false`: 仅手动停止（Push-to-Talk模式）

### 3.8 SoX录音实现 (398-466行)

```typescript
function startSoxRecording(
  onData: (chunk: Buffer) => void,
  onEnd: () => void,
  options?: { silenceDetection?: boolean },
): boolean {
  const useSilenceDetection = options?.silenceDetection !== false

  // Record raw PCM: 16 kHz, 16-bit signed, mono, to stdout.
  // --buffer 1024 forces SoX to flush audio in small chunks instead of
  // accumulating data in its internal buffer. Without this, SoX may buffer
  // several seconds of audio before writing anything to stdout when piped.
  const args = [
    '-q',                    // quiet
    '--buffer', '1024',      // small buffer for real-time streaming
    '-t', 'raw',             // raw PCM output
    '-r', '16000',           // sample rate
    '-e', 'signed',          // signed encoding
    '-b', '16',              // 16-bit
    '-c', '1',               // mono
    '-',                     // stdout
  ]

  // Add silence detection filter (auto-stop on silence)
  if (useSilenceDetection) {
    args.push(
      'silence',
      '1', '0.1', SILENCE_THRESHOLD,     // 开始: 0.1秒低于阈值触发
      '1', SILENCE_DURATION_SECS, SILENCE_THRESHOLD,  // 结束: 2秒静音后停止
    )
  }

  const child = spawn('rec', args, {
    stdio: ['pipe', 'pipe', 'pipe'],
  })

  activeRecorder = child

  child.stdout?.on('data', (chunk: Buffer) => {
    onData(chunk)
  })

  child.stderr?.on('data', () => {})  // Consume stderr to prevent backpressure

  child.on('close', () => {
    activeRecorder = null
    onEnd()
  })

  child.on('error', err => {
    logError(err)
    activeRecorder = null
    onEnd()
  })

  return true
}
```

**关键参数解析**:
- `--buffer 1024`: 强制小缓冲区，确保实时流（否则SoX会缓冲数秒）
- `silence` filter: `1 0.1 3%` 表示0.1秒低于3%阈值触发，`1 2.0 3%` 表示2秒静音后停止

### 3.9 arecord录音实现 (468-513行)

```typescript
function startArecordRecording(
  onData: (chunk: Buffer) => void,
  onEnd: () => void,
): boolean {
  // Record raw PCM: 16 kHz, 16-bit signed little-endian, mono, to stdout.
  // arecord does not support built-in silence detection, so this backend
  // is best suited for push-to-talk (silenceDetection: false).
  const args = [
    '-f', 'S16_LE',     // signed 16-bit little-endian
    '-r', '16000',
    '-c', '1',
    '-t', 'raw',        // raw PCM, no WAV header
    '-q',               // quiet
    '-',                // stdout
  ]

  const child = spawn('arecord', args, {
    stdio: ['pipe', 'pipe', 'pipe'],
  })

  activeRecorder = child

  child.stdout?.on('data', (chunk: Buffer) => {
    onData(chunk)
  })

  child.stderr?.on('data', () => {})

  child.on('close', () => {
    activeRecorder = null
    onEnd()
  })

  child.on('error', err => {
    logError(err)
    activeRecorder = null
    onEnd()
  })

  return true
}
```

**限制**: arecord不支持静音检测，仅适用于Push-to-Talk。

## 4. 音频捕获详解

### 4.1 音频格式

所有录音backend统一输出：
- **格式**: Raw PCM (无WAV头)
- **采样率**: 16kHz
- **位深度**: 16-bit signed
- **声道**: Mono
- **字节序**: Little-endian (S16_LE)

**计算带宽**:
```
16000 samples/s × 2 bytes/sample × 1 channel = 32,000 bytes/s ≈ 32KB/s
```

### 4.4 音频级别计算 (hooks/useVoice.ts 185-197行)

```typescript
// Compute RMS amplitude from a 16-bit signed PCM buffer and return a
// normalized 0-1 value. A sqrt curve spreads quieter levels across more
// of the visual range so the waveform uses the full set of block heights.
export function computeLevel(chunk: Buffer): number {
  const samples = chunk.length >> 1  // 16-bit = 2 bytes per sample
  if (samples === 0) return 0
  let sumSq = 0
  for (let i = 0; i < chunk.length - 1; i += 2) {
    // Read 16-bit signed little-endian
    const sample = ((chunk[i]! | (chunk[i + 1]! << 8)) << 16) >> 16
    sumSq += sample * sample
  }
  const rms = Math.sqrt(sumSq / samples)
  const normalized = Math.min(rms / 2000, 1)
  return Math.sqrt(normalized)  // sqrt curve for visual spread
}
```

**RMS计算**:
1. 读取16-bit signed little-endian样本
2. 计算平方和
3. 取平均平方根 (RMS)
4. 归一化到 0-1
5. 应用sqrt曲线（低音量分布在更大视觉范围）

### 5.4 KeepAlive心跳 (322-347行)


### 5.5 转录文本处理 (357-460行)

```typescript
ws.on('message', (raw: Buffer | string) => {
  const text = raw.toString()
  let msg: VoiceStreamMessage
  try {
    msg = jsonParse(text) as VoiceStreamMessage
  } catch {
    return
  }

  switch (msg.type) {
    case 'TranscriptText': {
      const transcript = msg.data
      
      // Disarm no-data timer when data arrives after CloseStream
      if (finalized) {
        cancelNoDataTimer?.()
      }
      
      if (transcript) {
        // Detect new speech segment (non-cumulative)
        if (!isNova3 && lastTranscriptText) {
          const prev = lastTranscriptText.trimStart()
          const next = transcript.trimStart()
          if (
            prev &&
            next &&
            !next.startsWith(prev) &&
            !prev.startsWith(next)
          ) {
            // Auto-finalize previous segment
            callbacks.onTranscript(lastTranscriptText, true)
          }
        }
        
        lastTranscriptText = transcript
        callbacks.onTranscript(transcript, false)  // interim
      }
      break
    }
    
    case 'TranscriptEndpoint': {
      const finalText = lastTranscriptText
      lastTranscriptText = ''
      if (finalText) {
        callbacks.onTranscript(finalText, true)  // final
      }
      
      // Resolve finalize() when endpoint arrives after CloseStream
      if (finalized) {
        resolveFinalize?.('post_closestream_endpoint')
      }
      break
    }
    
    case 'TranscriptError': {
      const desc = msg.description ?? msg.error_code ?? 'unknown transcription error'
      if (!finalizing) {
        callbacks.onError(desc)
      }
      break
    }
  }
})
```

**Nova 3差异**:
- Nova 2: interims是独立片段，需要auto-finalize
- Nova 3: interims是累积+可修订，不auto-finalize

### 5.6 finalize机制 (239-305行)

```typescript
finalize(): Promise<FinalizeSource> {
  if (finalizing || finalized) {
    return Promise.resolve('ws_already_closed')
  }
  finalizing = true

  return new Promise<FinalizeSource>(resolve => {
    const safetyTimer = setTimeout(
      () => resolveFinalize?.('safety_timeout'),
      FINALIZE_TIMEOUTS_MS.safety,  // 5000ms
    )
    const noDataTimer = setTimeout(
      () => resolveFinalize?.('no_data_timeout'),
      FINALIZE_TIMEOUTS_MS.noData,  // 1500ms
    )
    cancelNoDataTimer = () => {
      clearTimeout(noDataTimer)
      cancelNoDataTimer = null
    }

    resolveFinalize = (source: FinalizeSource) => {
      clearTimeout(safetyTimer)
      clearTimeout(noDataTimer)
      resolveFinalize = null
      
      // Promote unreported interim before resolve
      if (lastTranscriptText) {
        const t = lastTranscriptText
        lastTranscriptText = ''
        callbacks.onTranscript(t, true)
      }
      
      resolve(source)
    }

    // If WebSocket already closed, resolve immediately
    if (ws.readyState === WebSocket.CLOSED || ws.readyState === WebSocket.CLOSING) {
      resolveFinalize('ws_already_closed')
      return
    }

    // Defer CloseStream to next event-loop iteration
    setTimeout(() => {
      finalized = true
      if (ws.readyState === WebSocket.OPEN) {
        ws.send(CLOSE_STREAM_MSG)
      }
    }, 0)
  })
}
```

**Finalize触发源**:
- `post_closestream_endpoint`: 正常完成（收到Endpoint）
- `no_data_timeout`: 1.5秒无数据（silent-drop检测）
- `safety_timeout`: 5秒兜底超时
- `ws_close`: WebSocket已关闭
- `ws_already_closed`: 重复调用

**Deferred CloseStream**:
```typescript
setTimeout(() => {
  finalized = true
  ws.send(CLOSE_STREAM_MSG)
}, 0)
```

**原因**: 等待native模块的pending音频回调flush到WebSocket。

### 5.7 Silent-Drop检测与重试

**问题**: 约1%的session遇到CE pod故障，接受音频但返回零转录。

```typescript
class VoiceStreamClient {
  private ws: WebSocket
  private transcript: string = ''
  
  async connect(language: string): Promise<void> {
    const url = `wss://api.anthropic.com/api/ws/speech_to_text/voice_stream`
    const params = new URLSearchParams({
      encoding: 'linear16',
      sample_rate: '16000',
      channels: '1',
      language,

    })

      this.startKeepalive()

      }
      if (msg.type === 'TranscriptEndpoint') {

      this.ws.send(chunk)
    }
  }

    this.ws.send(JSON.stringify({ type: 'CloseStream' }))

    this.ws.close()
  }

class PushToTalkController {

  private sttClient: VoiceStreamClient | null
  private isRecording = false

    // 启动音频捕获
    const started = await this.recorder.start((chunk) => {
      this.sttClient?.sendAudio(chunk)
    })
    
    if (!started) {
      throw new Error('Failed to start recording')
    }
    
    // 连接STT服务
    this.sttClient = new VoiceStreamClient()
    await this.sttClient.connect('en')
    
    this.isRecording = true
  }
  
  async stopRecording(): Promise<string> {
    this.recorder.stop()
    const transcript = await this.sttClient!.finalize()
    this.sttClient!.close()
    this.sttClient = null
    this.isRecording = false
    return transcript
  }
}
```
