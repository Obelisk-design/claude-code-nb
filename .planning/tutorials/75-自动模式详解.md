## 1. 自动模式概述

### 1.1 核心概念

自动模式（Auto Mode）是Claude Code CLI的一项高级权限管理功能，使用AI分类器自动决定工具调用是否应该被批准或拒绝，减少用户交互次数。

**核心文件：**
- `utils/permissions/yoloClassifier.ts` - 分类器核心实现
- `utils/permissions/permissions.ts` - 权限决策集成
- `utils/permissions/autoModeState.ts` - 状态管理
- `utils/permissions/classifierDecision.ts` - 安全工具白名单
- `cli/handlers/autoMode.ts` - CLI命令处理
- `utils/autoModeDenials.ts` - 拒绝记录跟踪

### 1.2 工作原理

```
工具调用请求 → 权限检查 → 自动模式判断 → 分类器决策 → 执行/拒绝
                                    ↓
                            快速路径检查
                            (白名单/acceptEdits)
```

### 2.1 核心分类函数

**入口函数：`classifyYoloAction`**

```typescript
// utils/permissions/yoloClassifier.ts (行1012-1306)

export async function classifyYoloAction(
  messages: Message[],           // 会话历史
  action: TranscriptEntry,       // 待分类的操作
  tools: Tools,                  // 工具注册表
  context: ToolPermissionContext, // 权限上下文
  signal: AbortSignal,           // 取消信号
): Promise<YoloClassifierResult>
```

**处理流程：**

```typescript
export async function classifyYoloAction(...) {
  const lookup = buildToolLookup(tools)
  const actionCompact = toCompact(action, lookup)
  
  // 1. 空操作快速返回
  if (actionCompact === '') {
    return {
      shouldBlock: false,
      reason: 'Tool declares no classifier-relevant input',
      model: getClassifierModel(),
    }
  }

  // 2. 构建系统提示
  const systemPrompt = await buildYoloSystemPrompt(context)
  const transcriptEntries = buildTranscriptEntries(messages)
  const claudeMdMessage = buildClaudeMdMessage()
  
  // 3. 构建用户内容块
  const userContentBlocks: Anthropic.TextBlockParam[] = []
  for (const entry of transcriptEntries) {
    // 序列化每个transcript块
    const serialized = toCompactBlock(block, entry.role, lookup)
    userContentBlocks.push({ type: 'text', text: serialized })
  }
  
  // 4. 选择分类器模式
  const model = getClassifierModel()
  
  if (isTwoStageClassifierEnabled()) {
    // 两阶段XML分类器
    return classifyYoloActionXml(...)
  }
  
  // 5. 单阶段tool_use分类器
  const result = await sideQuery({
    model,
    max_tokens: 4096,
    system: [{ type: 'text', text: systemPrompt }],
    temperature: 0,
    messages: [...prefixMessages, { role: 'user', content: userContentBlocks }],
    tools: [YOLO_CLASSIFIER_TOOL_SCHEMA],
    tool_choice: { type: 'tool', name: YOLO_CLASSIFIER_TOOL_NAME },
    querySource: 'auto_mode',
  })
  
  // 6. 解析响应
  const toolUseBlock = extractToolUseBlock(result.content, YOLO_CLASSIFIER_TOOL_NAME)
  const parsed = parseClassifierResponse(toolUseBlock, yoloClassifierResponseSchema())
  
  return {
    shouldBlock: parsed.shouldBlock,
    reason: parsed.reason,
    model,
    usage: extractUsage(result),
    durationMs: Date.now() - start,
  }
}
```

### 2.2 两阶段XML分类器

**三种模式：**

```typescript
type TwoStageMode = 'both' | 'fast' | 'thinking'

// 'both' (默认): 先运行Stage 1快速决策，如阻止则进入Stage 2详细推理
// 'fast': 仅Stage 1，决策立即生效
// 'thinking': 仅Stage 2，跳过快速阶段
```

**Stage 1 - 快速决策：**

```typescript
// utils/permissions/yoloClassifier.ts (行769-857)

async function classifyYoloActionXml(...) {
  if (mode !== 'thinking') {
    const stage1Content = [
      ...wrappedContent,
      { type: 'text', text: XML_S1_SUFFIX },  // "\nErr on the side of blocking. <block> immediately."
    ]
    
    const stage1Opts = {
      model,
      max_tokens: (mode === 'fast' ? 256 : 64) + thinkingPadding,
      system: systemBlocks,
      temperature: 0,
      messages: [...prefixMessages, { role: 'user', content: stage1Content }],
      ...(mode !== 'fast' && { stop_sequences: ['</block>'] }),
      querySource: 'auto_mode',
    }
    
    const stage1Raw = await sideQuery(stage1Opts)
    const stage1Text = extractTextContent(stage1Raw.content)
    const stage1Block = parseXmlBlock(stage1Text)
    
    // 允许则立即返回
    if (stage1Block === false) {
      return {
        shouldBlock: false,
        reason: 'Allowed by fast classifier',
        stage: 'fast',
      }
    }
    
    // fast模式下阻止即最终
    if (mode === 'fast') {
      return {
        shouldBlock: true,
        reason: parseXmlReason(stage1Text) ?? 'Blocked by fast classifier',
        stage: 'fast',
      }
    }
  }
}
```

```typescript
// utils/permissions/yoloClassifier.ts (行578-596)

function parseXmlBlock(text: string): boolean | null {
  // 先移除thinking内容避免匹配到思考中的标签
  const stripped = stripThinking(text)
  const matches = [...stripped.matchAll(/<block>(yes|no)\b(<\/block>)?/gi)]
  if (matches.length === 0) return null
  return matches[0][1].toLowerCase() === 'yes'  // yes=阻止, no=允许
}

function parseXmlReason(text: string): string | null {
  const matches = [...stripThinking(text).matchAll(/<reason>([\s\S]*?)<\/reason>/g)]
  return matches[0]?.[1]?.trim()
}

function stripThinking(text: string): string {
  return text
    .replace(/<thinking>[\s\S]*?<\/thinking>/g, '')
    .replace(/<thinking>[\s\S]*$/, '')
}
```

```typescript
  const usingExternal = isUsingExternalPermissions()

      (_m, defaults) => userDeny ?? defaults)
```

### 2.4 Transcript构建

**消息转换为分类器输入：**

```typescript
// utils/permissions/yoloClassifier.ts (行302-360)

export function buildTranscriptEntries(messages: Message[]): TranscriptEntry[] {
  const transcript: TranscriptEntry[] = []
  
  for (const msg of messages) {
    if (msg.type === 'attachment' && msg.attachment.type === 'queued_command') {
      // 排队命令转为用户文本
      transcript.push({ role: 'user', content: [{ type: 'text', text }] })
    } else if (msg.type === 'user') {
      // 用户消息保留文本块
      transcript.push({ role: 'user', content: textBlocks })
    } else if (msg.type === 'assistant') {
      // 仅保留tool_use块（排除assistant文本）
      // assistant文本可能被模型精心构造以影响分类器
      const blocks = msg.message.content
        .filter(block => block.type === 'tool_use')
        .map(block => ({ type: 'tool_use', name: block.name, input: block.input }))
      transcript.push({ role: 'assistant', content: blocks })
    }
  }
  return transcript
}
```

**紧凑格式序列化：**

```typescript
// utils/permissions/yoloClassifier.ts (行384-424)

function toCompactBlock(
  block: TranscriptBlock,
  role: TranscriptEntry['role'],
  lookup: ToolLookup,
): string {
  if (block.type === 'tool_use') {
    const tool = lookup.get(block.name)
    if (!tool) return ''
    
    // 使用工具的toAutoClassifierInput投影
    let encoded = tool.toAutoClassifierInput(input) ?? input
    
    if (encoded === '') return ''
    
    // JSONL格式: {"Bash":"ls"}
    if (isJsonlTranscriptEnabled()) {
      return jsonStringify({ [block.name]: encoded }) + '\n'
    }
    // 文本格式: Bash ls
    return `${block.name} ${jsonStringify(encoded)}\n`
  }
  
  if (block.type === 'text' && role === 'user') {
    return isJsonlTranscriptEnabled()
      ? jsonStringify({ user: block.text }) + '\n'
      : `User: ${block.text}\n`
  }
  return ''
}
```

## 3. 分类器决策

### 3.1 决策流程集成

**权限检查中的自动模式处理：**

```typescript
// utils/permissions/permissions.ts (行518-926)

async function checkToolPermission(...) {
  // ...
  
  // 应用自动模式：使用AI分类器代替用户提示
  if (
    feature('TRANSCRIPT_CLASSIFIER') &&
    (appState.toolPermissionContext.mode === 'auto' ||
      (appState.toolPermissionContext.mode === 'plan' &&
        autoModeStateModule?.isAutoModeActive()))
  ) {
    // 1. 安全检查绕过
    if (
      result.decisionReason?.type === 'safetyCheck' &&
      !result.decisionReason.classifierApprovable
    ) {
      // 非分类器可批准的安全检查仍需交互
      return result
    }
    
    // 2. PowerShell特殊处理
    if (tool.name === POWERSHELL_TOOL_NAME && !feature('POWERSHELL_AUTO_MODE')) {
      return result  // 需要显式用户权限
    }
    
    // 3. acceptEdits快速路径
    if (result.behavior === 'ask' && tool.name !== AGENT_TOOL_NAME) {
      const acceptEditsResult = await tool.checkPermissions(parsedInput, {
        ...context,
        getAppState: () => ({
          ...state,
          toolPermissionContext: { ...state.toolPermissionContext, mode: 'acceptEdits' },
        }),
      })
      if (acceptEditsResult.behavior === 'allow') {
        return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'auto' } }
      }
    }
    
    // 4. 白名单工具直接通过
    if (classifierDecisionModule.isAutoModeAllowlistedTool(tool.name)) {
      return { behavior: 'allow', decisionReason: { type: 'mode', mode: 'auto' } }
    }
    
    // 5. 运行分类器
    const action = formatActionForClassifier(tool.name, input)
    setClassifierChecking(toolUseID)
    
    const classifierResult = await classifyYoloAction(
      context.messages,
      action,
      context.options.tools,
      appState.toolPermissionContext,
      context.abortController.signal,
    )
    
    clearClassifierChecking(toolUseID)
    
    // 6. 处理分类器结果
    if (classifierResult.shouldBlock) {
      // 见第4节：权限绕过机制
    }
    
    return {
      behavior: 'allow',
      decisionReason: { type: 'classifier', classifier: 'auto-mode', reason: classifierResult.reason },
    }
  }
}
```

## 4. 权限绕过机制

### 4.1 Iron Gate机制

**核心概念：**

Iron Gate是分类器不可用时的安全fallback策略：
- **fail-closed**: 拒绝所有操作，要求用户重试
- **fail-open**: 回退到正常权限提示

```typescript
// utils/permissions/permissions.ts (行845-876)

if (classifierResult.unavailable) {
  if (
    getFeatureValue_CACHED_WITH_REFRESH(
      'tengu_iron_gate_closed',
      true,  // 默认fail-closed
      CLASSIFIER_FAIL_CLOSED_REFRESH_MS,  // 30分钟刷新
    )
  ) {
    // fail-closed：拒绝并提示重试
    return {
      behavior: 'deny',
      decisionReason: { type: 'classifier', classifier: 'auto-mode', reason: 'Classifier unavailable' },
      message: buildClassifierUnavailableMessage(tool.name, classifierResult.model),
    }
  }
  
  // fail-open：回退到正常权限处理
  return result
}
```

```typescript
// utils/permissions/yoloClassifier.ts (行100-118)

export function getDefaultExternalAutoModeRules(): AutoModeRules {
  return {
    allow: extractTaggedBullets('user_allow_rules_to_replace'),
    soft_deny: extractTaggedBullets('user_deny_rules_to_replace'),
    environment: extractTaggedBullets('user_environment_to_replace'),
  }
}

function extractTaggedBullets(tagName: string): string[] {
  const match = EXTERNAL_PERMISSIONS_TEMPLATE.match(
    new RegExp(`<${tagName}>([\\s\\S]*?)</${tagName}>`)
  )
  return (match[1] ?? '')
    .split('\n')
    .map(line => line.trim())
    .filter(line => line.startsWith('- '))
    .map(line => line.slice(2))  // 移除"- "前缀
}
```

**模板替换语义：**

```
REPLACE语义：
- 用户提供的section完全替换默认section
- 空/缺失section保留默认值

<user_allow_rules_to_replace>
- Default rule 1
- Default rule 2
</user_allow_rules_to_replace>

用户设置allow后：
- User rule 1
- User rule 2
```

```json
  }
```
