## 一、会话存储概述

### 1.1 系统定位

会话存储系统是Claude Code CLI的**核心持久化层**，负责：

- **对话历史保存**: 每条消息写入JSONL文件
- **会话恢复**: `--resume`加载历史对话继续交互
- **元数据管理**: 会话标题、标签、代理设置等
- **压缩边界处理**: compact后的历史截断与保留
- **多代理支持**: 子代理独立transcript文件

### 1.2 存储架构

```
~/.claude/projects/
├── -Users-foo-project-a/          # 项目目录（路径hash后命名）
│   ├── abc123.jsonl               # 主会话transcript
│   ├── def456.jsonl               # 另一个会话
│   ├── def456/                    # 会话子目录
│   │   ├── subagents/             # 子代理transcript
│   │   │   ├── agent-789.jsonl
│   │   │   ├── workflows/         # workflow代理分组
│   │   │   │   ├── run-001/
│   │   │   │   │   └── agent-abc.jsonl
│   │   ├── remote-agents/         # 远程代理元数据
│   │       ├── remote-agent-task-xyz.meta.json
```

#### drainWriteQueue() - 批量写入执行

```typescript
private async drainWriteQueue(): Promise<void> {
  for (const [filePath, queue] of this.writeQueues) {
    if (queue.length === 0) continue
    
    const batch = queue.splice(0)
    let content = ''
    const resolvers: Array<() => void> = []
    
    for (const { entry, resolve } of batch) {
      const line = jsonStringify(entry) + '\n'
      
      // 分块写入（避免超过100MB的单次写入）
      if (content.length + line.length >= this.MAX_CHUNK_BYTES) {
        await this.appendToFile(filePath, content)
        for (const r of resolvers) r()
        resolvers.length = 0
        content = ''
      }
      
      content += line
      resolvers.push(resolve)
    }
    
    if (content.length > 0) {
      await this.appendToFile(filePath, content)
      for (const r of resolvers) r()
    }
  }
  
  // 清理空队列
  for (const [filePath, queue] of this.writeQueues) {
    if (queue.length === 0) {
      this.writeQueues.delete(filePath)
    }
  }
}
```

**写入策略**: 
- 100ms刷新间隔（CCR模式下10ms）
- 最大100MB单块写入
- 目录不存在时自动创建

## 三、JSONL格式详解

### 3.1 格式规范

JSONL（JSON Lines）是每行一个JSON对象的文本格式：

```
{"type":"user","uuid":"abc","parentUuid":null,"message":{...},"timestamp":"..."}
{"type":"assistant","uuid":"def","parentUuid":"abc","message":{...},"timestamp":"..."}
{"type":"custom-title","sessionId":"abc","customTitle":"我的会话"}
{"type":"tag","sessionId":"abc","tag":"重要"}
```

**优势**:
- **增量追加**: 新消息直接追加一行
- **流式解析**: 逐行解析无需加载全部
- **人类可读**: 文本格式便于调试
- **容错性**: 某行损坏不影响其他行

### 3.2 parseJSONL实现（utils/json.ts）

```typescript
// Bun优化路径（使用内置JSONL解析器）
function parseJSONLBun<T>(data: string | Buffer): T[] {
  const parse = bunJSONLParse as BunJSONLParseChunk
  const len = data.length
  const result = parse(data)
  
  if (!result.error || result.done || result.read >= len) {
    return result.values as T[]
  }
  
  // 错误恢复：跳过损坏行继续解析
  let values = result.values as T[]
  let offset = result.read
  
  while (offset < len) {
    const newlineIndex = data.indexOf('\n', offset)
    if (newlineIndex === -1) break
    offset = newlineIndex + 1
    const next = parse(data, offset)
    if (next.values.length > 0) {
      values = values.concat(next.values as T[])
    }
    if (!next.error || next.done || next.read >= len) break
    offset = next.read
  }
  
  return values
}

// 标准路径（Node.js/Bun fallback）
function parseJSONLBuffer<T>(buf: Buffer): T[] {
  const bufLen = buf.length
  let start = 0
  
  // BOM处理
  if (buf[0] === 0xef && buf[1] === 0xbb && buf[2] === 0xbf) {
    start = 3
  }
  
  const results: T[] = []
  while (start < bufLen) {
    let end = buf.indexOf(0x0a, start)
    if (end === -1) end = bufLen
    
    const line = buf.toString('utf8', start, end).trim()
    start = end + 1
    
    if (!line) continue
    try {
      results.push(JSON.parse(line) as T)
    } catch {
      // 跳过损坏行
    }
  }
  
  return results
}
```

## 四、增量写入机制

### 4.1 写入流程概览

```
用户输入 → recordTranscript()
         ↓
     insertMessageChain()
         ↓
     appendEntry()
         ↓
     enqueueWrite() [加入队列]
         ↓
     scheduleDrain() [100ms后执行]
         ↓
     drainWriteQueue() [批量写入]
         ↓
     appendToFile() [实际IO]
```

### 4.2 写入队列设计

```typescript
// 每文件独立队列
private writeQueues = new Map<
  string,
  Array<{ entry: Entry; resolve: () => void }>
>()

// 加入写入队列
private enqueueWrite(filePath: string, entry: Entry): Promise<void> {
  return new Promise<void>(resolve => {
    let queue = this.writeQueues.get(filePath)
    if (!queue) {
      queue = []
      this.writeQueues.set(filePath, queue)
    }
    queue.push({ entry, resolve })
    this.scheduleDrain()
  })
}

// 定时刷新
private scheduleDrain(): void {
  if (this.flushTimer) return
  
  this.flushTimer = setTimeout(async () => {
    this.flushTimer = null
    this.activeDrain = this.drainWriteQueue()
    await this.activeDrain
    this.activeDrain = null
    
    // 刷新期间新条目到达，再次调度
    if (this.writeQueues.size > 0) {
      this.scheduleDrain()
    }
  }, this.FLUSH_INTERVAL_MS) // 默认100ms，CCR模式10ms
}
```

### 4.3 去重机制

```typescript
async appendEntry(entry: Entry, sessionId: UUID = getSessionId() as UUID) {
  if (this.shouldSkipPersistence()) return
  
  // 当前会话缓冲处理
  if (this.sessionFile === null) {
    this.pendingEntries.push(entry)
    return
  }
  
  // Metadata类型直接追加
  if (entry.type === 'summary') {
    void this.enqueueWrite(sessionFile, entry)
  } else if (entry.type === 'custom-title') {
    void this.enqueueWrite(sessionFile, entry)
  } else {
    // Transcript消息需要去重
    const messageSet = await getSessionMessages(sessionId)
    const isNewUuid = !messageSet.has(entry.uuid)
    
    if (isNewUuid) {
      void this.enqueueWrite(targetFile, entry)
      messageSet.add(entry.uuid)  // 更新缓存
      
      // 远程持久化
      if (isTranscriptMessage(entry)) {
        await this.persistToRemote(sessionId, entry)
      }
    }
  }
}
```

**去重原因**: compact后`messagesToKeep`的UUID与pre-compact相同，跳过避免重复写入。

```typescript
export async function loadTranscriptFile(
  filePath: string,
  opts?: { keepAllLeaves?: boolean },
): Promise<{
  messages: Map<UUID, TranscriptMessage>
  summaries: Map<UUID, string>
  customTitles: Map<UUID, string>
  tags: Map<UUID, string>
  agentNames: Map<UUID, string>
  agentColors: Map<UUID, string>
  agentSettings: Map<UUID, string>
  prNumbers: Map<UUID, number>
  prUrls: Map<UUID, string>
  prRepositories: Map<UUID, string>
  modes: Map<UUID, string>
  worktreeStates: Map<UUID, PersistedWorktreeSession | null>
  fileHistorySnapshots: Map<UUID, FileHistorySnapshotMessage>
  attributionSnapshots: Map<UUID, AttributionSnapshotMessage>
  contentReplacements: Map<UUID, ContentReplacementRecord[]>
  agentContentReplacements: Map<AgentId, ContentReplacementRecord[]>
  contextCollapseCommits: ContextCollapseCommitEntry[]
  contextCollapseSnapshot: ContextCollapseSnapshotEntry | undefined
  leafUuids: Set<UUID>
}> {
  // 初始化所有Map
  const messages = new Map<UUID, TranscriptMessage>()
  const summaries = new Map<UUID, string>()
  // ...
  
  try {
    // 大文件优化：跳过pre-compact内容
    let buf: Buffer | null = null
    let metadataLines: string[] | null = null
    let hasPreservedSegment = false
    
    if (!isEnvTruthy(process.env.CLAUDE_CODE_DISABLE_PRECOMPACT_SKIP)) {
      const { size } = await stat(filePath)
      if (size > SKIP_PRECOMPACT_THRESHOLD) {
        const scan = await readTranscriptForLoad(filePath, size)
        buf = scan.postBoundaryBuf
        hasPreservedSegment = scan.hasPreservedSegment
        
        // 恢复pre-boundary元数据
        if (scan.boundaryStartOffset > 0) {
          metadataLines = await scanPreBoundaryMetadata(
            filePath,
            scan.boundaryStartOffset,
          )
        }
      }
    }
    
    buf ??= await readFile(filePath)
    
    // 死链修剪：移除fork/rewind产生的孤立分支
    if (
      !opts?.keepAllLeaves &&
      !hasPreservedSegment &&
      !isEnvTruthy(process.env.CLAUDE_CODE_DISABLE_PRECOMPACT_SKIP) &&
      buf.length > SKIP_PRECOMPACT_THRESHOLD
    ) {
      buf = walkChainBeforeParse(buf)
    }
    
    // 处理pre-boundary元数据
    if (metadataLines && metadataLines.length > 0) {
      const metaEntries = parseJSONL<Entry>(
        Buffer.from(metadataLines.join('\n')),
      )
      for (const entry of metaEntries) {
        if (entry.type === 'custom-title' && entry.sessionId) {
          customTitles.set(entry.sessionId, entry.customTitle)
        }
        // ... 其他元数据类型
      }
    }
    
    // 解析transcript
    const entries = parseJSONL<Entry>(buf)
    
    // Legacy progress桥接
    const progressBridge = new Map<UUID, UUID | null>()
    
    for (const entry of entries) {
      if (isLegacyProgressEntry(entry)) {
        // 记录progress→parent映射
        const parent = entry.parentUuid
        progressBridge.set(
          entry.uuid,
          parent && progressBridge.has(parent)
            ? (progressBridge.get(parent) ?? null)
            : parent,
        )
        continue
      }
      
      if (isTranscriptMessage(entry)) {
        // 桥接parentUuid
        if (entry.parentUuid && progressBridge.has(entry.parentUuid)) {
          entry.parentUuid = progressBridge.get(entry.parentUuid) ?? null
        }
        messages.set(entry.uuid, entry)
        
        // compact边界清空marble commits
        if (isCompactBoundaryMessage(entry)) {
          contextCollapseCommits.length = 0
          contextCollapseSnapshot = undefined
        }
      } else if (entry.type === 'summary' && entry.leafUuid) {
        summaries.set(entry.leafUuid, entry.summary)
      } else if (entry.type === 'custom-title' && entry.sessionId) {
        customTitles.set(entry.sessionId, entry.customTitle)
      }
      // ... 处理其他entry类型
    }
  } catch {
    // 文件不存在或无法读取
  }
  
  // 后处理
  applyPreservedSegmentRelinks(messages)
  applySnipRemovals(messages)
  
  // 计算leaf UUIDs
  const allMessages = [...messages.values()]
  const parentUuids = new Set(
    allMessages
      .map(msg => msg.parentUuid)
      .filter((uuid): uuid is UUID => uuid !== null),
  )
  
  const terminalMessages = allMessages.filter(msg => !parentUuids.has(msg.uuid))
  
  const leafUuids = new Set<UUID>()
  for (const terminal of terminalMessages) {
    const seen = new Set<UUID>()
    let current: TranscriptMessage | undefined = terminal
    
    while (current) {
      if (seen.has(current.uuid)) break
      seen.add(current.uuid)
      
      if (current.type === 'user' || current.type === 'assistant') {
        leafUuids.add(current.uuid)
        break
      }
      
      current = current.parentUuid
        ? messages.get(current.parentUuid)
        : undefined
    }
  }
  
  return {
    messages,
    summaries,
    customTitles,
    tags,
    agentNames,
    agentColors,
    agentSettings,
    prNumbers,
    prUrls,
    prRepositories,
    modes,
    worktreeStates,
    fileHistorySnapshots,
    attributionSnapshots,
    contentReplacements,
    agentContentReplacements,
    contextCollapseCommits,
    contextCollapseSnapshot,
    leafUuids,
  }
}
```

### 5.3 buildConversationChain - 链重建

```typescript
export function buildConversationChain(
  messages: Map<UUID, TranscriptMessage>,
  leafMessage: TranscriptMessage,
): TranscriptMessage[] {
  const transcript: TranscriptMessage[] = []
  const seen = new Set<UUID>()
  let currentMsg: TranscriptMessage | undefined = leafMessage
  
  // 从leaf向root遍历
  while (currentMsg) {
    if (seen.has(currentMsg.uuid)) {
      logError(new Error(
        `Cycle detected in parentUuid chain at message ${currentMsg.uuid}`,
      ))
      logEvent('tengu_chain_parent_cycle', {})
      break
    }
    
    seen.add(currentMsg.uuid)
    transcript.push(currentMsg)
    
    currentMsg = currentMsg.parentUuid
      ? messages.get(currentMsg.parentUuid)
      : undefined
  }
  
  // 反转：root → leaf
  transcript.reverse()
  
  // 恢复孤立的工具结果
  return recoverOrphanedParallelToolResults(messages, transcript, seen)
}
```

### 5.4 recoverOrphanedParallelToolResults - 并行工具结果恢复

**问题背景**: streaming时并行tool_use产生多个Assistant块，每个有自己的UUID，但buildConversationChain的单父遍历只保留一个分支。

```typescript
function recoverOrphanedParallelToolResults(
  messages: Map<UUID, TranscriptMessage>,
  chain: TranscriptMessage[],
  seen: Set<UUID>,
): TranscriptMessage[] {
  type ChainAssistant = Extract<TranscriptMessage, { type: 'assistant' }>
  const chainAssistants = chain.filter(
    (m): m is ChainAssistant => m.type === 'assistant',
  )
  if (chainAssistants.length === 0) return chain
  
  // 找到每个message.id组的anchor（链上最后一个）
  const anchorByMsgId = new Map<string, ChainAssistant>()
  for (const a of chainAssistants) {
    if (a.message.id) anchorByMsgId.set(a.message.id, a)
  }
  
  // 预计算兄弟组和工具结果索引
  const siblingsByMsgId = new Map<string, TranscriptMessage[]>()
  const toolResultsByAsst = new Map<UUID, TranscriptMessage[]>()
  
  for (const m of messages.values()) {
    if (m.type === 'assistant' && m.message.id) {
      const group = siblingsByMsgId.get(m.message.id)
      if (group) group.push(m)
      else siblingsByMsgId.set(m.message.id, [m])
    } else if (
      m.type === 'user' &&
      m.parentUuid &&
      Array.isArray(m.message.content) &&
      m.message.content.some(b => b.type === 'tool_result')
    ) {
      const group = toolResultsByAsst.get(m.parentUuid)
      if (group) group.push(m)
      else toolResultsByAsst.set(m.parentUuid, [m])
    }
  }
  
  // 收集孤立兄弟和工具结果
  const inserts = new Map<UUID, TranscriptMessage[]>()
  const processedGroups = new Set<string>()
  let recoveredCount = 0
  
  for (const asst of chainAssistants) {
    const msgId = asst.message.id
    if (!msgId || processedGroups.has(msgId)) continue
    processedGroups.add(msgId)
    
    const group = siblingsByMsgId.get(msgId) ?? [asst]
    const orphanedSiblings = group.filter(s => !seen.has(s.uuid))
    const orphanedTRs: TranscriptMessage[] = []
    
    for (const member of group) {
      const trs = toolResultsByAsst.get(member.uuid)
      if (!trs) continue
      for (const tr of trs) {
        if (!seen.has(tr.uuid)) orphanedTRs.push(tr)
      }
    }
    
    if (orphanedSiblings.length === 0 && orphanedTRs.length === 0) continue
    
    // 时间排序保持顺序
    orphanedSiblings.sort((a, b) => a.timestamp.localeCompare(b.timestamp))
    orphanedTRs.sort((a, b) => a.timestamp.localeCompare(b.timestamp))
    
    const anchor = anchorByMsgId.get(msgId)!
    const recovered = [...orphanedSiblings, ...orphanedTRs]
    
    for (const r of recovered) seen.add(r.uuid)
    recoveredCount += recovered.length
    inserts.set(anchor.uuid, recovered)
  }
  
  if (recoveredCount === 0) return chain
  
  // 插入恢复的消息
  const result: TranscriptMessage[] = []
  for (const m of chain) {
    result.push(m)
    const toInsert = inserts.get(m.uuid)
    if (toInsert) result.push(...toInsert)
  }
  
  return result
}
```

### 5.5 大文件优化

**问题**: 151MB session文件，92%内容是compact后的死链，parseJSONL花费47.3ms。

**解决方案**: `walkChainBeforeParse` - 解析前byte级修剪

```typescript
function walkChainBeforeParse(buf: Buffer): Buffer {
  const NEWLINE = 0x0a
  const OPEN_BRACE = 0x7b
  const PARENT_PREFIX = Buffer.from('{"parentUuid":')
  const UUID_KEY = Buffer.from('"uuid":"')
  const SIDECHAIN_TRUE = Buffer.from('"isSidechain":true')
  const UUID_LEN = 36
  const TS_SUFFIX = Buffer.from('","timestamp":"')
  
  // 索引数组：[lineStart, lineEnd, parentStart]
  const msgIdx: number[] = []
  const metaRanges: number[] = []
  const uuidToSlot = new Map<string, number>()
  
  let pos = 0
  while (pos < buf.length) {
    const nl = buf.indexOf(NEWLINE, pos)
    const lineEnd = nl === -1 ? buf.length : nl + 1
    
    // 检查是否为transcript消息（以{"parentUuid":开头）
    if (
      lineEnd - pos > PARENT_PREFIX.length &&
      buf[pos] === OPEN_BRACE &&
      buf.compare(PARENT_PREFIX, 0, PARENT_PREFIX.length, pos, pos + PARENT_PREFIX.length) === 0
    ) {
      // 提取parentUuid位置
      const parentStart =
        buf[pos + PARENT_PREFIX.length] === 0x22 // QUOTE
          ? pos + PARENT_PREFIX.length + 1
          : -1
      
      // 找到top-level uuid（通过timestamp suffix验证）
      let firstAny = -1
      let suffix0 = -1
      let suffixN: number[] | undefined
      
      let from = pos
      for (;;) {
        const next = buf.indexOf(UUID_KEY, from)
        if (next < 0 || next >= lineEnd) break
        
        if (firstAny < 0) firstAny = next
        
        const after = next + UUID_KEY.length + UUID_LEN
        if (
          after + TS_SUFFIX.length <= lineEnd &&
          buf.compare(TS_SUFFIX, 0, TS_SUFFIX.length, after, after + TS_SUFFIX.length) === 0
        ) {
          if (suffix0 < 0) suffix0 = next
          else (suffixN ??= [suffix0]).push(next)
        }
        
        from = next + UUID_KEY.length
      }
      
      const uk = suffixN
        ? pickDepthOneUuidCandidate(buf, pos, suffixN)
        : suffix0 >= 0 ? suffix0 : firstAny
      
      if (uk >= 0) {
        const uuidStart = uk + UUID_KEY.length
        const uuid = buf.toString('latin1', uuidStart, uuidStart + UUID_LEN)
        uuidToSlot.set(uuid, msgIdx.length)
        msgIdx.push(pos, lineEnd, parentStart)
      } else {
        metaRanges.push(pos, lineEnd)
      }
    } else {
      metaRanges.push(pos, lineEnd)
    }
    
    pos = lineEnd
  }
  
  // 找到leaf（最后一个非sidechain消息）
  let leafSlot = -1
  for (let i = msgIdx.length - 3; i >= 0; i -= 3) {
    const sc = buf.indexOf(SIDECHAIN_TRUE, msgIdx[i])
    if (sc === -1 || sc >= msgIdx[i + 1]) {
      leafSlot = i
      break
    }
  }
  
  if (leafSlot < 0) return buf
  
  // 从leaf向root遍历，收集keep链
  const seen = new Set<number>()
  const chain = new Set<number>()
  let chainBytes = 0
  let slot: number | undefined = leafSlot
  
  while (slot !== undefined) {
    if (seen.has(slot)) break
    seen.add(slot)
    chain.add(msgIdx[slot])
    chainBytes += msgIdx[slot + 1] - msgIdx[slot]
    
    const parentStart = msgIdx[slot + 2]
    if (parentStart < 0) break
    
    const parent = buf.toString('latin1', parentStart, parentStart + UUID_LEN)
    slot = uuidToSlot.get(parent)
  }
  
  // 只有当死链超过50%时才修剪
  if (buf.length - chainBytes < buf.length >> 1) return buf
  
  // 拼接keep链和元数据
  const parts: Buffer[] = []
  let m = 0
  for (let i = 0; i < msgIdx.length; i += 3) {
    const start = msgIdx[i]
    
    while (m < metaRanges.length && metaRanges[m] < start) {
      parts.push(buf.subarray(metaRanges[m], metaRanges[m + 1]))
      m += 2
    }
    
    if (chain.has(start)) {
      parts.push(buf.subarray(start, msgIdx[i + 1]))
    }
  }
  
  while (m < metaRanges.length) {
    parts.push(buf.subarray(metaRanges[m], metaRanges[m + 1]))
    m += 2
  }
  
  return Buffer.concat(parts)
}
```

**效果**:
- 151MB, 92%死链: 47.3ms → 9.4ms (-80%)
- 41MB, 99%死链: 56.0ms → 3.9ms (-93%)

```typescript
export function parseJSONL(data: string): Entry[] {
  const lines = data.split('\n')
  const results: Entry[] = []
  
  for (const line of lines) {
    if (!line.trim()) continue
    try {
      results.push(JSON.parse(line) as Entry)
    } catch {
      // 跳过损坏行
    }
  }
  
  return results
}
```

```typescript
    if (this.queue.length === 0) return

    fs.appendFileSync(this.filePath, content)
    this.queue = []
    this.flushTimer = null
  }
}
```

```typescript
export function buildChain(
  messages: Map<UUID, TranscriptMessage>,
  leafUuid: UUID
): TranscriptMessage[] {
  const chain: TranscriptMessage[] = []
  const seen = new Set<UUID>()
  let current = messages.get(leafUuid)
  
  while (current) {
    if (seen.has(current.uuid)) break
    seen.add(current.uuid)
    chain.push(current)
    current = current.parentUuid
      ? messages.get(current.parentUuid)
      : undefined
  }
  
  chain.reverse()
  return chain
}
```

```typescript
export async function loadSession(filePath: string): Promise<...> {
  const { size } = await fs.stat(filePath)
  
  // 超过5MB只读取最后部分
  if (size > 5 * 1024 * 1024) {
    const fd = await fs.open(filePath, 'r')
    const buf = Buffer.allocUnsafe(1024 * 1024)
    await fd.read(buf, 0, buf.length, size - buf.length)
    await fd.close()
    
    // 找到compact边界
    const boundaryIdx = buf.indexOf('{"type":"system","subtype":"compact_boundary"')
    if (boundaryIdx >= 0) {
      return parseJSONL(buf.toString('utf-8', boundaryIdx))
    }
  }
  
  // 正常读取
  const data = await fs.readFile(filePath, 'utf-8')
  return parseJSONL(data)
}
```

## 八、关键技术点总结

### 8.1 设计亮点

**1. 增量追加架构**
- 避免重写整个文件
- 每条消息独立一行
- 100ms批量写入平衡性能与实时性

**2. parentUuid链结构**
- 从leaf向root重建对话
- 支持fork/rewind的多分支
- compact边界截断链

**3. 大文件优化**
- 只读post-compact部分
- byte级死链修剪（-93%时间）
- 流式解析避免OOM

**4. 元数据分离**
- 独立entry类型
- cleanup时重追加保证可见
- SDK/CLI双向写入协调

### 8.2 性能优化清单

| 优化项 | 方法 | 效果 |
|--------|------|------|
| 写入延迟 | 100ms队列 | 减少IO次数 |
| 大文件加载 | 只读post-compact | 避免5MB+加载 |
| 死链修剪 | walkChainBeforeParse | -93%解析时间 |
| JSONL解析 | Bun.JSONL.parseChunk | 比JSON.parse快 |
| Metadata读取 | 64KB head/tail | 不加载全文件 |
| UUID去重 | Set缓存 | 避免重复写入 |

### 8.3 常见问题排查

**问题1: 会话文件未创建**
- 原因: 启动后未发送消息
- 解决: 发送首条user消息触发materializeSessionFile

**问题2: --resume加载历史不完整**
- 原因: compact边界后的preserved segment处理错误
- 解决: 检查applyPreservedSegmentRelinks逻辑

**问题3: 标题丢失**
- 原因: 元数据在tail窗口外
- 解决: reAppendSessionMetadata重追加
