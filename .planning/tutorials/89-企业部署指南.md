## 企业部署概述

### 架构设计理念

Claude Code CLI的企业部署系统采用**多层策略优先级**架构，实现了灵活的企业管理能力：

```
优先级（从高到低）:
┌─────────────────────────────────────────────────────┐
│  Remote Managed Settings (API远程策略)              │ ← 最高优先级
│  - 企业云端推送                                      │
│  - 实时同步更新                                      │
│  - 适用于SaaS企业管理                                │
├─────────────────────────────────────────────────────┤
│  MDM配置 (macOS plist / Windows HKLM注册表)        │ ← 系统级管理
│  - macOS: /Library/Managed Preferences/            │
│  - Windows: HKLM\SOFTWARE\Policies\ClaudeCode      │
│  - 需要管理员/root权限                              │
├─────────────────────────────────────────────────────┤
│  文件策略 (managed-settings.json + drop-ins)       │ ← 本地文件管理
│  - Linux: /etc/claude-code/                        │
│  - macOS: /Library/Application Support/ClaudeCode/ │
│  - Windows: C:\Program Files\ClaudeCode\           │
│  - 支持managed-settings.d/目录碎片化配置           │
├─────────────────────────────────────────────────────┤
│  HKCU注册表 (Windows用户级策略)                     │ ← 用户级管理
│  - HKCU\SOFTWARE\Policies\ClaudeCode              │
│  - 用户可写，最低优先级                             │
└─────────────────────────────────────────────────────┘
```

**核心特性:**

1. **First-Source-Wins策略**：首个存在内容的源提供所有策略，避免配置冲突
2. **热加载机制**：策略变更自动生效，无需重启应用
3. **安全对话框**：危险配置变更需用户确认
4. **碎片化配置**：支持多个独立策略文件合并（如`10-security.json`, `20-otel.json`）

### 企业部署场景对比

| 场景 | 推荐方案 | 适用平台 | 管理方式 |
|------|---------|---------|---------|
| **SaaS企业管理** | Remote Managed Settings | 全平台 | API推送，实时同步 |
| **Active Directory域控** | MDM + Group Policy | Windows | GPO推送注册表 |
| **Jamf/Munki管理** | MDM plist | macOS | MDM工具推送 |
| **传统IT运维** | managed-settings.json | Linux/Unix | 配置文件分发 |
| **混合云环境** | Remote + 文件策略 | 全平台 | 双重保障 |
| **开发团队自治** | managed-settings.d/ | 全平台 | 多团队独立配置 |

#### plutil读取机制

源码位置：`utils/settings/mdm/rawRead.ts` 第54-88行

```typescript
// macOS通过plutil命令读取plist配置
export function fireRawRead(): Promise<RawReadResult> {
  if (process.platform === 'darwin') {
    const plistPaths = getMacOSPlistPaths()
    
    // 并行读取所有可能的plist路径
    const allResults = await Promise.all(
      plistPaths.map(async ({ path, label }) => {
        // 快速路径：文件不存在时跳过子进程
        if (!existsSync(path)) {
          return { stdout: '', label, ok: false }
        }
        
        // 使用plutil转换为JSON
        const { stdout, code } = await execFilePromise(PLUTIL_PATH, [
          ...PLUTIL_ARGS_PREFIX,  // ['-convert', 'json', '-o', '-', '--']
          path,
        ])
        return { stdout, label, ok: code === 0 && !!stdout }
      }),
    )
    
    // First-Source-Wins: 选取第一个成功的结果
    const winner = allResults.find(r => r.ok)
    return { plistStdouts: winner ? [{ stdout: winner.stdout, label: winner.label }] : [] }
  }
}
```

**关键优化:**
- 使用`existsSync()`快速检查避免不必要的子进程启动（约5ms开销）
- 并行读取多个plist路径提升启动速度
- **在main.tsx模块加载时立即启动**，与剩余模块加载并行执行

```typescript
// Windows注册表路径（WOW64共享键，避免重定向问题）
export const WINDOWS_REGISTRY_KEY_PATH_HKLM =
  'HKLM\\SOFTWARE\\Policies\\ClaudeCode'  // 管理员级策略

export const WINDOWS_REGISTRY_KEY_PATH_HKCU =
  'HKCU\\SOFTWARE\\Policies\\ClaudeCode'  // 用户级策略（最低优先级）

export const WINDOWS_REGISTRY_VALUE_NAME = 'Settings'  // JSON值名称
```

**重要说明:**
- HKLM路径位于`SOFTWARE\Policies`，这是WOW64共享键列表
- 32位和64位进程都能读取相同值，无需处理注册表重定向
- **不要移动到`SOFTWARE\ClaudeCode`**，该路径会被WOW6432Node重定向

#### Group Policy配置

1. 打开Group Policy Management Editor
2. 导航到：`Computer Configuration → Preferences → Windows Settings → Registry`
3. 创建新注册表项：

```
Action: Update
Hive: HKLM
Key Path: SOFTWARE\Policies\ClaudeCode
Value Name: Settings
Value Type: REG_SZ
Value Data: {"permissions":{"allow":["Bash(npm *)"]}}
```

```typescript
// 解析reg query输出提取JSON值
export function parseRegQueryStdout(stdout: string, valueName = 'Settings'): string | null {
  const lines = stdout.split(/\r?\n/)
  // 正则匹配：Settings    REG_SZ    {"json":"value"}
  const re = new RegExp(`^\\s+${valueName}\\s+REG_(?:EXPAND_)?SZ\\s+(.*)$`, 'i')
  
  for (const line of lines) {
    const match = line.match(re)
    if (match && match[1]) {
      return match[1].trimEnd()
    }
  }
  return null
}
```

```typescript
// systemd/sudoers风格的碎片化配置
export function loadManagedFileSettings(): {
  settings: SettingsJson | null
  errors: ValidationError[]
} {
  let merged: SettingsJson = {}
  
  // 1. 基础文件（最低优先级）
  const baseFile = join(getManagedFilePath(), 'managed-settings.json')
  merged = mergeWith(merged, baseSettings, settingsMergeCustomizer)
  
  // 2. Drop-in目录（按字母顺序合并，后文件覆盖前文件）
  const dropInDir = join(getManagedFilePath(), 'managed-settings.d')
  const entries = readdirSync(dropInDir)
    .filter(d => d.name.endsWith('.json') && !d.name.startsWith('.'))
    .sort()  // 字母排序
  
  for (const name of entries) {
    const dropInSettings = parseSettingsFile(join(dropInDir, name))
    merged = mergeWith(merged, dropInSettings, settingsMergeCustomizer)
  }
  
  return { settings: merged, errors }
}
```

### 策略优先级详解

源码位置：`utils/settings/settings.ts` 第645-796行

```typescript
function loadSettingsFromDisk(): SettingsWithErrors {
  let mergedSettings: SettingsJson = {}
  
  // 按优先级顺序合并设置源
  for (const source of getEnabledSettingSources()) {
    if (source === 'policySettings') {
      // "First-Source-Wins"策略
      
      // 1. Remote（最高优先级）
      const remoteSettings = getRemoteManagedSettingsSyncFromCache()
      if (remoteSettings && Object.keys(remoteSettings).length > 0) {
        policySettings = remoteSettings
        mergedSettings = mergeWith(mergedSettings, policySettings, ...)
        continue  // 找到最高优先级源，停止
      }
      
      // 2. MDM（HKLM / macOS plist）
      const mdmResult = getMdmSettings()
      if (Object.keys(mdmResult.settings).length > 0) {
        policySettings = mdmResult.settings
        mergedSettings = mergeWith(mergedSettings, policySettings, ...)
        continue
      }
      
      // 3. managed-settings.json + drop-ins
      const { settings: fileSettings } = loadManagedFileSettings()
      if (fileSettings) {
        policySettings = fileSettings
        mergedSettings = mergeWith(mergedSettings, policySettings, ...)
        continue
      }
      
      // 4. HKCU（最低优先级）
      const hkcu = getHkcuSettings()
      if (Object.keys(hkcu.settings).length > 0) {
        policySettings = hkcu.settings
        mergedSettings = mergeWith(mergedSettings, policySettings, ...)
      }
    }
    
    // 其他设置源（userSettings, projectSettings等）
    // 后续源覆盖前面源的配置
  }
  
  return { settings: mergedSettings, errors: allErrors }
}
```

**关键设计:**

1. **policySettings使用First-Source-Wins**：首个有内容的源提供所有策略
2. **其他设置源使用Merge**：后源覆盖前源，实现灵活配置
3. **数组配置合并去重**：`mergeArrays()`合并并去重数组元素

```typescript
// 计算checksum用于HTTP缓存
export function computeChecksumFromSettings(settings: SettingsJson): string {
  const sorted = sortKeysDeep(settings)  // 递归排序键
  const normalized = jsonStringify(sorted)  // 无空格分隔符
  const hash = createHash('sha256').update(normalized).digest('hex')
  return `sha256:${hash}`  // 必须匹配服务端Python实现
}
```

```typescript
async function fetchRemoteManagedSettings(cachedChecksum?: string) {
  const headers: Record<string, string> = {
    ...authHeaders.headers,
    'User-Agent': getClaudeCodeUserAgent(),
  }
  
  // 添加ETag缓存头
  if (cachedChecksum) {
    headers['If-None-Match'] = `"${cachedChecksum}"`
  }
  
  const response = await axios.get(endpoint, {
    headers,
    timeout: SETTINGS_TIMEOUT_MS,  // 10秒超时
    validateStatus: status =>
      status === 200 || status === 204 || status === 304 || status === 404
  })
  
  // 304 Not Modified: 缓存仍有效
  if (response.status === 304) {
    return {
      success: true,
      settings: null,  // 信号：缓存有效
      checksum: cachedChecksum,
    }
  }
  
  // 204/404: 无远程设置
  if (response.status === 204 || response.status === 404) {
    return { success: true, settings: {}, checksum: undefined }
  }
  
  // 200: 新设置内容
  return {
    success: true,
    settings: parsed.data.settings,
    checksum: parsed.data.checksum,
  }
}
```

```typescript
export async function checkManagedSettingsSecurity(
  cachedSettings: SettingsJson | null,
  newSettings: SettingsJson | null
): Promise<SecurityCheckResult> {
  // 新设置无危险项 → 无需检查
  if (!newSettings || !hasDangerousSettings(extractDangerousSettings(newSettings))) {
    return 'no_check_needed'
  }
  
  // 危险项未变更 → 无需检查
  if (!hasDangerousSettingsChanged(cachedSettings, newSettings)) {
    return 'no_check_needed'

  // 非交互模式 → 自动批准（与信任对话框一致）
  if (!getIsInteractive()) {
    return 'no_check_needed'

  // 显示阻塞式安全对话框

    render(
      <ManagedSettingsSecurityDialog
        settings={newSettings}
        onAccept={() => resolve('approved')}
        onReject={() => resolve('rejected')}
      />
    )
  })
}
```

```typescript
  'hooks',    // 钩子
```

```typescript
// 权限规则验证
export function validatePermissionRule(rule: string): {
  valid: boolean
  error?: string
  suggestion?: string
  examples?: string[]
} {
  // 格式：ToolName(pattern) 或 ToolName
  
  // MCP规则特殊处理
  const mcpInfo = mcpInfoFromString(parsed.toolName)
  if (mcpInfo) {
    // MCP不支持pattern，只能用serverName
    // mcp__server (服务器级)
    // mcp__server__* (全部工具)
    // mcp__server__tool (特定工具)
    if (parsed.ruleContent !== undefined) {
      return {
        valid: false,
        error: 'MCP rules do not support patterns in parentheses',
        suggestion: 'Use server name without parentheses'
      }
    }
    return { valid: true }
  }
  
  // Bash规则验证
  if (isBashPrefixTool(parsed.toolName)) {
    // 支持通配符匹配
    // Bash(npm *) - 通配符匹配
    // Bash(npm:*) - 传统前缀匹配（已废弃）
    if (content.includes(':*') && !content.endsWith(':*')) {
      return { valid: false, error: 'The :* pattern must be at the end' }
    }
  }
  
  // 文件工具验证
  if (isFilePatternTool(parsed.toolName)) {
    // 文件pattern使用glob
    // Read(src/**) - glob递归匹配
    // Write(*.ts) - 扩展名匹配
    if (content.includes(':*')) {
      return { valid: false, error: 'The ":*" syntax is only for Bash prefix rules' }
    }
  }
  
  return { valid: true }
}
```

```typescript
// 过滤无效权限规则（防止一个坏规则污染整个配置）
export function filterInvalidPermissionRules(
  data: unknown,
  filePath: string
): ValidationError[] {
  if (!data || typeof data !== 'object') return []
  const obj = data as Record<string, unknown>
  if (!obj.permissions || typeof obj.permissions !== 'object') return []
  
  const perms = obj.permissions as Record<string, unknown>
  const warnings: ValidationError[] = []
  
  for (const key of ['allow', 'deny', 'ask']) {
    const rules = perms[key]
    if (!Array.isArray(rules)) continue
    
    // 过滤无效规则
    perms[key] = rules.filter(rule => {
      if (typeof rule !== 'string') {
        warnings.push({
          file: filePath,
          path: `permissions.${key}`,
          message: `Non-string value in ${key} array was removed`,
          invalidValue: rule,
        })
        return false
      }
      
      const result = validatePermissionRule(rule)
      if (!result.valid) {
        let message = `Invalid permission rule "${rule}" was skipped`
        if (result.error) message += `: ${result.error}`
        if (result.suggestion) message += `. ${result.suggestion}`
        
        warnings.push({
          file: filePath,
          path: `permissions.${key}`,
          message,
          invalidValue: rule,
        })
        return false
      }
      
      return true
    })
  }
  
  return warnings
}
```

```yaml
  },
```

```json
  }
}
```

**优先级合并:**

```
Remote策略（最高） → 核心安全规则（deny, sandbox, strict）
本地文件策略 → 团队特定规则（allow, availableModels）
用户设置 → 个人偏好（不影响安全规则）
```

### 故障排查清单

1. **策略未生效**
   - 检查优先级：Remote > MDM > 文件 > HKCU
   - 验证文件权限：需要管理员/root写入
   - 运行`claude-code /status`查看生效源

2. **配置解析错误**
   - 运行`claude-code doctor`诊断
   - 检查JSON语法有效性
   - 验证权限规则格式

3. **热加载不触发**
   - 检查文件监控日志
   - 验证MDM轮询间隔（30分钟）
   - 确认文件写入稳定性（1秒阈值）

4. **安全对话框阻塞**
   - 确认用户交互模式
   - 检查危险配置变更
   - 验证`getIsInteractive()`状态
