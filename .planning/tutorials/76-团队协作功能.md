# Claude Code CLI 团队协作功能详解

**分析日期:** 2026-04-02

---

## 1. 团队协作概述

Claude Code CLI 的团队协作功能（Agent Swarms）允许多个 Claude 实例协同工作，形成一个团队来完成复杂任务。这个系统支持两种运行模式：

### 1.1 核心概念

| 概念 | 说明 |
|------|------|
| **Team Lead** | 团队领导者，负责协调任务、审批权限请求、管理团队成员 |
| **Teammate** | 团队成员，执行具体任务的 Claude 实例 |
| **Team File** | 团队配置文件，存储在 `~/.claude/teams/{team_name}/config.json` |
| **Mailbox** | 消息邮箱系统，用于团队成员间的异步通信 |
| **Backend** | 执行后端，支持 tmux、iTerm2 和 in-process 三种模式 |

### 1.2 执行模式对比

| 模式 | 描述 | 适用场景 |
|------|------|---------|
| **tmux** | 在 tmux 窗格中运行独立的 Claude 进程 | 终端环境，支持可视化布局 |
| **iTerm2** | 在 iTerm2 窗格中运行（需要 it2 CLI） | macOS iTerm2 用户 |
| **in-process** | 在同一 Node.js 进程中运行，使用 AsyncLocalStorage 隔离 | 资源高效，无终端依赖 |

### 1.3 文件结构

```
~/.claude/
├── teams/
│   └── {team_name}/
│       ├── config.json       # 团队配置文件
│       ├── inboxes/
│       │   ├── {agent_name}.json  # 各成员的邮箱
│       │   └── team-lead.json     # 领导者邮箱
│       └── permissions/
│           ├── pending/      # 待处理权限请求
│           └── resolved/     # 已解决权限请求
```

```typescript
export function isTeammate(): boolean {
  // In-process teammates run within the same process
  const inProcessCtx = getTeammateContext()
  if (inProcessCtx) return true
  // Tmux teammates require both agent ID and team name
  return !!(dynamicTeamContext?.agentId && dynamicTeamContext?.teamName)
}
```

**判断逻辑：**
- in-process 模式：检查 AsyncLocalStorage 是否有 teammate context
- tmux/iTerm2 模式：检查 dynamicTeamContext 是否同时存在 agentId 和 teamName

#### 2.2.5 isTeamLead() - 判断是否为团队领导

```typescript
// 第171-198行
export function isTeamLead(
  teamContext: { leadAgentId: string } | undefined,
): boolean {
  if (!teamContext?.leadAgentId) return false

  const myAgentId = getAgentId()
  const leadAgentId = teamContext.leadAgentId

  // If my agent ID matches the lead agent ID, I'm the lead
  if (myAgentId === leadAgentId) return true

  // Backwards compat: if no agent ID is set and we have a team context,
  // this is the original session that created the team (the lead)
  if (!myAgentId) return true

  return false
}
```

**关键点：**
- 领导者的 agentId 匹配 teamContext.leadAgentId
- 向后兼容：没有设置 agentId 但有 teamContext 的会话被认为是领导者

#### 2.2.6 等待队友空闲

```typescript
// 第238-292行 - waitForTeammatesToBecomeIdle()
export function waitForTeammatesToBecomeIdle(
  setAppState: (f: (prev: AppState) => AppState) => void,
  appState: AppState,

```

**用途：** 在发送关闭提示前等待所有工作中的 teammate 完成当前任务。

## 3. Teammate 模式详解

### 3.1 In-Process Teammate（进程内模式）

`utils/teammateContext.ts` 提供基于 AsyncLocalStorage 的上下文隔离：

```typescript

import { AsyncLocalStorage } from 'async_hooks'

export type TeammateContext = {
  agentId: string            // 完整ID
  agentName: string          // 显示名
  teamName: string           // 团队名
  color?: string             // UI颜色
  planModeRequired: boolean  // 计划模式要求

const teammateContextStorage = new AsyncLocalStorage<TeammateContext>()

export function getTeammateContext(): TeammateContext | undefined {
  return teammateContextStorage.getStore()
}

export function runWithTeammateContext<T>(
  context: TeammateContext,
  fn: () => T,

  return teammateContextStorage.run(context, fn)
```

**关键特性：**
- 使用 Node.js AsyncLocalStorage 实现并发隔离
- 多个 in-process teammate 可以在同一进程并发执行
- 上下文自动传播到所有异步操作

### 4.2 写入邮箱（带锁）

```typescript
// 第134-192行
export async function writeToMailbox(
  recipientName: string,
  message: Omit<TeammateMessage, 'read'>,
  teamName?: string,
): Promise<void> {
  await ensureInboxDir(teamName)

  const inboxPath = getInboxPath(recipientName, teamName)
  const lockFilePath = `${inboxPath}.lock`

  // 使用 proper-lockfile 防止并发写入冲突
  let release: (() => Promise<void>) | undefined
  try {
    release = await lockfile.lock(inboxPath, {
      lockfilePath: lockFilePath,
      retries: { retries: 10, minTimeout: 5, maxTimeout: 100 },
    })

    const messages = await readMailbox(recipientName, teamName)
    messages.push({ ...message, read: false })
    await writeFile(inboxPath, jsonStringify(messages, null, 2), 'utf-8')
  } finally {
    if (release) await release()
  }
}
```

### 4.3 结构化协议消息类型

系统定义了多种结构化消息类型，用于特定功能：

| 消息类型 | 方向 | 用途 |
|---------|------|------|
| `idle_notification` | Teammate → Leader | 通知空闲状态 |
| `permission_request` | Teammate → Leader | 权限审批请求 |
| `permission_response` | Leader → Teammate | 权限审批结果 |
| `sandbox_permission_request` | Teammate → Leader | 网络访问请求 |
| `sandbox_permission_response` | Leader → Teammate | 网络访问结果 |
| `shutdown_request` | Leader → Teammate | 关闭请求 |
| `shutdown_approved` | Teammate → Leader | 同意关闭 |
| `shutdown_rejected` | Teammate → Leader | 拒绝关闭 |
| `plan_approval_request` | Teammate → Leader | 计划审批请求 |
| `plan_approval_response` | Leader → Teammate | 计划审批结果 |
| `team_permission_update` | Leader → Teammates | 团队权限更新 |
| `mode_set_request` | Leader → Teammate | 模式变更请求 |

### 4.4 useInboxPoller 消息路由

`hooks/useInboxPoller.ts` 是消息轮询和路由的核心：

```typescript
// 每秒轮询未读消息
const INBOX_POLL_INTERVAL_MS = 1000

export function useInboxPoller({ enabled, isLoading, focusedInputDialog, onSubmitMessage }): void {
  // 主轮询逻辑
  const poll = useCallback(async () => {
    const unread = await readUnreadMessages(agentName, teamName)

    // 分离不同类型消息
    const permissionRequests: TeammateMessage[] = []
    const permissionResponses: TeammateMessage[] = []
    const shutdownRequests: TeammateMessage[] = []
    const regularMessages: TeammateMessage[] = []
    // ...

    for (const m of unread) {
      if (isPermissionRequest(m.text)) permissionRequests.push(m)
      else if (isShutdownRequest(m.text)) shutdownRequests.push(m)
      else regularMessages.push(m)
    }

    // 处理权限请求（领导者端）
    if (permissionRequests.length > 0 && isTeamLead(appState.teamContext)) {
      // 路由到 ToolUseConfirmQueue 显示审批对话框
      const setToolUseConfirmQueue = getLeaderToolUseConfirmQueue()
      for (const m of permissionRequests) {
        const parsed = isPermissionRequest(m.text)
        setToolUseConfirmQueue(queue => [...queue, {
          tool,
          workerBadge: { name: parsed.agent_id, color: 'cyan' },
          onAllow(updatedInput, permissionUpdates) {
            sendPermissionResponseViaMailbox(parsed.agent_id, { decision: 'approved' }, ...)
          },
          onReject(feedback) {
            sendPermissionResponseViaMailbox(parsed.agent_id, { decision: 'rejected' }, ...)
          }
        }])
      }
    }

    // 处理普通消息
    if (!isLoading && !focusedInputDialog) {
      onSubmitMessage(formatted) // 立即提交为新回合
    } else {
      queueMessages() // 排队等待
    }
  }, [enabled, isLoading, ...])
}
```

```typescript
// utils/swarm/inProcessRunner.ts - createInProcessCanUseTool()
function createInProcessCanUseTool(identity, abortController): CanUseToolFn {
  return async (tool, input, toolUseContext, ...) => {
    const result = await hasPermissionsToUseTool(tool, input, ...)

    if (result.behavior !== 'ask') return result

    // 使用领导者的 ToolUseConfirmQueue（共享UI）
    const setToolUseConfirmQueue = getLeaderToolUseConfirmQueue()

    if (setToolUseConfirmQueue) {
      return new Promise(resolve => {
        setToolUseConfirmQueue(queue => [...queue, {
          tool,
          workerBadge: { name: identity.agentName, color: identity.color },
          onAllow(updatedInput, permissionUpdates) {
            // 应用权限更新到领导者的上下文
            persistPermissionUpdates(permissionUpdates)
            const setToolPermissionContext = getLeaderSetToolPermissionContext()
            if (setToolPermissionContext) {
              const updatedContext = applyPermissionUpdates(...)
              setToolPermissionContext(updatedContext, { preserveMode: true })
            }
            resolve({ behavior: 'allow', updatedInput })
          },
          onReject(feedback) {
            resolve({ behavior: 'deny', message: feedback })
          }
        }])
      })
    }
  }
}
```

### 7.3 Step 2: 实现邮箱系统

```typescript
// teammateMailbox.ts
import { readFile, writeFile, mkdir } from 'fs/promises'
import * as lockfile from 'proper-lockfile'

export type TeammateMessage = {
  from: string
  text: string
  timestamp: string
  read: boolean
}

export function getInboxPath(agentName: string, teamName: string): string {
  return join(getTeamsDir(), teamName, 'inboxes', `${agentName}.json`)
}

export async function writeToMailbox(
  recipient: string,
  message: TeammateMessage,
  teamName: string,
): Promise<void> {
  const path = getInboxPath(recipient, teamName)
  await mkdir(join(getTeamsDir(), teamName, 'inboxes'), { recursive: true })

  const release = await lockfile.lock(path)
  try {
    const messages = await readMailbox(recipient, teamName)
    messages.push({ ...message, read: false })
    await writeFile(path, JSON.stringify(messages, null, 2))
  } finally {
    await release()
  }
}

export async function readUnreadMessages(
  agentName: string,
  teamName: string,
): Promise<TeammateMessage[]> {
  const messages = await readMailbox(agentName, teamName)
  return messages.filter(m => !m.read)
}
```

```typescript
export function readTeamFile(teamName: string): TeamFile | null {
  try {
    const content = readFileSync(getTeamFilePath(teamName), 'utf-8')
    return JSON.parse(content)
  } catch {
    return null
  }
}
```

```typescript
// useInboxPoller.ts
import { useInterval } from 'usehooks-ts'

export function useInboxPoller({ enabled, onSubmitMessage }) {
  const poll = async () => {
    if (!enabled) return

    const agentName = getAgentName()
    if (!agentName) return

    const unread = await readUnreadMessages(agentName, getTeamName())
    if (unread.length === 0) return

    // 格式化为 XML 消息
    const formatted = unread.map(m =>
      `<teammate_message teammate_id="${m.from}">\n${m.text}\n</teammate_message>`
    ).join('\n\n')

    onSubmitMessage(formatted)
    await markMessagesAsRead(agentName, getTeamName())
  }

  useInterval(() => void poll(), enabled ? 1000 : null)
}
```

```typescript
function createInProcessCanUseTool(identity): CanUseToolFn {
  return async (tool, input, ...) => {
    const result = await checkPermissions(tool, input)
    if (result.behavior !== 'ask') return result

    const setQueue = getLeaderToolUseConfirmQueue()
    if (!setQueue) {
      // 回退到邮箱系统
      return await requestPermissionViaMailbox(tool, input)
    }

    // 直接使用领导者的 UI
    return new Promise(resolve => {
      setQueue(queue => [...queue, {
        tool,
        workerBadge: { name: identity.agentName },
        onAllow: (input, updates) => resolve({ behavior: 'allow', input }),
        onReject: (feedback) => resolve({ behavior: 'deny', message: feedback })
      }])
    })
  }
}
```

### 7.7 关键设计要点

1. **并发安全：** 使用 `proper-lockfile` 保护邮箱文件写入
2. **上下文隔离：** 使用 `AsyncLocalStorage` 实现 in-process teammate 隔离
3. **消息优先级：** 结构化协议消息优先路由，普通消息作为上下文
4. **权限继承：** teammate 审批后的权限更新应用到领导者上下文
5. **向后兼容：** 领导者判断逻辑兼容没有 agentId 的旧会话
