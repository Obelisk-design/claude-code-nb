# 文件历史追踪系统

**分析日期:** 2026-04-02

## 1. 文件历史概述

### 1.1 系统定位

文件历史追踪系统（File History Tracking System）是 Claude Code CLI 的核心基础设施，提供了类似 Git 的文件版本控制能力，但专为 AI 辅助编程场景设计。它允许用户在任何对话节点进行"时光倒流"，将文件系统状态恢复到该节点对应的历史版本。

### 1.2 核心能力

| 能力 | 描述 |
|------|------|
| **快照追踪** | 每个用户消息节点自动创建文件快照 |
| **内容备份** | 文件修改前自动备份原始内容 |
| **差异计算** | 计算文件变化的行级统计（增/删行数） |
| **时光回滚** | 恢复到任意历史快照状态 |
| **会话恢复** | 跨会话保留文件历史，支持 resume |
| **VSCode 通知** | 文件变化时通知 VSCode 扩展 |

### 1.3 与 Git 的区别

| 特性 | Git | File History |
|------|-----|--------------|
| 触发时机 | 手动 commit | 每个用户消息自动触发 |
| 粒度 | 整个仓库 | 仅追踪被工具修改的文件 |
| 存储位置 | `.git/` | `~/.claude/file-history/{sessionId}/` |
| 回滚方式 | `git reset` | UI 点击消息节点 |
| 会话关联 | 无 | 与对话消息 UUID 绑定 |

### 2.4 备份文件命名规则

备份文件名采用 **内容哈希 + 版本号** 的确定性命名策略：

```typescript
function getBackupFileName(filePath: string, version: number): string {
  const fileNameHash = createHash('sha256')
    .update(filePath)
    .digest('hex')
    .slice(0, 16)  // 取前 16 字符
  return `${fileNameHash}@v${version}`
}
```

**示例：**
- 文件路径：`/project/src/index.ts`
- 版本 1 备份名：`a1b2c3d4e5f6g7h8@v1`
- 版本 2 备份名：`a1b2c3d4e5f6g7h8@v2`

**备份存储路径：**
```
~/.claude/file-history/{sessionId}/{hash}@v{version}
```

### 2.5 路径压缩优化

为减少会话存储空间，系统使用相对路径作为追踪键：

```typescript
function maybeShortenFilePath(filePath: string): string {
  if (!isAbsolute(filePath)) return filePath
  const cwd = getOriginalCwd()
  if (filePath.startsWith(cwd)) {
    return relative(cwd, filePath)  // 转为相对路径
  }
  return filePath
}

function maybeExpandFilePath(filePath: string): string {
  if (isAbsolute(filePath)) return filePath
  return join(getOriginalCwd(), filePath)  // 恢复为绝对路径
}
```

## 3. 快照机制

### 3.1 核心函数：fileHistoryMakeSnapshot

**位置：** `utils/fileHistory.ts:198-342`

**职责：** 在每个用户消息节点创建新快照，备份所有已追踪文件的当前状态。

**三阶段执行流程：**

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1: 同步捕获当前状态（无操作更新器）                          │
├─────────────────────────────────────────────────────────────────┤
│  updateFileHistoryState(state => {                              │
│    captured = state                                              │
│    return state  // 返回同一引用，避免不必要的渲染                  │
│  })                                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 2: 异步 IO 操作（文件备份）                                 │
├─────────────────────────────────────────────────────────────────┤
│  for each trackedFile:                                           │
│    - stat 文件检查是否存在                                         │
│    - 比较修改时间判断是否需要备份                                    │
│    - 如需要则创建新备份                                            │
│    - 否则复用最近备份                                              │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 3: 同步提交新状态                                          │
├─────────────────────────────────────────────────────────────────┤
│  updateFileHistoryState(state => {                              │
│    - 继承上一快照中已追踪但本次未修改的文件                           │
│    - 创建新快照并入队                                             │
│    - 如超过 MAX_SNAPSHOTS 则裁剪旧快照                             │
│    - 递增 snapshotSequence                                      │
│    - 记录快照到会话存储                                            │
│    - 通知 VSCode 文件变化                                         │
│    return updatedState                                           │
│  })                                                              │
└─────────────────────────────────────────────────────────────────┘
```

**关键代码解析：**

```typescript
export async function fileHistoryMakeSnapshot(
  updateFileHistoryState: (
    updater: (prev: FileHistoryState) => FileHistoryState,
  ) => void,
  messageId: UUID,
): Promise<void> {
  // Phase 1: 捕获当前状态
  let captured: FileHistoryState | undefined
  updateFileHistoryState(state => {
    captured = state
    return state  // 无操作返回，避免触发渲染
  })
  if (!captured) return

  // Phase 2: 异步备份
  const trackedFileBackups: Record<string, FileHistoryBackup> = {}
  const mostRecentSnapshot = captured.snapshots.at(-1)

  if (mostRecentSnapshot) {
    await Promise.all(
      Array.from(captured.trackedFiles, async trackingPath => {
        const filePath = maybeExpandFilePath(trackingPath)
        const latestBackup = mostRecentSnapshot.trackedFileBackups[trackingPath]
        const nextVersion = latestBackup ? latestBackup.version + 1 : 1

        // 检查文件是否存在
        let fileStats: Stats | undefined
        try {
          fileStats = await stat(filePath)
        } catch (e: unknown) {
          if (!isENOENT(e)) throw e
        }

        if (!fileStats) {
          // 文件被删除，记录 null 备份
          trackedFileBackups[trackingPath] = {
            backupFileName: null,
            version: nextVersion,
            backupTime: new Date(),
          }
          return
        }

        // 检查是否需要新备份（内容是否变化）
        if (
          latestBackup &&
          latestBackup.backupFileName !== null &&
          !(await checkOriginFileChanged(filePath, latestBackup.backupFileName, fileStats))
        ) {
          // 内容未变化，复用旧备份
          trackedFileBackups[trackingPath] = latestBackup
          return
        }

        // 内容已变化，创建新备份
        trackedFileBackups[trackingPath] = await createBackup(filePath, nextVersion)
      }),
    )
  }

  // Phase 3: 提交状态
  updateFileHistoryState((state: FileHistoryState) => {
    // 继承未在本轮处理的文件备份
    const lastSnapshot = state.snapshots.at(-1)
    if (lastSnapshot) {
      for (const trackingPath of state.trackedFiles) {
        if (trackingPath in trackedFileBackups) continue
        const inherited = lastSnapshot.trackedFileBackups[trackingPath]
        if (inherited) trackedFileBackups[trackingPath] = inherited
      }
    }

    const newSnapshot: FileHistorySnapshot = {
      messageId,
      trackedFileBackups,
      timestamp: new Date(),
    }

    const allSnapshots = [...state.snapshots, newSnapshot]
    const updatedState: FileHistoryState = {
      ...state,
      snapshots: allSnapshots.length > MAX_SNAPSHOTS
        ? allSnapshots.slice(-MAX_SNAPSHOTS)
        : allSnapshots,
      snapshotSequence: (state.snapshotSequence ?? 0) + 1,
    }

    // 持久化到会话存储
    void recordFileHistorySnapshot(messageId, newSnapshot, false)

    // 通知 VSCode
    void notifyVscodeSnapshotFilesUpdated(state, updatedState)

    return updatedState
  })
}
```

### 3.2 文件追踪：fileHistoryTrackEdit

**位置：** `utils/fileHistory.ts:86-193`

**职责：** 在文件被修改前，记录其当前状态（创建 v1 备份）。

**调用时机：** 所有写文件工具（Edit、Write、NotebookEdit、Bash sed）在执行修改前调用。

**三阶段执行流程：**

```
┌─────────────────────────────────────────────────────────────────┐
│  Phase 1: 检查是否需要备份                                         │
├─────────────────────────────────────────────────────────────────┤
│  - 获取最近快照                                                   │
│  - 如果文件已在快照中追踪，则跳过（后续 makeSnapshot 会处理）        │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 2: 创建 v1 备份（异步）                                     │
├─────────────────────────────────────────────────────────────────┤
│  backup = await createBackup(filePath, 1)                       │
│  - 如果文件不存在，backupFileName = null                          │
│  - 如果文件存在，复制到备份目录                                     │
└─────────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────────┐
│  Phase 3: 提交追踪记录                                             │
├─────────────────────────────────────────────────────────────────┤
│  - 将备份信息添加到最近快照的 trackedFileBackups                    │
│  - 添加文件路径到 trackedFiles 集合                                │
│  - 记录快照更新到会话存储                                           │
└─────────────────────────────────────────────────────────────────┘
```

**关键代码片段：**

```typescript
export async function fileHistoryTrackEdit(
  updateFileHistoryState: (
    updater: (prev: FileHistoryState) => FileHistoryState,
  ) => void,
  filePath: string,
  messageId: UUID,
): Promise<void> {
  if (!fileHistoryEnabled()) return

  const trackingPath = maybeShortenFilePath(filePath)

  // Phase 1: 检查是否已追踪
  let captured: FileHistoryState | undefined
  updateFileHistoryState(state => {
    captured = state
    return state
  })
  if (!captured) return

  const mostRecent = captured.snapshots.at(-1)
  if (!mostRecent) {
    logError(new Error('FileHistory: Missing most recent snapshot'))
    return
  }

  if (mostRecent.trackedFileBackups[trackingPath]) {
    // 已追踪，跳过（makeSnapshot 会处理后续版本）
    return
  }

  // Phase 2: 创建 v1 备份
  let backup: FileHistoryBackup
  try {
    backup = await createBackup(filePath, 1)
  } catch (error) {
    logError(error)
    return
  }

  // Phase 3: 提交
  updateFileHistoryState((state: FileHistoryState) => {
    const mostRecentSnapshot = state.snapshots.at(-1)
    if (!mostRecentSnapshot || mostRecentSnapshot.trackedFileBackups[trackingPath]) {
      return state  // 被其他操作抢先
    }

    const updatedTrackedFiles = state.trackedFiles.has(trackingPath)
      ? state.trackedFiles
      : new Set(state.trackedFiles).add(trackingPath)

    const updatedMostRecentSnapshot = {
      ...mostRecentSnapshot,
      trackedFileBackups: {
        ...mostRecentSnapshot.trackedFileBackups,
        [trackingPath]: backup,
      },
    }

    // 更新快照数组
    const copy = state.snapshots.slice()
    copy[copy.length - 1] = updatedMostRecentSnapshot

    const updatedState = {
      ...state,
      snapshots: copy,
      trackedFiles: updatedTrackedFiles,
    }

    // 持久化
    void recordFileHistorySnapshot(messageId, updatedMostRecentSnapshot, true)

    return updatedState
  })
}
```

### 3.3 备份创建：createBackup

**位置：** `utils/fileHistory.ts:748-798`

```typescript
async function createBackup(
  filePath: string | null,
  version: number,
): Promise<FileHistoryBackup> {
  // 文件不存在的情况
  if (filePath === null) {
    return { backupFileName: null, version, backupTime: new Date() }
  }

  const backupFileName = getBackupFileName(filePath, version)
  const backupPath = resolveBackupPath(backupFileName)

  // 先检查源文件是否存在
  let srcStats: Stats
  try {
    srcStats = await stat(filePath)
  } catch (e: unknown) {
    if (isENOENT(e)) {
      return { backupFileName: null, version, backupTime: new Date() }
    }
    throw e
  }

  // 使用 copyFile 避免大文件 OOM
  // 延迟创建目录（99% 情况目录已存在）
  try {
    await copyFile(filePath, backupPath)
  } catch (e: unknown) {
    if (!isENOENT(e)) throw e
    await mkdir(dirname(backupPath), { recursive: true })
    await copyFile(filePath, backupPath)
  }

  // 保留文件权限
  await chmod(backupPath, srcStats.mode)

  return { backupFileName, version, backupTime: new Date() }
}
```

**设计要点：**

1. **避免 OOM**：使用 `copyFile` 而非 `readFile + writeFile`，大文件不会加载到内存
2. **延迟目录创建**：先尝试复制，失败时才创建目录，优化热路径
3. **权限保留**：备份文件保持与源文件相同的权限位
4. **ENOENT 处理**：文件不存在时记录 `null` 备份，表示"此版本文件不存在"

### 3.4 变化检测：checkOriginFileChanged

**位置：** `utils/fileHistory.ts:600-634`

```typescript
export async function checkOriginFileChanged(
  originalFile: string,
  backupFileName: string,
  originalStatsHint?: Stats,
): Promise<boolean> {
  const backupPath = resolveBackupPath(backupFileName)

  // 获取双方 stat
  let originalStats: Stats | null = originalStatsHint ?? null
  if (!originalStats) {
    try {
      originalStats = await stat(originalFile)
    } catch (e: unknown) {
      if (!isENOENT(e)) return true
    }
  }

  let backupStats: Stats | null = null
  try {
    backupStats = await stat(backupPath)
  } catch (e: unknown) {
    if (!isENOENT(e)) return true
  }

  return compareStatsAndContent(originalStats, backupStats, async () => {
    // 需要比较内容
    const [originalContent, backupContent] = await Promise.all([
      readFile(originalFile, 'utf-8'),
      readFile(backupPath, 'utf-8'),
    ])
    return originalContent !== backupContent
  })
}
```

**优化策略（分层检测）：**

```typescript
function compareStatsAndContent<T extends boolean | Promise<boolean>>(
  originalStats: Stats | null,
  backupStats: Stats | null,
  compareContent: () => T,
): T | boolean {
  // 1. 存在性检测
  if ((originalStats === null) !== (backupStats === null)) {
    return true  // 一方存在一方不存在
  }
  if (originalStats === null || backupStats === null) {
    return false  // 都不存在
  }

  // 2. 元数据检测
  if (
    originalStats.mode !== backupStats.mode ||
    originalStats.size !== backupStats.size
  ) {
    return true  // 权限或大小不同
  }

  // 3. 修改时间优化（最关键的优化）
  if (originalStats.mtimeMs < backupStats.mtimeMs) {
    return false  // 文件比备份旧，一定没变
  }

  // 4. 内容比较（最后的兜底）
  return compareContent()
}
```

### 4.2 快照应用：applySnapshot

**位置：** `utils/fileHistory.ts:537-591`

```typescript
async function applySnapshot(
  state: FileHistoryState,
  targetSnapshot: FileHistorySnapshot,
): Promise<string[]> {
  const filesChanged: string[] = []

  for (const trackingPath of state.trackedFiles) {
    const filePath = maybeExpandFilePath(trackingPath)
    const targetBackup = targetSnapshot.trackedFileBackups[trackingPath]

    // 解析备份文件名
    const backupFileName: BackupFileName | undefined = targetBackup
      ? targetBackup.backupFileName
      : getBackupFileNameFirstVersion(trackingPath, state)

    if (backupFileName === undefined) {
      // 无法解析备份，跳过
      continue
    }

    if (backupFileName === null) {
      // 目标版本文件不存在 -> 删除当前文件
      try {
        await unlink(filePath)
        filesChanged.push(filePath)
      } catch (e: unknown) {
        if (!isENOENT(e)) throw e
      }
      continue
    }

    // 文件应存在，检查是否需要恢复
    if (await checkOriginFileChanged(filePath, backupFileName)) {
      await restoreBackup(filePath, backupFileName)
      filesChanged.push(filePath)
    }
  }

  return filesChanged
}
```

### 4.4 差异统计：fileHistoryGetDiffStats

```typescript
export async function fileHistoryGetDiffStats(
  state: FileHistoryState,
  messageId: UUID,
): Promise<DiffStats> {
  const targetSnapshot = state.snapshots.findLast(
    snapshot => snapshot.messageId === messageId,
  )
  if (!targetSnapshot) return undefined

  const results = await Promise.all(
    Array.from(state.trackedFiles, async trackingPath => {
      const filePath = maybeExpandFilePath(trackingPath)
      const targetBackup = targetSnapshot.trackedFileBackups[trackingPath]

      const backupFileName = targetBackup
        ? targetBackup.backupFileName
        : getBackupFileNameFirstVersion(trackingPath, state)

      if (backupFileName === undefined) return null

      const stats = await computeDiffStatsForFile(
        filePath,
        backupFileName === null ? undefined : backupFileName,
      )

      if (stats?.insertions || stats?.deletions) {
        return { filePath, stats }
      }
      return null
    }),
  )

  // 汇总统计
  const filesChanged: string[] = []
  let insertions = 0
  let deletions = 0
  for (const r of results) {
    if (!r) continue
    filesChanged.push(r.filePath)
    insertions += r.stats?.insertions || 0
    deletions += r.stats?.deletions || 0
  }

  return { filesChanged, insertions, deletions }
}
```

### 4.5 轻量变化检测：fileHistoryHasAnyChanges

**位置：** `utils/fileHistory.ts:494-531`

```typescript
export async function fileHistoryHasAnyChanges(
  state: FileHistoryState,
  messageId: UUID,
): Promise<boolean> {
  if (!fileHistoryEnabled()) return false

  const targetSnapshot = state.snapshots.findLast(
    snapshot => snapshot.messageId === messageId,
  )
  if (!targetSnapshot) return false

  for (const trackingPath of state.trackedFiles) {
    const filePath = maybeExpandFilePath(trackingPath)
    const targetBackup = targetSnapshot.trackedFileBackups[trackingPath]
    const backupFileName = targetBackup
      ? targetBackup.backupFileName
      : getBackupFileNameFirstVersion(trackingPath, state)

    if (backupFileName === undefined) continue
    if (backupFileName === null) {
      if (await pathExists(filePath)) return true
      continue
    }
    if (await checkOriginFileChanged(filePath, backupFileName)) return true
  }

  return false
}
```

### 5.4 存储限制

- **快照数量上限**：100 个（超出后裁剪最旧的）
- **备份存储位置**：`~/.claude/file-history/{sessionId}/`
- **会话隔离**：每个会话有独立的备份目录

### 5.5 会话恢复支持

当用户使用 `--resume` 恢复会话时，系统会：

1. 从会话存储加载 `FileHistorySnapshot[]`
2. 调用 `fileHistoryRestoreStateFromLog` 重建状态
3. 调用 `copyFileHistoryForResume` 复制备份文件到新会话

**复制逻辑：**

```typescript
export async function copyFileHistoryForResume(log: LogOption): Promise<void> {
  const previousSessionId = lastMessage?.sessionId
  const sessionId = getSessionId()

  if (previousSessionId === sessionId) return  // 同一会话，无需复制

  // 创建新会话备份目录
  const newBackupDir = join(getClaudeConfigHomeDir(), 'file-history', sessionId)
  await mkdir(newBackupDir, { recursive: true })

  // 硬链接复制所有备份文件
  for (const snapshot of fileHistorySnapshots) {
    for (const backup of Object.values(snapshot.trackedFileBackups)) {
      if (backup.backupFileName === null) continue

      const oldPath = resolveBackupPath(backup.backupFileName, previousSessionId)
      const newPath = join(newBackupDir, backup.backupFileName)

      try {
        await link(oldPath, newPath)  // 硬链接
      } catch (e: unknown) {
        if (getErrnoCode(e) === 'EEXIST') continue  // 已存在
        await copyFile(oldPath, newPath)  // 回退到复制
      }
    }
  }
}
```

// fileHistory.ts
import { createHash } from 'crypto'
import { diffLines } from 'diff'
import { copyFile, mkdir, readFile, stat, unlink } from 'fs/promises'
import { dirname, join } from 'path'

// ==================== 类型定义 ====================

type BackupFileName = string | null

export type FileHistoryBackup = {
  backupFileName: BackupFileName
  version: number
  backupTime: Date
}

export type FileHistorySnapshot = {
  messageId: string
  trackedFileBackups: Record<string, FileHistoryBackup>
  timestamp: Date
}

export type FileHistoryState = {
  snapshots: FileHistorySnapshot[]
  trackedFiles: Set<string>
  snapshotSequence: number
}

const MAX_SNAPSHOTS = 100

// ==================== 核心函数 ====================

/**
 * 检查功能是否启用
 */
export function fileHistoryEnabled(): boolean {
  return process.env.CLAUDE_CODE_DISABLE_FILE_CHECKPOINTING !== '1'
}

/**
 * 追踪文件修改（修改前调用）
 */
export async function fileHistoryTrackEdit(
  updateFileHistoryState: (updater: (prev: FileHistoryState) => FileHistoryState) => void,
  filePath: string,
  messageId: string,
): Promise<void> {
  if (!fileHistoryEnabled()) return

  let captured: FileHistoryState | undefined
  updateFileHistoryState(state => {
    captured = state
    return state
  })
  if (!captured) return

  const mostRecent = captured.snapshots.at(-1)
  if (!mostRecent) return
  if (mostRecent.trackedFileBackups[filePath]) return  // 已追踪

  const backup = await createBackup(filePath, 1)

  updateFileHistoryState(state => {
    const snapshot = state.snapshots.at(-1)
    if (!snapshot || snapshot.trackedFileBackups[filePath]) return state

    return {
      ...state,
      snapshots: (() => {
        const copy = state.snapshots.slice()
        copy[copy.length - 1] = {
          ...snapshot,
          trackedFileBackups: {
            ...snapshot.trackedFileBackups,
            [filePath]: backup,
          },
        }
        return copy
      })(),
      trackedFiles: new Set(state.trackedFiles).add(filePath),
    }
  })
}

/**
 * 创建快照（每个消息节点调用）
 */
export async function fileHistoryMakeSnapshot(
  updateFileHistoryState: (updater: (prev: FileHistoryState) => FileHistoryState) => void,
  messageId: string,
): Promise<void> {
  if (!fileHistoryEnabled()) return

  let captured: FileHistoryState | undefined
  updateFileHistoryState(state => {
    captured = state
    return state
  })
  if (!captured) return

  const backups: Record<string, FileHistoryBackup> = {}
  const recentSnapshot = captured.snapshots.at(-1)

  if (recentSnapshot) {
    await Promise.all(
      Array.from(captured.trackedFiles, async filePath => {
        const latest = recentSnapshot.trackedFileBackups[filePath]
        const nextVersion = latest ? latest.version + 1 : 1

        try {
          const fileStats = await stat(filePath)
          if (latest?.backupFileName && !await checkOriginFileChanged(filePath, latest.backupFileName, fileStats)) {
            backups[filePath] = latest  // 复用旧备份
          } else {
            backups[filePath] = await createBackup(filePath, nextVersion)
          }
        } catch {
          backups[filePath] = { backupFileName: null, version: nextVersion, backupTime: new Date() }
        }
      }),
    )
  }

  updateFileHistoryState(state => {
    // 继承未处理的文件
    const lastSnapshot = state.snapshots.at(-1)
    if (lastSnapshot) {
      for (const path of state.trackedFiles) {
        if (!(path in backups) && lastSnapshot.trackedFileBackups[path]) {
          backups[path] = lastSnapshot.trackedFileBackups[path]
        }
      }
    }

    const newSnapshot: FileHistorySnapshot = {
      messageId,
      trackedFileBackups: backups,
      timestamp: new Date(),
    }

    const allSnapshots = [...state.snapshots, newSnapshot]
    return {
      ...state,
      snapshots: allSnapshots.length > MAX_SNAPSHOTS
        ? allSnapshots.slice(-MAX_SNAPSHOTS)
        : allSnapshots,
      snapshotSequence: state.snapshotSequence + 1,
    }
  })
}

/**
 * 回滚到指定快照
 */
export async function fileHistoryRewind(
  updateFileHistoryState: (updater: (prev: FileHistoryState) => FileHistoryState) => void,
  messageId: string,
): Promise<void> {
  if (!fileHistoryEnabled()) return

  let captured: FileHistoryState | undefined
  updateFileHistoryState(state => {
    captured = state
    return state
  })
  if (!captured) return

  const target = captured.snapshots.findLast(s => s.messageId === messageId)
  if (!target) throw new Error('Snapshot not found')

  for (const filePath of captured.trackedFiles) {
    const backup = target.trackedFileBackups[filePath]

    if (backup?.backupFileName === null) {
      // 目标版本文件不存在，删除当前文件
      try { await unlink(filePath) } catch {}
    } else if (backup?.backupFileName) {
      if (await checkOriginFileChanged(filePath, backup.backupFileName)) {
        await restoreBackup(filePath, backup.backupFileName)
      }
    }
  }
}

/**
 * 检查是否可以恢复
 */
export function fileHistoryCanRestore(state: FileHistoryState, messageId: string): boolean {
  return state.snapshots.some(s => s.messageId === messageId)
}

// ==================== 辅助函数 ====================

function getBackupFileName(filePath: string, version: number): string {
  const hash = createHash('sha256').update(filePath).digest('hex').slice(0, 16)
  return `${hash}@v${version}`
}

function resolveBackupPath(backupFileName: string): string {
  return join(process.env.CLAUDE_CONFIG_HOME || '.', 'file-history', getSessionId(), backupFileName)
}

function getSessionId(): string {
  return 'session-id'  // 实际从 bootstrap/state 获取
}

async function createBackup(filePath: string, version: number): Promise<FileHistoryBackup> {
  const backupFileName = getBackupFileName(filePath, version)
  const backupPath = resolveBackupPath(backupFileName)

  try {
    const srcStats = await stat(filePath)
    try {
      await copyFile(filePath, backupPath)
    } catch {
      await mkdir(dirname(backupPath), { recursive: true })
      await copyFile(filePath, backupPath)
    }
    return { backupFileName, version, backupTime: new Date() }
  } catch {
    return { backupFileName: null, version, backupTime: new Date() }
  }
}

async function restoreBackup(filePath: string, backupFileName: string): Promise<void> {
  const backupPath = resolveBackupPath(backupFileName)
  try {
    await copyFile(backupPath, filePath)
  } catch {
    await mkdir(dirname(filePath), { recursive: true })
    await copyFile(backupPath, filePath)
  }
}

async function checkOriginFileChanged(
  originalFile: string,
  backupFileName: string,
  hint?: Stats,
): Promise<boolean> {
  const backupPath = resolveBackupPath(backupFileName)

  const originalStats = hint ?? await stat(originalFile).catch(() => null)
  const backupStats = await stat(backupPath).catch(() => null)

  if (!originalStats || !backupStats) return originalStats !== backupStats
  if (originalStats.size !== backupStats.size) return true
  if (originalStats.mtimeMs < backupStats.mtimeMs) return false

  const [orig, backup] = await Promise.all([
    readFile(originalFile, 'utf-8'),
    readFile(backupPath, 'utf-8'),
  ])
  return orig !== backup
}

## 7. 总结

### 7.1 设计亮点

1. **确定性命名**：基于内容哈希的备份文件名，避免重复备份
2. **延迟创建目录**：99% 情况下避免不必要的 mkdir 系统调用
3. **分层变化检测**：mtime 优化避免大量文件内容比较
4. **三阶段异步**：同步捕获状态 → 异步 IO → 同步提交，避免竞态
5. **会话隔离**：每个会话独立备份目录，支持 resume 跨会话复制

### 7.2 关键数据流

```
用户消息
    │
    ├──> fileHistoryMakeSnapshot (为所有追踪文件创建快照)
    │         │
    │         └──> fileHistoryTrackEdit 调用积累的 trackedFiles
    │
工具调用
    │
    ├──> fileHistoryTrackEdit (记录 v1 备份)
    │
    └──> 实际文件修改
```
