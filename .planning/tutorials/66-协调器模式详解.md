## 1. 协调器概述

### 1.1 核心概念

协调器模式(Coordinator Mode)是Claude Code CLI的多代理编排系统。它将单一Claude代理转变为**任务分发协调中心**，协调器本身不直接执行文件操作或代码编写，而是调度多个Worker代理并行处理复杂任务。

**架构特点:**
- **分层设计**: 协调器(编排层) + Worker(执行层)
- **并行处理**: 多个Worker同时运行独立任务
- **异步通信**: Worker完成后发送通知，协调器汇总结果
- **工具隔离**: 协调器仅有编排工具，Worker拥有执行工具

### 1.2 激活机制

协调器模式通过双重开关控制：

```typescript
// coordinator/coordinatorMode.ts:36-41

    return isEnvTruthy(process.env.CLAUDE_CODE_COORDINATOR_MODE)  // 环境变量

```

**启用方式:**
```bash
# 方式一: 环境变量直接启用
CLAUDE_CODE_COORDINATOR_MODE=1 claude-code

# 方式二: 在会话中切换
# 系统自动检测并同步存储的会话模式
```

### 1.3 会话恢复同步

恢复会话时自动匹配模式状态：

```typescript
// coordinator/coordinatorMode.ts:49-78
export function matchSessionMode(
  sessionMode: 'coordinator' | 'normal' | undefined,
): string | undefined {
  if (!sessionMode) return undefined  // 旧会话无模式记录

  const currentIsCoordinator = isCoordinatorMode()
  const sessionIsCoordinator = sessionMode === 'coordinator'

  if (currentIsCoordinator === sessionIsCoordinator) return undefined

  // 翻转环境变量匹配会话
  if (sessionIsCoordinator) {
    process.env.CLAUDE_CODE_COORDINATOR_MODE = '1'
  } else {
    delete process.env.CLAUDE_CODE_COORDINATOR_MODE
  }

  return sessionIsCoordinator
    ? 'Entered coordinator mode to match resumed session.'
    : 'Exited coordinator mode to match resumed session.'
}
```

```typescript
// tasks/LocalAgentTask/LocalAgentTask.tsx:197-262
export function enqueueAgentNotification({
  taskId,
  description,
  status,
  finalMessage,
  usage,
}): void {
  // 防重复通知
  let shouldEnqueue = false
  updateTaskState(taskId, setAppState, task => {
    if (task.notified) return task
    shouldEnqueue = true
    return { ...task, notified: true }
  })
  if (!shouldEnqueue) return

  // 构建通知消息
  const summary = status === 'completed' 
    ? `Agent "${description}" completed`
    : status === 'failed' 
      ? `Agent "${description}" failed: ${error}`
      : `Agent "${description}" was stopped`

  const message = `<task-notification>
    <task-id>${taskId}</task-id>
    <output-file>${outputPath}</output-file>
    <status>${status}</status>
    <summary>${summary}</summary>
    ${resultSection}${usageSection}
  </task-notification>`

  enqueuePendingNotification({ value: message, mode: 'task-notification' })
}
```

### 5.3 Worker生命周期管理

```typescript
// tools/AgentTool/agentToolUtils.ts:508-686
export async function runAsyncAgentLifecycle({
  taskId,
  abortController,
  makeStream,
  metadata,
}): Promise<void> {
  const agentMessages: Message[] = []
  
  try {
    // 1. 创建进度跟踪器
    const tracker = createProgressTracker()
    
    // 2. 运行Agent流
    for await (const message of makeStream()) {
      agentMessages.push(message)
      updateProgressFromMessage(tracker, message)
      updateAsyncAgentProgress(taskId, getProgressUpdate(tracker))
    }

    // 3. 完成任务
    const agentResult = finalizeAgentTool(agentMessages, taskId, metadata)
    completeAsyncAgent(agentResult, rootSetAppState)

    // 4. 安全检查(可选)
    if (feature('TRANSCRIPT_CLASSIFIER')) {
      const handoffWarning = await classifyHandoffIfNeeded(...)
      if (handoffWarning) finalMessage = `${handoffWarning}\n${finalMessage}`
    }

    // 5. 发送通知
    enqueueAgentNotification({
      taskId,
      status: 'completed',
      finalMessage,
      usage: {...},
    })

  } catch (error) {
    if (error instanceof AbortError) {
      killAsyncAgent(taskId, rootSetAppState)
      enqueueAgentNotification({ taskId, status: 'killed' })
    } else {
      failAsyncAgent(taskId, errorMessage(error), rootSetAppState)
      enqueueAgentNotification({ taskId, status: 'failed', error })
    }
  }
}
```

```typescript
      tools,
    })
    
    // 5. 处理响应(包括task-notification)
    for (const message of response) {
      if (message.type === 'user' && 
          message.content.includes('<task-notification>')) {
        // Worker完成通知
        const result = parseNotification(message.content)
        history.push(message)  // 协调器看到通知
      } else {
        history.push(message)
      }
    }
    
    // 6. 显示给用户
    renderOutput(response)
  }
}
```

## 9. 设计亮点总结

1. **严格分层**: 协调器只编排，Worker只执行，职责清晰
2. **异步解耦**: Worker独立运行，通知驱动通信
3. **并行优先**: 单消息多Agent调用实现真正并行
4. **综合责任**: 协调器必须理解结果，不能转发理解任务
5. **工具隔离**: 协调器工具池严格过滤，避免越界操作
6. **会话持久**: 模式状态存储，恢复时自动同步
7. **权限自动化**: Worker权限检查通过hooks+classifier自动处理
